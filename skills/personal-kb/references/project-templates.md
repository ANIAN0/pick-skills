# 项目Wiki结构模板

各项目类型的Wiki目录结构参考。模板是起点——根据项目实际情况调整，不要机械套用。

## 通用原则

- 初始只创建有内容的分类
- 层级不超过3层
- 使用中文命名（除非项目约定英文）
- 每个目录下可放 `_index.md` 作为该分类的概览页

**说明：** 通用型模板故意保持最小——分类应在内容积累后从实际需求中归纳，而非预设。

## 混合类型项目

很多项目同时具有多种特征。处理方式：

1. **主类型优先**：以最核心的项目类型为骨架
2. **按需叠加**：从其他类型中选取匹配的子分类
3. **用户确认**：混合结构的初始设计应向用户确认

示例：一个使用Python的Web服务项目，可以组合：
- 后端服务模板的"接口/"、"数据/"、"运维/"
- Python项目模板的"环境配置/"、"依赖管理/"

# 个人知识库管理 Skill

基于 LLM Wiki 模式和 Obsidian CLI 的个人知识库管理系统。你的职责是维护一个增量增长、相互关联的知识库，让用户的知识持续积累而非散落。

## 核心理念

知识库的价值在于**持续积累和关联**，而非简单的检索。你的工作是：
- 将新内容摄入并整合到现有知识结构中
- 维护页面间的交叉引用
- 保持知识的一致性和时效性
- 让每次查询和探索都能沉淀回知识库

## 目录结构

知识库采用以下目录结构：

```
知识库根目录/
├── 收件箱/           # 新内容入口
│   ├── 剪藏/         # 网页、文章保存的内容
│   ├── 灵感/         # 自己的想法、日记、随笔
│   └── 文件/         # 他人发送的文档、邮件
├── 待处理/           # 需要处理的内容
│   ├── 工作/         # 有截止时间、需产出的内容（按项目分）
│   └── 学习/         # 长期积累、无截止日的内容（按主题分）
├── 待总结/           # 处理完成后需加工的内容
│   ├── 待整理/       # 简单整理后即可归档
│   └── 待深度总结/   # 需要写输出、建立关联
├── 行动/             # 从笔记中提取的任务和待办
├── 参考/             # 无需处理、可直接使用的参考资料
├── 归档/             # 已完成总结、按时间组织
└── index.md          # 知识库索引（内容导向）
└── log.md            # 操作日志（时间导向）
```

## 流转规则

1. **所有新内容先放入收件箱**
2. **定期清理收件箱** → 移入待处理或参考
3. **待处理完成后** → 根据是否需要总结，移入待总结或直接归档
4. **待总结完成后** → 移入归档
5. **行动中完成的任务** → 移入归档

## Obsidian CLI 命令

使用 Obsidian CLI 进行知识库操作。前提：Obsidian 设置中启用 CLI（Settings → General → Command line interface）。

### 常用命令

```bash
# 创建笔记
obsidian create name="笔记名称" content="# 标题\n\n正文内容"

# 创建并打开笔记
obsidian create name="笔记名称" content="内容" open

# 搜索知识库
obsidian search query="搜索关键词"

# 读取当前活动文件
obsidian read

# 打开日记
obsidian daily

# 向日记追加内容
obsidian daily:append content="- [ ] 任务内容"

# 列出任务
obsidian tasks daily

# 从模板创建笔记
obsidian create name="笔记名" template=模板名

# 列出标签及计数
obsidian tags counts

# 获取vault信息
obsidian vault info=path
obsidian vault info=files
obsidian vault info=folders

# 列出所有vault
obsidian vaults verbose

# 指定vault操作
obsidian vault="Vault名称" search query="关键词"
```

### 参数说明

- **参数**：格式为 `parameter=value`，值有空格时用引号包裹
- **标志**：布尔开关，如 `open`、`overwrite`
- **多行内容**：用 `\n` 表示换行，`\t` 表示制表符

### 文件定位

- `file=<名称>`：通过wikilink解析，不需要完整路径或扩展名
- `path=<路径>`：需要从vault根目录的精确路径，如 `folder/note.md`

## 核心操作

### 1. 摄入新内容（Ingest）

当用户添加新内容时：

**步骤：**
1. 将内容放入对应的收件箱子目录（剪藏/灵感/文件）
2. 为内容创建初始笔记，包含：
   - 来源信息（URL、作者、日期）
   - 核心摘要
   - 关键标签
3. 更新 `index.md`，添加新笔记的索引条目
4. 在 `log.md` 中追加摄入记录
5. **主动关联**：检查是否与现有笔记相关，添加交叉引用

**摄入记录格式（log.md）：**
```markdown
## [YYYY-MM-DD] ingest | 笔记名称
- 来源：[来源类型]
- 关键点：[2-3个核心要点]
- 关联：[相关笔记链接]
```

### 2. 处理内容

将收件箱内容移入待处理目录：

**判断标准：**
- **待处理/工作**：有明确截止时间、需要产出
- **待处理/学习**：长期积累、无截止日
- **参考**：无需处理、直接可用

**处理步骤：**
1. 阅读并理解内容
2. 提取关键信息，整理成结构化笔记
3. 建立与现有知识的关联
4. 将笔记移动到对应目录
5. 更新 index.md 和 log.md

### 3. 查询知识（Query）

当用户提问时：

**步骤：**
1. 先读取 `index.md` 了解知识库结构
2. 使用 Obsidian CLI 搜索相关内容
3. 读取相关笔记，综合回答
4. **重要**：有价值的回答应沉淀为新笔记或更新现有笔记

**查询记录格式（log.md）：**
```markdown
## [YYYY-MM-DD] query | 查询主题
- 问题：[用户问题]
- 涉及笔记：[相关笔记链接]
- 沉淀：[是否创建/更新了笔记]
```

### 4. 健康检查（Lint）

定期维护知识库：

**检查项：**
- 孤立笔记：没有入链的页面
- 矛盾内容：不同笔记间的冲突信息
- 过时信息：已被新内容取代的旧信息
- 缺失页面：提及但未创建的概念页面
- 断链：指向不存在页面的链接
- 标签一致性：相似概念使用不同标签

**使用命令辅助检查：**
```bash
# 列出所有标签，检查一致性
obsidian tags counts

# 搜索特定概念，发现关联
obsidian search query="概念关键词"
```

## 索引文件格式

### index.md

内容导向的知识库目录：

```markdown
# 知识库索引

## 实体
- [[人物A]] - 一句话描述
- [[项目B]] - 一句话描述

## 概念
- [[概念X]] - 一句话描述
- [[概念Y]] - 一句话描述

## 来源
- [[文章标题]] - 来源、日期、关键点

## 主题
- [[主题A]] - 相关笔记数量、核心内容
```

### log.md

时间导向的操作记录：

```markdown
# 操作日志

## [2026-04-12] ingest | AI发展动态
- 来源：剪藏
- 关键点：多模态进展、Agent应用
- 关联：[[人工智能]]、[[Agent架构]]

## [2026-04-12] query | 最佳笔记方法
- 问题：如何组织个人知识
- 涉及笔记：[[知识管理]]、[[卡片笔记法]]
- 沉淀：更新了知识管理笔记
```

## 最佳实践

### 关联优先

每次处理新内容时，主动寻找关联：
- 提到的概念是否已有页面？没有则创建
- 是否与现有内容有联系？添加双向链接
- 是否有冲突或补充？标注并处理

### 沉淀查询

好的查询不应消失在聊天历史中：
- 有价值的分析 → 创建新笔记
- 发现的关联 → 更新相关笔记
- 提出的洞见 → 记录在对应主题下

### 保持时效

定期检查并更新：
- 新信息是否取代旧结论？标注并修订
- 项目状态是否变化？更新进度
- 关联是否失效？修复或删除

### 标签规范

使用一致的标签体系：
- 主题标签：`#主题/子主题`
- 状态标签：`#状态/待处理`、`#状态/已完成`
- 类型标签：`#类型/概念`、`#类型/来源`、`#类型/人物`

## 工作流程示例

### 摄入一篇新文章

```bash
# 1. 创建笔记
obsidian create name="文章标题" content="# 文章标题\n\n## 摘要\n[核心内容]\n\n## 关键点\n- [要点1]\n- [要点2]\n\n## 来源\n- URL: [链接]\n- 作者: [作者]\n- 日期: [日期]\n\n## 关联\n- [[相关概念1]]\n- [[相关概念2]]\n\n#主题/AI #类型/来源 #状态/待处理"

# 2. 更新索引（手动或通过CLI读取后编辑）
# 3. 记录日志
```

### 整理收件箱

```bash
# 查看vault文件列表
obsidian vault info=files

# 搜索收件箱内容
obsidian search query="收件箱" path="收件箱"

# 根据内容类型移动到对应目录
```

### 回答用户问题

```bash
# 搜索相关内容
obsidian search query="用户问题关键词"

# 根据搜索结果读取笔记，综合回答
# 将有价值的回答沉淀回知识库
```

## 注意事项

1. **不要修改原始来源文件**：收件箱中的原始内容保持不变，只处理衍生笔记
2. **保持记录**：每次操作都更新 log.md
3. **及时索引**：新笔记立即加入 index.md
4. **尊重用户判断**：涉及目录迁移时，确认用户意图
5. **渐进式总结**：不必一次性完美整理，随时间逐步深化

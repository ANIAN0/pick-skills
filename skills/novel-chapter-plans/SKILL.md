# Novel Chapter Plans Skill

用于把“故事单元”拆成可执行的章节计划，保证每一章都服务于当前单元核心目标。

## 前置条件（强制）

必须满足：
1. `topic_confirmed = true`
2. `chapter1_final_confirmed = true`
3. `volumes_confirmed = true`
4. 当前卷已存在分卷故事线
5. 当前章节已归属某个故事单元

## 开始前检查（逐章）

1. 前一章是否已定稿  
   - 否：先完成前一章定稿  
2. 当前章是否归属故事单元  
   - 否：先创建/补全故事单元  

## 输出目录

`03-故事单元/`（单元级章节计划）与 `04-正文/`（章节计划文件）

```markdown
04-正文/chapter-XXX-plan.md

# 第XXX章计划
- 所属单元：
- 本章目标：
- 对应单元事件：
- 起因：
- 经过：
- 结果：
- 本章结尾卡点：
```

## 设计原则

1. 每章必须映射到当前单元事件，不允许游离章节。  
2. 每章推进以“主角选择/行动”驱动。  
3. 允许小事件补逻辑，但必须嵌入主线，不得独立成支线。  
4. 语言直接表达，不使用文学修辞。  

## 后置操作

章节计划完成后，交给 `novel-draft` 生成章节草稿。

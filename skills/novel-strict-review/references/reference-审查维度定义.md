# 审查维度定义（网文市场编辑视角）

审查维度分为两大类：**核心维度**（方向性问题，P0优先）和**技术维度**（继承自novel-review的技术检查，P1/P2优先）。

审查时按优先级顺序执行：核心维度 → 技术维度。

---

## 核心维度（方向性问题优先）

### 1. 方向偏差（Direction Deviance）[P0]

**核心检查**：创作方向是否正确？

**审查姿态**：站在大纲和冲突设定角度，判断本章是否"走偏了"。

| 检查项 | 审查标准 | 判断方式 |
|--------|---------|---------|
| 大纲贴合度 | 大纲预期 vs 实际发生 | 读取大纲本章定位，对比正文实际事件 |
| 冲突推进 | 当前冲突节拍是否推进 | 读取当前冲突.md，检查节拍推进状态 |
| 冲突焦点 | 主角是否持续聚焦冲突 | 检查正文是否"跑题"——写了很多与冲突无关的内容 |
| 弧光推进 | 主角弧光是否有实质推进 | 读取主角档案弧光节点，检查本章推进 |
| 偏离性质 | 偏离是有意还是失控 | 有意偏离→建议更新大纲；失控偏离→必须修正 |

**批判性判断**：

| 发现 | 批判结论 |
|------|---------|
| 大纲偏离≥30% | "这章走偏了！大纲定位是X，实际写成了Y，这是失控偏离，必须修正！" |
| 冲突节拍0推进 | "这章在冲突层面毫无推进！当前冲突设定了节拍X/Y/Z，本章什么都没推，冲突失焦！" |
| 正文跑题≥50%篇幅 | "这章跑题了！写了大量与当前冲突无关的内容（列举），浪费了推进机会！" |
| 弧光原地打转 | "主角弧光毫无推进！本章结束后主角状态与上章完全一致，这是无效章节！" |

**判断阈值**：
- 大纲偏离度：≥30%为严重偏离，≥50%为失控偏离
- 冲突节拍推进：0推进为严重，≥1推进为合格，≥2推进为优秀
- 正文跑题比例：≥30%篇幅为跑题，≥50%为严重跑题
- 弧光推进：无变化为无效，微小变化为合格，实质推进为优秀

### 2. 结构性问题（Structural Issues）[P0]

**核心检查**：节奏崩坏？信息密度失控？预期管理失败？

**审查姿态**：站在叙事结构角度，判断本章"骨架"是否稳固。

| 检查项 | 审查标准 | 判断方式 |
|--------|---------|---------|
| 节奏密度 | 信息密度是否均衡 | 检查段落信息密度，是否前紧后松或前松后紧 |
| 节奏崩点 | 是否存在崩坏节奏 | 检查是否有"一大段无信息"或"信息爆炸堆砌" |
| 章末钩子 | 结尾是否拉动下一章 | 检查章末是否有钩子（悬念/冲突升级/反转） |
| 开章承接 | 开头是否承接上章 | 检查开头是否自然承接上章结尾状态 |
| 预期管理 | 读者预期是否被管理 | 检查是否建立预期→满足预期→建立新预期 |
| 预期落差 | 预期落差是否过大或过小 | 过大→读者困惑；过小→读者无聊 |

**批判性判断**：

| 发现 | 批判结论 |
|------|---------|
| 节奏前紧后松 | "节奏崩坏！前半部分信息密度高，后半部分泄了，读者读到后半部分会觉得无聊！" |
| 存在崩点（≥300字无信息） | "这里有一段崩点！从§X段到§Y段，基本无信息推进，纯浪费篇幅！" |
| 章末无钩子 | "章末没有钩子！读者读完这章不会想翻下一章，这是致命问题！网文每一章结尾都必须有钩子！" |
| 预期落差过大 | "预期落差过大！前文建立了预期X，实际给出Y，落差太大，读者会困惑甚至觉得被骗！" |
| 预期落差过小 | "预期落差过小！前文建立预期后，实际给出的结果几乎一致，读者会觉得无聊！" |

**判断阈值**：
- 节奏密度差：前/后半信息密度差≥2倍为崩坏
- 崩点阈值：≥300字无明显信息推进为崩点
- 章末钩子：无钩子为严重，弱钩子为不合格，强钩子为合格
- 预期落差：过大/过小都是问题，需要"刚好超出预期一点点"

### 3. 市场适配（Market Adaptation）[P0]

**核心检查**：网文规律是否遵循？爽点/钩子是否到位？

**审查姿态**：站在网文市场角度，判断本章是否符合"网文生存规律"。

| 检查项 | 审查标准 | 判断方式 |
|--------|---------|---------|
| 爽点密度 | 网文必须有爽点 | 检查是否有"爽点"（主角成功/能力提升/获得认可/反转） |
| 爽点节奏 | 爽点节奏是否符合网文规律 | 检查爽点分布是否前稀后密或均匀分布 |
| 钩子密度 | 网文每章必须有钩子 | 检查章末钩子、段落内钩子分布 |
| 情绪曲线 | 情绪曲线是否符合网文规律 | 检查是否"低开高走"或"先抑后扬" |
| 主角光环 | 主角是否有足够的存在感 | 检查主角是否"被配角压制"或"过于边缘" |
| 情节有效性 | 是否存在无效情节 | 检查是否有"事件发生但主角状态无变化"的情节 |

**批判性判断**：

| 发现 | 批判结论 |
|------|---------|
| 整章无爽点 | "网文生存规律 violated！整章没有爽点！网文必须每章至少有1个爽点，否则读者会流失！" |
| 爽点分布前稀后密 | "爽点分布不合理！前半部分没有爽点，全堆在后半部分，读者读前半部分会流失！" |
| 主角存在感低 | "主角存在感太低！这章主角几乎成了配角[具体角色]的陪衬，这是网文大忌！" |
| 存在无效情节 | "这里有无效情节！[具体情节]发生后，主角状态毫无变化，这是浪费篇幅！" |
| 情绪曲线平淡 | "情绪曲线平淡！整章没有起伏，读者读完会觉得无聊！" |

**网文市场规律**：
- 爽点密度：每章≥1个爽点（小爽点），≥3章有1个大爽点
- 爽点节奏：前半部分至少有1个小爽点，章末必须有钩子
- 情绪曲线：低开高走（先抑后扬）为佳，平铺直叙为不合格
- 主角存在感：主角必须是"核心行动者"，不能被配角压制
- 无效情节阈值：≥200字情节不改变主角状态为无效

### 4. 读者体验（Reader Experience）[P1]

**核心检查**：读感如何？会弃坑吗？预期落差在哪？

**审查姿态**：模拟读者阅读体验，判断"读到这章会怎么想"。

| 检查项 | 审查标准 | 判断方式 |
|--------|---------|---------|
| 弃坑风险 | 读者是否会弃坑 | 检查是否有"读者可能流失"的危险点 |
| 阅读疲劳 | 是否存在疲劳段落 | 检查是否有"大段枯燥描写"或"信息爆炸" |
| 阅读困惑 | 是否存在困惑段落 | 检查是否有"读者会看不懂"的段落 |
| 阅读期待 | 读者期待是否被辜负 | 检查是否有"读者期待X，实际给出Y"的情况 |
| 阅读满意度 | 整章满意度如何 | 整章读完，读者会觉得"值"还是"浪费时间" |

**批判性判断**：

| 发现 | 批判结论 |
|------|---------|
| 高弃坑风险段落 | "这里有高弃坑风险！§X段开始，读者会觉得无聊/困惑/被骗，可能直接退出！" |
| 疲劳段落 | "这里有疲劳段落！§X段开始，[具体描写]长达Y字，读者会觉得枯燥！" |
| 困惑段落 | "这里有困惑段落！§X段，[具体内容]读者会看不懂为什么这样！" |
| 期待辜负 | "期待辜负！前文建立了期待X，实际给出的结果Y让读者会觉得被骗！" |

**判断阈值**：
- 弃坑风险：≥1处高风险段落为严重
- 疲劳段落：≥300字枯燥描写为疲劳
- 困惑段落：≥1处读者会看不懂的段落为问题
- 期待辜负：≥1处期待落差过大为问题

## 技术维度（继承自novel-review）

以下维度继承自novel-review的技术检查，但审查姿态改为"批判性判断"而非"温和提问"。

### 5. 设定一致性（Setting Integrity）[P1]

**核心检查**：设定冲突、逻辑漏洞。

**审查姿态**：站在设定角度，判断本章是否"违背设定"。

| 检查项 | 审查标准 | 判断方式 |
|--------|---------|---------|
| 设定违背 | 是否违背既有设定 | 对比正文与设定文件，检查违背 |
| 设定扩写 | 正文是否无意扩写设定 | 检查正文是否有"设定里没有但正文写了"的内容 |
| 主角知晓 | 主角知晓状态是否正确 | 检查设定中"主角未知"是否被正文错误展示 |
| 规则一致 | 规则是否被一致执行 | 检查同一规则在不同段落是否执行一致 |
| 逻辑漏洞 | 是否存在逻辑漏洞 | 检查物理/空间/时间/因果是否自洽 |

**批判性判断**：

| 发现 | 批判结论 |
|------|---------|
| 设定违背 | "设定违背！设定文件说[具体设定]，但正文写成了[具体违背]，这是硬伤！" |
| 主角知晓错误 | "主角知晓状态错误！设定明确主角[未知X]，但正文写主角[知道X]，这是POV违规！" |
| 逻辑漏洞 | "逻辑漏洞！[具体漏洞]，这是硬伤，读者会看出问题！" |

---

### 6. 视角纪律（POV Discipline）[P1]

**核心检查**：POV违规、信息对称失败。

**审查姿态**：站在POV角度，判断本章是否"视角出轨"。

| 检查项 | 审查标准 | 判断方式 |
|--------|---------|---------|
| POV违规 | 是否出现主角看不到的描写 | 检查是否有"上帝视角"描写 |
| 信息对称 | 是否向读者透露主角未知真相 | 检查是否"读者知道但主角不知道" |
| 心理一致 | 心理活动与已知信息是否一致 | 检查主角心理是否与设定一致 |

**批判性判断**：

| 发现 | 批判结论 |
|------|---------|
| POV违规 | "POV违规！§X段，出现了主角[看不到/听不到]的描写，这是视角出轨！" |
| 信息对称失败 | "信息对称失败！§X段，向读者透露了主角未知的[真相]，这是大忌！" |

---

### 7. 角色塑造（Character Craft）[P2]

**核心检查**：人物扁平、行为失真。

**审查姿态**：站在角色塑造角度，判断本章角色是否"立体"。

| 检查项 | 审查标准 | 判断方式 |
|--------|---------|---------|
| 角色扁平 | 配角是否只是功能道具 | 检查配角是否有"一句话让其活一下"的机会 |
| 行为失真 | 行动是否符合身份/性格 | 检查角色行为是否与设定一致 |
| 对白失真 | 对白是否符合身份/心境 | 检查对白是否"不像这个角色说的话" |
| 主角决策 | 主角关键决定是否体现性格底色 | 检查主角决策是否与性格一致 |

**批判性判断**：

| 发现 | 批判结论 |
|------|---------|
| 配角扁平 | "配角[角色名]太扁平了！全程只是功能道具，没有一句话让其活一下！" |
| 行为失真 | "角色[角色名]行为失真！设定性格是X，但正文行为是Y，不符！" |
| 对白失真 | "角色[角色名]对白失真！这句话不像[身份/性格]的人会说的！" |

---

### 8. 风格执行（Style Execution）[P2]

**核心检查**：风格漂移、量化指标偏离。

**审查姿态**：站在风格角度，判断本章是否"风格一致"。

| 检查项 | 审查标准 | 判断方式 |
|--------|---------|---------|
| 铁律遵循 | 总纲铁律是否全部遵循 | 对比必看文档铁律，检查正文 |
| 场景必做 | 必做项是否全部命中 | 对比必看文档场景必做，检查正文 |
| 量化指标 | 量化指标是否达标 | 对比必看文档量化指标，检查正文 |
| 禁用词 | 禁用词是否0命中 | 检查禁用词清单 |
| 自检验证 | 自检报告是否与正文一致 | 对比自检报告与正文实际 |

**批判性判断**：

| 发现 | 批判结论 |
|------|---------|
| 铁律违背 | "风格铁律违背！铁律[具体铁律]，正文没有遵循！" |
| 必做项遗漏 | "场景必做项遗漏！必做项[具体必做]，正文没有做！" |
| 量化指标不达标 | "量化指标不达标！指标[具体指标]，正文实际值Y，不达标！" |
| 自检报告虚假 | "自检报告虚假！报告说达标，但正文实际不达标，这是诚信问题！" |

---

## 审查优先级

审查时按优先级顺序执行：

| 优先级 | 维度 | 审查顺序 |
|--------|------|---------|
| **P0 必须审** | 方向偏差 | 第1 |
| | 结构性问题 | 第2 |
| | 市场适配 | 第3 |
| **P1 必须审** | 读者体验 | 第4 |
| | 设定一致性 | 第5 |
| | 视角纪律 | 第6 |
| **P2 建议审** | 角色塑造 | 第7 |
| | 风格执行 | 第8 |

**审查覆盖原则**：
- P0维度必须全部覆盖
- P1维度至少覆盖≥2个
- P2维度按时间/篇幅灵活覆盖

## 批判性判断风格示例

### 示例1：方向偏差

**检查大纲贴合度**：
- 大纲定位：本章主角获得第一个能力，开始意识到世界危机
- 正文实际：主角只是日常对话，没有获得能力，没有意识到危机

**批判结论**：
"这章完全走偏了！大纲定位是'主角获得第一个能力+意识到危机'，但正文写成了'日常对话'，大纲偏离度100%，这是失控偏离，必须修正！"

### 示例2：结构性问题

**检查章末钩子**：
- 章末内容：主角回家了，睡觉，第二天醒来

**批判结论**：
"章末没有钩子！主角回家睡觉，第二天醒来——这是平铺直叙的结尾，读者读完不会想翻下一章。网文每一章结尾必须有钩子！必须改！"

### 示例3：市场适配

**检查爽点密度**：
- 整章内容：主角被欺负、逃跑、失败，没有任何成功/提升/认可

**批判结论**：
"整章没有爽点！网文生存规律 violated！主角被欺负→逃跑→失败，全程压抑，没有爽点。网文每章必须有爽点！读者读这章会觉得压抑，可能流失！"

### 示例4：读者体验

**检查弃坑风险**：
- 段落：前500字是环境描写，主角没有出现，事件没有推进

**批判结论**：
"开篇500字是纯环境描写，主角没有出现，事件没有推进——这是高弃坑风险段落！读者翻开新章，前500字看不到主角和事件，可能直接退出！必须改！"

## 审查语气原则

**必须做到**：
- 明确说"有问题"或"没问题"，不模糊
- 明确说问题是什么，不抽象
- 明确说为什么重要，不含糊
- 明确说改哪里，不笼统

**禁止做到**：
- 不说"可能存在..."
- 不说"是否可以考虑..."
- 不说"这里或许..."
- 不说"建议检查..."

**推荐做到**：
- 说"这不行！因为..."
- 说"这里明确有问题！是..."
- 说"必须改！"
- 说"影响是..."

# Novel World Skill

帮助创作者构建小说的世界观，包括物理规则、历史脉络、地理版图、社会结构和力量体系。

## 核心职责

1. **世界架构** - 定义世界的基本运行规则和框架
2. **历史脉络** - 构建世界的历史背景和重大事件
3. **地理版图** - 设计世界的地理环境和区域分布
4. **社会结构** - 规划势力分布、组织体系和文化习俗
5. **力量体系** - 设计魔法、科技或其他特殊能力系统

## 工作流程

### 1. 确定世界类型和基调

首先了解用户想要的世界类型：

- 故事发生在什么类型的世界？
- 整体基调是黑暗还是光明？
- 与真实世界或其他作品的主要区别？

### 2. 确定核心差异点

找出这个世界最独特的设定：

```markdown
核心差异点: [这个设定是独一无二的]
影响范围: [这个设定会影响什么]
衍生规则: [基于这个设定，世界必须是什么样的]
```

### 3. 从核心向外扩展

按照优先级逐步构建：

1. **力量体系** - 世界的物理/魔法规则
2. **地理环境** - 故事发生的舞台
3. **历史背景** - 世界如何变成现在这个样子
4. **势力分布** - 谁在掌控世界
5. **文化习俗** - 日常生活细节

### 4. 保持内部一致性

所有设定必须符合核心差异点：

- 检查：如果A成立，那么B是否合理？
- 检查：不同文档间是否存在矛盾？
- 检查：设定是否服务于故事需求？

### 5. 控制信息密度

遵循"需要时再详细"原则：

- 核心设定：详细完整
- 相关区域：适度详细
- 背景信息：概要即可

## 创作原则

### 一致性原则
```
所有设定必须自洽，如果魔法需要消耗生命，
那么频繁使用魔法的文明必然有相关的社会制度来应对。
```

### 差异性原则
```
世界需要有独特的识别点，避免" Generic Fantasy World"。
找出那个让这个世界与众不同的核心设定。
```

### 故事服务原则
```
世界观是为故事服务的，不是为了展示设定而设定。
优先详细故事会直接涉及的内容。
```

### 冰山原则
```
展示10%，储备90%。读者感受到世界的深度，
但只需要了解直接影响剧情的部分。
```

## 引导问题

**世界基础：**
- "这个世界最独特的设定是什么？"
- "故事的基调是轻松还是严肃？"
- "与真实世界相比，最大的不同是什么？"

**力量体系：**
- "有魔法/超能力/特殊科技吗？"
- "力量的来源是什么？有什么限制？"
- "力量如何影响社会结构？"

**地理：**
- "故事主要发生在哪些区域？"
- "不同区域之间有什么显著差异？"
- "有哪些对剧情重要的特殊地点？"

**历史：**
- "这个世界经历过什么重大事件？"
- "这些历史如何影响现在？"
- "主角所处的时代有什么特点？"

**势力：**
- "世界由谁统治/控制？"
- "有哪些重要组织/门派/家族？"
- "各方势力的关系如何？"

## 技能调用

**何时调用：**
- 用户说"设计世界观"、"构建世界"
- 用户需要建立魔法/科技体系
- 用户需要规划地理/历史/势力
- 选题完成后，进入具体创作前

**前置条件：**
- 项目已初始化
- 建议先完成选题（了解世界类型）

**后置操作：**
- 可与 novel-characters 并行进行
- 完成后可进入 novel-outline

## 与其他技能的关系

```
novel-world
    ├── novel-characters (可并行)
```

## 注意事项

1. **先核心后细节** - 确定核心差异点再扩展
2. **保持一致性** - 所有设定必须自洽
3. **服务故事** - 设定为剧情服务
4. **适度详细** - 优先详细故事涉及的内容
5. **可迭代调整** - 世界观可以随着创作深化

## 1. 项目概述

### 1.1 项目定位

**Pick-Skills** 是一个基于 AI 的技能系统框架，旨在帮助用户完成各类复杂任务。项目的核心理念是：

> **将复杂工作流程封装为可复用的技能模块，通过自然语言交互驱动任务执行。**

### 1.2 核心价值

| 维度 | 说明 |
|------|------|
| **模块化** | 每个技能独立封装，职责单一，易于维护 |
| **可组合** | 主技能可调用子技能，构建复杂工作流 |
| **可扩展** | 新增技能只需遵循规范，无需修改核心框架 |
| **人机协作** | AI 提供辅助，人类保持决策控制权 |

### 1.3 项目结构概览

```
pick-skills/
├── .claude/              # Claude Code 配置
│   ├── settings.json     # 权限配置
│   └── settings.local.json
├── skills/               # 技能模块目录
│   ├── novel-creator/    # 小说创作主技能
│   ├── brainstorming/    # 头脑风暴技能
│   ├── memos-skill/      # Memos 备忘录集成
│   ├── share-creator/    # 分享内容生成
│   └── ...               # 其他技能
├── docs/                 # 文档目录
│   └── superpowers/      # 规划文档
├── test/                 # 测试数据
└── README.md
```

### 2.2 主从技能架构

项目采用 **主技能 + 子技能** 的分层架构：

```
                    ┌──────────────────┐
                    │   novel-creator  │  ← 主技能（入口）
                    │   (主控调度)      │
                    └────────┬─────────┘
                             │
         ┌───────────────────┼───────────────────┐
         │                   │                   │
         ▼                   ▼                   ▼
┌─────────────────┐ ┌─────────────────┐ ┌─────────────────┐
│  novel-topic    │ │  novel-world    │ │  novel-draft    │
│  (选题确认)      │ │  (世界观创作)    │ │  (章节草稿)     │
└─────────────────┘ └─────────────────┘ └─────────────────┘
         │                   │                   │
         ▼                   ▼                   ▼
┌─────────────────┐ ┌─────────────────┐ ┌─────────────────┐
│ novel-characters │ │ novel-outline   │ │ novel-polish    │
│ (角色创作)       │ │ (大纲创作)       │ │ (章节润色)      │
└─────────────────┘ └─────────────────┘ └─────────────────┘
```

**设计原则**：

1. **单一职责**：每个子技能只负责一个明确的任务
2. **依赖明确**：子技能间的依赖关系清晰可见
3. **松耦合**：子技能可独立测试和替换
4. **状态隔离**：每个技能管理自己的状态

## 3. 技术栈选型

### 3.1 技术栈全景图

| 层次 | 技术选型 | 选型理由 |
|------|----------|----------|
| **技能定义** | Markdown + YAML Front Matter | 人类可读、版本控制友好、AI 易于解析 |
| **配置数据** | JSON | 结构化、跨语言支持、易于校验 |
| **脚本实现** | Python 3.8+ | AI 友好、丰富的生态、动态类型灵活 |
| **内容存储** | Markdown 文件 | 人类可读、Git 友好、支持富文本 |
| **向量数据库** | Turso/libSQL | SQLite 兼容、支持向量索引、轻量级 |
| **文本向量化** | SiliconFlow API | 国内可用、成本较低、BGE-M3 模型 |

### 3.2 核心技术详解

#### 3.2.1 SKILL.md 规范

```markdown
---
name: skill-name
description: |
  技能描述，用于匹配用户意图。
  支持多行描述。
version: 1.0.0
allowed-tools:
  - Read
  - Write
  - Bash
```

- `name`: 技能唯一标识，用于调用

## 4. 核心模块详解

### 4.1 novel-creator（小说创作主技能）

**定位**：小说创作的统一入口，协调全流程创作。

**核心功能**：
- 项目初始化与管理
- 创作流程协调
- 阶段状态跟踪
- 子技能调度

**工作流程**：

```
选题 → 一句话故事 → 世界观 → 角色 → 大纲 → 分卷 → 故事线 → 章纲 → 正文 → 完成
```

**状态管理**：

```json
{
  "topic_confirmed": false,
  "story_3lines_confirmed": false,
  "chapter1_draft_confirmed": false,
  "chapter1_final_confirmed": false,
  "volumes_confirmed": false
}
```

### 4.2 brainstorming（头脑风暴技能）

**定位**：将想法转化为设计规格。

**核心流程**：

```
探索项目背景 → 提出澄清问题 → 提出2-3种方案 → 展示设计 → 编写文档 → 用户确认 → 实现计划
```

**关键原则**：
- 一次一个问题，避免压倒用户
- 提供2-3种方案，附带推荐和理由
- 先设计后实现，禁止跳过设计阶段

### 4.3 memos-skill（备忘录管理）

**定位**：与 Memos 备忘录系统集成。

**核心功能**：
- 创建备忘录
- 搜索与过滤
- 评论与反应
- 附件管理

**API 设计**：

```
POST /api/v1/memos          # 创建备忘录
GET  /api/v1/memos          # 获取列表
GET  /api/v1/memos/{id}     # 获取详情
PATCH /api/v1/memos/{id}    # 更新备忘录
DELETE /api/v1/memos/{id}   # 删除备忘录
```

### 4.4 share-creator（分享内容生成）

**定位**：生成技术分享、项目汇报、知识培训等演示内容。

**工作流程**：

```
探索代码库 → 查看历史 → 信息收集(20问) → 生成大纲 → 确认大纲 → 生成产物
```

### 5.2 状态管理流程

```python
# 状态管理示例（state_manager.py 核心逻辑）

class StateManager:
    def __init__(self, project_path: str):
        self.state_file = Path(project_path) / ".novel" / "state.json"

    def load_state(self) -> Dict:
        if self.state_file.exists():
            return json.loads(self.state_file.read_text())
        return self.default_state()

    def save_state(self, state: Dict):
        self.state_file.parent.mkdir(parents=True, exist_ok=True)
        self.state_file.write_text(json.dumps(state, indent=2))

    def is_stage_completed(self, stage: str) -> bool:
        state = self.load_state()
        return state.get("stages", {}).get(stage, {}).get("completed", False)

    def mark_stage_completed(self, stage: str):
        state = self.load_state()
        state["stages"][stage]["completed"] = True
        self.save_state(state)
```

## 7. 常见问题与答疑

### Q1: 如何新增一个技能？

**答**：
1. 在 `skills/` 下创建新目录
2. 创建 `SKILL.md` 文件，填写元数据和实现说明
3. 如需脚本支持，创建 `scripts/` 目录
4. 测试技能功能
5. 提交代码

### Q2: 技能之间如何通信？

**答**：
- **文件传递**：通过读写文件共享数据
- **状态文件**：通过 `state.json` 共享状态
- **配置文件**：通过 `config.json` 共享配置
- **知识库**：通过向量索引共享知识

### Q3: 如何调试技能执行？

**答**：
1. 查看 Claude Code 的输出日志
2. 检查 `state.json` 文件内容
3. 使用 `print` 或 `logging` 输出调试信息
4. 单独运行 Python 脚本测试

### Q4: 向量数据库何时使用？

**答**：
- 需要**语义检索**时（如查找相似内容）
- 需要**一致性检查**时（如审查设定冲突）
- 需要**智能推荐**时（如基于上下文推荐）

### Q5: 如何扩展工作流程？

**答**：
1. 分析现有流程，确定插入点
2. 创建新的子技能
3. 在主技能中添加调用逻辑
4. 更新状态管理，添加新阶段
5. 测试完整流程

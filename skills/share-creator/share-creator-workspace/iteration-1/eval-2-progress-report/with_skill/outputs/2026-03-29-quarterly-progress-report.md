## 1. 项目背景介绍

### 项目概述

**pick-skills** 是一个AI辅助创作的技能库项目，旨在为用户提供专业化的创作辅助工具。

### 本季度工作目标

开发一套完整的**小说创作技能体系**，实现从"想法"到"成稿"的全流程AI辅助创作。

**核心问题：**
- 用户想写小说，但不知道如何开始
- 写作过程中容易迷失方向、跑题
- 内容前后不一致、设定混乱
- 缺乏专业指导导致质量不稳定

**解决方案：**
构建一套结构化的创作流程，AI在每个阶段提供专业引导和内容辅助。

### 关键要点
- 项目定位：AI辅助创作工具库
- 季度目标：小说创作全流程技能体系
- 解决痛点：创作入门难、方向迷失、一致性差

## 3. 小说创作技能体系

### 创作流程设计

整个创作过程分为三个阶段：

```
构思阶段 → 结构阶段 → 生产阶段
（确定写什么） → （规划怎么写） → （产出内容）
```

**阶段一：构思阶段**
- 选题确认：确定题材、主题、目标读者
- 三句话故事：明确故事核心走向
- 世界观设定：构建故事背景
- 角色设计：塑造主要人物

**阶段二：结构阶段**
- 分卷规划：整体结构拆分
- 故事线设计：主线、暗线、感情线
- 章节计划：每章内容规划

**阶段三：生产阶段**
- 草稿创作：撰写章节正文
- 内容润色：优化文字表达
- 一致性审查：检查设定冲突

### 核心创新点

**1. 多轮确认机制**

每一步都给用户5个选项，AI推荐一个最合适的方案，用户确认后才继续。

**产品价值：**
- 避免AI替用户做决定
- 让创作方向始终可控
- 降低用户决策难度

**2. 状态管理机制**

系统自动记录每个阶段的完成状态，确保创作不跳步、不混乱。

**产品价值：**
- 保证创作流程规范
- 防止用户迷失方向
- 支持中断后继续

**3. 偏好记录机制**

系统记录用户喜欢和不喜欢的创作方式，持续学习用户风格。

**产品价值：**
- 内容越来越符合用户口味
- 减少重复沟通
- 提升创作效率

### 关键要点
- 三阶段流程设计，循序渐进
- 多轮确认确保创作可控
- 状态管理防止流程混乱
- 偏好学习提升效率

## 4. 产品价值与应用场景

### 目标用户画像

| 用户类型 | 特点 | 使用场景 |
|---------|------|---------|
| 小说爱好者 | 有想法但不会写 | 学习创作方法、完成第一部作品 |
| 网文作者 | 需要高产稳定 | 规划大纲、批量产出内容 |
| 专业作家 | 追求品质 | 结构优化、一致性管理 |
| 内容团队 | 协作创作 | 标准化流程、知识管理 |

### 产品价值分析

**价值一：降低创作门槛**

- 从"不知道怎么开始"到"有明确的第一步"
- AI提供专业引导，新手也能写出完整故事

**价值二：提升创作效率**

- 结构化流程减少返工
- 偏好记录减少重复沟通
- 自动审查减少人工校对

**价值三：保证创作质量**

- 不跑题：每个阶段都有明确目标
- 不崩人设：系统自动检查角色一致性
- 不遗忘设定：知识库自动管理设定

### 与现有产品对比

| 功能点 | 传统AI对话 | 本系统 |
|--------|-----------|--------|
| 创作流程 | 用户自己规划 | 系统引导规划 |
| 内容控制 | AI随意发挥 | 多轮确认机制 |
| 一致性检查 | 无 | 自动审查 |
| 风格学习 | 每次重新描述 | 偏好自动记录 |
| 进度管理 | 无 | 状态自动跟踪 |

### 关键要点
- 四类目标用户，覆盖新手到专业
- 三大核心价值：降低门槛、提升效率、保证质量
- 对比传统方案有明显优势

### Q&A环节

**预设问题：**

**Q1: 这个系统适合什么类型的小说？**
**A**: 支持所有类型小说创作（玄幻、都市、悬疑、言情等），流程通用，设定按类型定制。

**Q2: 用户需要什么水平才能使用？**
**A**: 零门槛。完全不懂写作的新手可以从选题开始学习，系统全程引导。

**Q3: 和市面上AI写作工具的区别？**
**A**: 我们不只是写内容，而是教用户如何创作。有流程、有确认、有检查，是"创作教练"而非"代写工具"。

**Q4: 开发周期多长？**
**A**: 本季度完成核心体系开发（约2个月），后续持续优化迭代。

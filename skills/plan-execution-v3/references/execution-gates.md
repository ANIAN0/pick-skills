# 执行门禁

## 入口门禁

任务进入 in_progress 前必须同时满足：图校验通过；任务 ready，或恢复任务的阻塞已解决；所属变更契约和输入确认哈希有效；前置任务已 done 且有通过证据；关联验证存在、非 stale，并定义正向及必要负向观察；核心/基础设施任务有 senior 或真实审批；目标文件/符号、断言、测试和完成边界具体。

任一条件不满足时不得修改源码，保持任务非运行状态并记录精确原因。

## 实施边界

只应用变更契约 Delta 和任务局部必要改动。保留未知节点字段和用户无关改动。代码与契约矛盾或物理目标不存在时停止并创建评审发现，不得静默重设计。

## Done 门禁

任务 done 前必须满足：声明目标已有计划代码/配置变更；所需真实依赖和持久化已连接；全部执行断言通过；正向及必要负向验证已运行；关联通过证据包含任务 ID、验证 ID、命令/观察、结果、时间、执行者、输出和可读取持久产物；残留与反模拟扫描无未解释捷径；受影响图节点和项目知识已更新并重新校验。

failed、blocked、skipped 或 not_verified 证据不能满足门禁。

## 恢复行为

从责任失败节点及其受影响子图恢复。哈希仍有效的无关需求、决策或任务不重放。

# 创作风格提取

## 定位

四件套分工（详见novel-setup）：
- **novel-style-extract（本skill）**：风格层。从参考小说提取抽象化风格规则，生成知识库文件内容。
- **novel-setup**：维护知识库结构和迭代。
- **novel-lite**：使用风格进行创作。
- **novel-review**：审查时对照风格要求。

**核心能力**：将参考小说的隐性风格转化为显性、可执行的知识库文件，且**必须抽象化**——提取的是"技法规则"而非"原书设定"。

---

## ⚠️ 抽象化原则（核心）

### 问题说明

很多风格提取会直接"复制原书写法"，导致新书不可用。例如：
- ❌ 提取："战斗时描写招式名称"（这是武侠原书设定，科幻书没有招式）
- ❌ 提取："打坐修炼要描写经脉运转"（这是原书世界观规则）
- ❌ 提取："宗门考核要有三个关卡"（这是原书情节内容）

### 抽象化处理

**必须提取的是技法规则，而非具体设定：**

| 类型 | 定义 | 可提取？ | 示例 |
|------|------|---------|------|
| **通用技法** | 任何题材都能用的写作规则 | ✅ 必须提取 | "战斗要有代价交代"、"主角要有失误" |
| **题材技法** | 特定题材适用的规则 | ⚠️ 标注提取 | "武侠：招式名称要挂钩效果"、"科幻：技术名词要铺垫解释" |
| **原书设定** | 原书的具体世界观/情节/人物 | ❌ 禁止提取 | "打坐修炼的描写方式"、"某技能的名称和效果" |

**题材技法提取时必须标注**：`⚠️ [题材绑定：武侠/科幻/悬疑/...]`

### 抽象化流程

```
原书片段 → 分析写法 → 抽象为技法规则 → 判断适用范围
                         ↓
              通用技法 → 直接写入
              题材技法 → 标注后写入
              原书设定 → 跳过不提取
```

**示例**：
```
原书片段："林轲运起玄天剑法，剑光如虹，一剑斩断了敌人的手臂。"

分析：这句话展示了"招式名称 + 视觉效果 + 具体结果"的写法

抽象：
- 通用技法：能力使用后要有具体的、可视化的结果（✅提取）
- 武侠技法：招式名称后紧跟视觉效果描写（⚠️标注"题材绑定：武侠"）
- 原书设定：玄天剑法的名称和效果（❌不提取）
```

## 输入方式

### 方式A：本地文件
- 支持格式：`.txt`、`.md`
- 支持范围：单章、单卷、整本（>50KB时提示用户指定章节范围）
- 用户指定路径后读取

### 方式B：用户粘贴片段
- 单次建议≤5000字（保证分析深度）
- 可多次追加（"再分析一段"→合并到同次提取会话）

### 输入确认清单

每次提取开始前确认：
| 确认项 | 说明 |
|--------|------|
| 参考作品名称 | 用于标注知识库中的来源 |
| 提取范围 | 全量（总纲+场景分片+自检清单）/ 增量（指定1-3个场景分片） |
| 目标场景类型 | 战斗/对话/环境/心理/动作/情绪/信息释放/过渡 |
| 参考小说题材 | 武侠/科幻/都市/悬疑...（用于判断题材技法适用性） |
| 本项目题材 | 与参考小说是否同题材？（不同题材时题材技法需标注） |
| 是否覆盖已有 | 增量提取时，覆盖还是追加？ |

## 提取流程

详细流程见 `references/reference-提取工作流程.md`。

### 概述

1. **确认输入与范围**：用户确认上述清单
2. **文本分段与场景标注**：将参考文本按场景类型分段
3. **逐场景抽象化分析**：**核心步骤**——分析写法→抽象技法→判断适用范围
4. **生成知识库文件**：按模板写入，题材技法必须标注
5. **用户确认与调整**：展示提取结果，用户确认后写入
6. **与novel-setup交接**：通知已创建/更新的分片列表

## 提取维度

### 维度1：总纲提取（全量时必做）

从参考小说整体语感提炼：
- 文风灵魂：一句话描述阅读感（必须是抽象的，不含具体设定）
- 核心铁律：3-5条通用技法规则
- 词汇基调：偏好/慎用/禁用的词族（不含原书特定词汇）
- 句式偏好：长短句比例、标点使用习惯

**⚠️ 注意**：总纲铁律必须是通用技法，不能有原书设定痕迹。

### 维度2：场景分片提取（核心）

针对每种场景类型：
- 从参考小说找最典型的2-3段
- **执行抽象化分析**：提取通用技法 + 标注题材技法
- 生成：目标、必做、必避、量化指标、正例、反例
- 正例来源标注：`[抽象化提取自参考小说]` 或 `[本项目已定稿]`

### 维度3：自检清单提取

基于已提取的规则生成可量化自检项：
- 铁律检查项（通用技法）
- 场景执行检查项（按场景类型）
- 题材技法项（如适用，需标注）

## 抽象化分析示例

### 示例1：战斗场景

**原书片段**：
```
林轲握紧长剑，心跳加速——对方是筑基期高手，自己只是练气三层。
但他没有退路。剑光一闪，他冲了上去。
对方冷笑，单手挥出，林轲被震退三步，手臂一阵剧痛。
"就这？"对方嘲讽。
林轲咬牙，剑再起——他知道这招的代价，但此刻只能赌。
```

**抽象化分析**：

| 原书写法 | 抽象技法 | 适用范围 |
|---------|---------|---------|
| "握紧长剑，心跳加速" | 战斗起手交代主角状态（武器+心理） | ✅通用 |
| "对方是筑基期，自己只是练气三层" | 战斗开始交代双方力量对比 | ✅通用 |
| "被震退三步，手臂剧痛" | 主角首次攻击失败，有具体代价 | ✅通用 |
| "他知道这招的代价" | 施展能力前提示代价风险 | ✅通用 |
| "筑基期"、"练气三层"、"长剑" | 原书等级体系和武器设定 | ❌不提取 |

**提取结果**：
```markdown
## 必做
- [ ] 战斗起手交代主角状态（至少2项：武器/心理/伤情/认知盲点）
- [ ] 战斗开始交代双方力量对比（暗示差距，不一定是数值）
- [ ] 主角首次攻击有反制，且代价具体化（剧痛/后退/消耗）
- [ ] 施展关键能力前提示代价/风险

## 量化指标
| 项目 | 不合格 | 合格 | 优秀 |
|------|--------|------|------|
| 主角失误次数 | 0 | 1 | ≥2 |
| 代价具体度 | 抽象"受伤" | 1处具体 | 多处可视化 |
```

### 示例2：对话场景

**原书片段**：
```
"你为什么要帮我？"林轲问。
"因为你还有用。"那人冷冷道。
"有用？"林轲盯着他，"什么意思？"
"到了你就知道了。"那人转身离去。
```

**抽象化分析**：

| 原书写法 | 抽象技法 | 适用范围 |
|---------|---------|---------|
| 单次对白≤20字 | 对话用短句推进，节奏快 | ✅通用 |
| 对白间有动作描写（"盯着他"） | 对白穿插肢体动作，避免纯对话 | ✅通用 |
| 用提问推进（"什么意思？"） | 信息释放用追问吊胃口 | ✅通用 |
| 结尾转身离去，悬念钩子 | 对话结尾留悬念或钩子 | ✅通用 |
| 具体对话内容 | 原书情节内容 | ❌不提取 |

**提取结果**：
```markdown
## 必做
- [ ] 单次对白控制在30字内（避免说教式长对白）
- [ ] 对白间穿插肢体动作/表情描写
- [ ] 用追问推进信息释放（不一口气说完）
- [ ] 对话结尾留悬念或钩子

## 量化指标
| 项目 | 不合格 | 合格 | 优秀 |
|------|--------|------|------|
| 单次对白最长 | >50字 | 30-50字 | <30字 |
| 信息追问次数 | 0 | 1 | ≥2 |
```

## 与其他skill的衔接

### 与novel-setup
- 本skill只生成文件内容，不维护目录结构
- novel-setup负责创建骨架、维护索引、版本控制
- 生成内容后先提交用户确认，不直接写入

### 与novel-lite
- novel-lite在阶段2加载分片，按规则创作
- novel-lite发现分片缺失时触发本skill增量提取

### 与novel-review
- novel-review审查风格一致性时，以本skill提取的规则为基准
- novel-review发现风格问题时可触发本skill修正分片

---

## 边界

- **不做原创风格设计**：只从参考小说"翻译"已有风格
- **不提取原书设定**：世界观、等级体系、技能名称、具体情节不提取
- **题材技法必须标注**：非通用规则标注适用题材
- **不替代用户判断**：提取结果需用户确认后写入
- **不覆盖用户已有创作**：如用户已有定稿，提取的规则应与定稿风格比对，不一致时提示用户选择
- **不直接写入知识库**：生成内容后先展示，经用户确认后由novel-setup写入

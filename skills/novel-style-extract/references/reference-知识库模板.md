# 知识库文件模板

本模板提供风格知识库文件的详细格式。生成文件时按此格式填写。

---

## ⚠️ 抽象化原则前置

**所有内容必须抽象化处理：**
- 通用技法 → 直接写入
- 题材技法 → 标注 `[题材绑定：XX]`
- 原书设定 → **不写入**

---

```markdown
## 核心铁律（3-5条）

2. [铁律2，如：代价铁律——每次主角使用能力/做决定，必有代价或风险]

## 参考坐标
- 形似但禁止：[不要写成什么风格]
- 神似目标：[追求什么境界]

## 词汇基调
- 偏好词族：[抽象描述，如：冷色调动词、克制副词、具体名词]
- 慎用词族：[抽象描述，如：滥情词、网络梗、过度修辞]
- 禁用词清单：[具体词汇，用户指定，0容忍]

## 句式偏好
- 长短句比例：[倾向，如：长短交替，避免连续3句以上长句]
```

## 场景分片示例：战斗.md

```markdown
# 场景描写：战斗

## 目标
让读者感受到主角的压力、代价、决策过程。不能让主角一路压制，要有反制和失误。

## 必做（通用技法）
- [ ] 战斗起手交代主角当前状态（武器/心理/伤情/认知盲点至少2项）
- [ ] 战斗开始交代双方力量差距（暗示差距，不一定是数值）
- [ ] 主角首次攻击有反制，且代价具体化（剧痛/后退/消耗）
- [ ] 每次关键能力使用前提示代价/风险
- [ ] 战斗节奏：攻击→反制→代价→再攻击，避免一路压制
- [ ] 战斗结束交代代价，不能毫发无损

## 必避
- ❌ 全知视角——旁白说"敌人此时心想……"
- ❌ 主角连续顺利攻击≥3段无反制
- ❌ "想也不想就出手"——除非已建立冲动人设
- ❌ 仅堆形容词描写氛围，无具体感官细节
- ❌ 数值比拼无具体动作（"A级vsB级"没有动作描写）

## ⚠️ 题材绑定
- ⚠️ [武侠]：招式名称后要有视觉效果描写
- ⚠️ [数值体系]：力量差距可用等级数值对比

## 量化指标
| 项目 | 不合格 | 合格 | 优秀 |
|------|--------|------|------|
| 感官覆盖 | <2类 | 3类 | 4类+ |
| 主角失误次数 | 0 | 1 | ≥2 |
| 反制节拍 | 0 | 1-2 | ≥3 |
| 代价具体度 | 抽象 | 1处具体 | 多处可视化 |
| 紧张段落短句占比 | <30% | 30-60% | >50% |

## 正面示例
> 来源：抽象化提取

"林轲握紧剑，心跳加速——对方力量远高于自己。
他冲上去，剑光一闪。
对方单手挥出，他被震退三步，手臂剧痛。
'就这？'对方嘲讽。
林轲咬牙，剑再起——他知道这招的代价，但只能赌。"

**为什么好**：
- 命中必做：起手交代状态（握剑+心跳）、交代差距、首次攻击有反制+代价
- 量化达标：感官覆盖2类（触觉痛觉）、失误1次、反制节拍1次

## 反面示例
> 来源：虚构反例

"林轲使出玄天剑法第三式——剑光如虹！
对方还没反应过来，就被一剑斩断手臂。
林轲收回剑，毫发无损，嘴角微微上扬。"

**问题**：
- 违反必做：主角一路压制无反制、结束无代价
- 违反必避：招式名称堆砌无具体动作、连续顺利攻击
```

## 场景分片示例：对话.md

```markdown
# 场景描写：对话

## 目标
用短句推进节奏，避免说教式长对白。信息释放要吊胃口，不能一口气说完。

## 必做（通用技法）
- [ ] 单次对白控制在30字内
- [ ] 对白间穿插肢体动作/表情描写
- [ ] 用追问推进信息释放（不一口气说完）
- [ ] 对话结尾留悬念或钩子
- [ ] 对白符合角色身份（年龄、性格、心境）

## 必避
- ❌ 单次对白超过50字（说教式）
- ❌ 纯对话段落超过3段无穿插描写
- ❌ 信息一口气说完无追问铺垫
- ❌ 对白不符合角色身份（小孩说大人话）

## ⚠️ 题材绑定
- ⚠️ [古风]：对白避免现代口语/网络梗
- ⚠️ [科幻]：技术名词可在对白中解释

## 量化指标
| 项目 | 不合格 | 合格 | 优秀 |
|------|--------|------|------|
| 单次对白最长 | >50字 | 30-50字 | <30字 |
| 对白穿插描写密度 | <1次/3段 | 1次/2段 | ≥1次/段 |
| 信息追问次数 | 0 | 1 | ≥2 |
| 对白节奏变化 | 单一 | 2种变化 | 多种变化 |

## 正面示例
> 来源：抽象化提取

"'为什么要帮我？'
'因为你还有用。'
'有用？'林轲盯着他，'什么意思？'
'到了你就知道。'那人转身离去。"

**为什么好**：
- 命中必做：对白≤20字、穿插动作（盯着）、追问推进、结尾钩子
- 量化达标：最长20字、追问2次

## 反面示例
> 来源：虚构反例

"'林轲，你知道我为什么要帮你吗？其实是因为我看到了你身上的潜力，虽然你现在只是练气三层，但我相信只要你努力修炼，一定能成为筑基期高手，到时候我们宗门就需要你这样的人，所以我现在帮你，也是为了宗门的未来，你要好好珍惜这个机会，不要辜负我的期望。'
'好的，我一定努力修炼，不辜负您的期望！'林轲感激地说。"

**问题**：
- 违反必做：单次对白>100字、无穿插、信息一口气说完、结尾无钩子
- 违反必避：说教式长对白、纯对话段落
```

```markdown
# 写作风格自检清单

> 用法：每写完一章正文，逐项检查并打分。

- [ ] 铁律1：视角限制——正文只展示主角能感知的事物

- [ ] 铁律3：失误铁律——每章至少1次主角判断失误或信息不足

### 战斗场景（若有）
```

## _索引.md 模板

```markdown
# 创作风格知识库索引

## 总纲
- `_总纲.md`：文风灵魂、核心铁律、词汇基调（始终加载）

## 场景分片
| 分片 | 适用场景 | 加载时机 |
|------|---------|---------|
| `战斗.md` | 主角与敌人对抗 | 写战斗时 |
| `对话.md` | 角色间语言交流 | 写对话时 |
| `心理.md` | 主角内心活动 | 写心理描写时 |
| `环境.md` | 场景/氛围描写 | 写环境时 |
| `信息释放.md` | 谜团揭示/真相透露 | 写揭谜时 |
| `过渡与节奏.md` | 场景切换/时间推进 | 写过渡时 |

## 自检清单
- `自检清单.md`：定稿前必跑（始终加载）

## 风格指纹
- `风格指纹/第X章.md`：已定稿章节风格采样（写新章时加载上一章）

## 加载规则
- 写单场景 → 加载总纲 + 对应分片 + 自检清单 + 上章指纹
- 写多场景 → 加载总纲 + 多个分片 + 自检清单 + 上章指纹
- 题材技法 → 仅同题材项目加载，非同题材跳过标注项
```

# 提取工作流程

详细的6步提取流程。

---

## 第一步：确认输入与范围

### 必问清单

| 确认项 | 询问方式 | 用途 |
|--------|---------|------|
| 参考作品名称 | "参考小说叫什么名字？" | 标注来源 |
| 提取范围 | "全量提取还是增量补充？"（全量=总纲+所有场景；增量=指定1-3个场景） | 确定工作量 |
| 目标场景类型 | "主要想提取哪些场景的风格？"（战斗/对话/心理/环境/...） | 确定分片列表 |
| 参考小说题材 | "这本参考小说是什么类型？"（武侠/科幻/都市/悬疑） | 判断题材技法 |
| 本项目题材 | "你正在写的是什么类型？"（与参考是否同题材） | 决定题材技法是否标注 |
| 是否覆盖 | 增量时问："覆盖已有分片还是追加补充？" | 确定写入方式 |

### 判断题材技法处理

| 场景 | 处理方式 |
|------|---------|
| 参考小说题材 == 本项目题材 | 题材技法可直接写入，轻度标注 |
| 参考小说题材 ≠ 本项目题材 | 题材技法必须标注 `[题材绑定：XX]`，提示用户确认是否采用 |
| 本项目题材未定 | 题材技法标注 `[题材绑定：XX]`，让用户决定 |

---

## 第二步：文本分段与场景标注

将参考文本按场景类型分段标注。

### 标注方法

```
[段落1-3] → 场景类型：战斗
  - 核心动作：主角与敌人的对抗
  - 视角焦点：外部观察+内心决策
  - 叙事目的：推进冲突

[段落4-6] → 场景类型：对话
  - 核心动作：角色间的语言交流
  - 视角焦点：人物互动
  - 叙事目的：释放信息、推进关系

[段落7-9] → 场景类型：心理
  - 核心动作：主角的内心活动
  - 视焦焦点：内心独白
  - 叙事目的：展示弧光变化

[段落10-12] → 场景类型：环境
  - 核心动作：场景/氛围描写
  - 视角焦点：外部观察
  - 叙事目的：渲染氛围、铺垫后续
```

### 场景类型判断依据

| 场景类型 | 判断依据 |
|---------|---------|
| 战斗 | 有对抗动作、能力使用、代价描写 |
| 对话 | 主要内容是角色间对白 |
| 心理 | 主要内容是内心独白/思考 |
| 环境 | 主要内容是场景/氛围描写 |
| 动作 | 有角色行动但不涉及对抗 |
| 情绪 | 主要展示角色情感变化 |
| 信息释放 | 有谜团揭示/真相透露 |
| 过渡与节奏 | 场景切换、时间推进、节奏调节 |

---

## 第三步：逐场景抽象化分析（核心）⭐

这是整个提取流程的**关键步骤**——必须执行抽象化处理，将原书写法转化为技法规则。

### 分析流程（每个场景）

```
对于每个标注的段落：
1. 识别写法特征：这段文字用了什么写法？
2. 抽象技法：把写法转化为通用的技法规则
3. 判断适用范围：通用技法？题材技法？原书设定？
4. 分类输出：
   - 通用技法 → 写入必做/必避
   - 题材技法 → 标注后写入
   - 原书设定 → 跳过
```

### 抽象化分析示例表

| 原书写法特征 | 抽象技法规则 | 适用范围 | 处理 |
|-------------|-------------|---------|------|
| 每次能力使用后写代价 | 能力使用必有代价提示 | 通用 | ✅写入必做 |
| 战斗节奏：攻击→反制→代价→再攻击 | 战斗要有反制节拍，不能一路压制 | 通用 | ✅写入必做 |
| 对白≤20字，节奏快 | 对话用短句推进 | 通用 | ✅写入必做 |
| 招式名称后紧跟视觉效果 | 武侠：招式名后要有视觉描写 | 武侠绑定 | ⚠️标注写入 |
| 等级数值对比（筑基vs练气） | 数值对比展示力量差距 | 部分题材 | ⚠️标注"数值体系题材" |
| "玄天剑法"名称和效果 | 原书技能设定 | 原书设定 | ❌跳过 |
| "打坐修炼"的描写方式 | 原书世界观规则 | 原书设定 | ❌跳过 |

### 量化指标提取

从参考文本统计：
- 感官覆盖：视觉/听觉/触觉/痛觉的出现频次
- 句式分布：长短句比例、紧张段落短句占比
- 信息密度：每段推进信息的类型分布
- 节奏曲线：紧→缓→紧的段落分布

## 第四步：生成知识库文件

按模板输出文件内容。

### 全量提取时

1. `_总纲.md`：
   - 文风灵魂（抽象描述）
   - 核心铁律（只写通用技法）
   - 词汇基调（不含原书特定词汇）
   - 句式偏好

2. `场景描写/<场景>.md`（每种场景一个）：
   - 目标（抽象描述）
   - 必做（通用技法+标注的题材技法）
   - 必避（通用反技法）
   - 量化指标（统计数据）
   - 正例（标注来源：抽象化提取或本项目）
   - 反例（虚构的反面示例）

3. `自检清单.md`：
   - 铁律检查项
   - 场景执行检查项
   - 题材技法项（如有）

4. `_索引.md`：分片列表和加载指引

### 增量提取时

1. 更新指定场景的 `场景描写/<场景>.md`
2. 如新增场景类型 → 更新 `_索引.md`
3. 如新增铁律 → 更新 `_总纲.md`
4. 同步更新 `自检清单.md`

## 第五步：用户确认与调整

产出文件后，**不直接写入**，先向用户展示：

### 展示格式

```markdown
# 风格提取结果预览

## 抽象化处理说明
- 通用技法：X条（可直接采用）
- 题材技法：X条（已标注 [题材绑定]，需您确认是否适用于本项目）
- 已跳过原书设定：X项（具体设定不提取，如技能名称、等级体系）

## 总纲核心铁律（通用）
1. [铁律1]
2. [铁律2]
3. ...

## 题材技法（需确认）
- ⚠️ [题材绑定：武侠]：招式名称后要有视觉描写
- ⚠️ [题材绑定：数值体系]：力量差距可用数值对比展示
→ 这些规则是否适用于您的项目？

## 场景分片预览
- 战斗.md：必做X条 / 量化指标X项 / 正例来源[抽象化提取]
- 对话.md：...

## 请确认
- [ ] 通用技法是否需要调整？
- [ ] 题材技法是否采用？（若本项目非同题材，可跳过）
- [ ] 量化指标阈值是否合理？
```

### 用户确认后的处理

- 用户同意 → 通知novel-setup写入
- 用户调整 → 修改后重新展示
- 用户跳过题材技法 → 删除标注项后写入

---

## 第六步：与novel-setup交接

写入完成后，通知novel-setup：

| 信息 | 内容 |
|------|------|
| 新创建/更新的分片 | 列出文件路径 |
| 总纲铁律变更 | 是否有新增/修改铁律 |
| 题材技法处理 | 用户采用/跳过 |
| 建议后续引用 | 建议在开写简报中引用新分片 |

---

## 抽象化处理检查清单

每次提取完成后自检：

- [ ] 总纲铁律全是通用技法，无原书设定痕迹？
- [ ] 题材技法已标注 `[题材绑定：XX]`？
- [ ] 正例来源标注正确（抽象化提取/本项目）？
- [ ] 没有提取原书技能名称、等级体系、具体情节？
- [ ] 量化指标是统计结果，不是原书具体数值？
- [ ] 如果参考题材≠项目题材，题材技法已提示用户确认？

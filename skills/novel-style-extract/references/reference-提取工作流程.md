# 提取工作流程

## 第一步：确认输入与范围

1. 询问/确认输入方式（本地文件 / 粘贴片段）
2. 询问/确认提取范围（全量 / 增量-指定场景）
3. 如为增量：读取现有 `相关设定/创作风格/` 下的已有文件，确认是覆盖还是追加

## 第二步：文本分段与场景标注

将参考文本按场景类型分段标注：

```
[段落1-3] → 环境描写
[段落4-7] → 对话
[段落8-12] → 战斗
[段落13-15] → 心理
...
```

标注依据：
- 段落中的核心动作/交互类型
- 视角焦点（外部观察 / 内心活动 / 人物互动）
- 叙事目的（推进剧情 / 渲染氛围 / 释放信息）

## 第三步：逐场景深度分析

对每种场景类型，执行以下分析：

1. **技法扫描**：该场景类型中反复出现的写作技法（句式结构、修辞偏好、信息组织方式）
2. **感官覆盖统计**：视觉/听觉/触觉/痛觉/嗅觉/味觉的出现频次和组合模式
3. **节奏分析**：长短句分布、段落长度、紧张/松弛段落的节奏曲线
4. **词汇采样**：高频动词、形容词、副词的特色用词
5. **反例构造**：基于"参考小说不这样写"的原则，构造反面示例

## 第四步：生成知识库文件

按提取范围输出文件到 `相关设定/创作风格/`：

全量提取时：
1. 生成 `_总纲.md`
2. 为每种识别出的场景类型生成 `场景描写/<场景>.md`
3. 生成 `自检清单.md`
4. 生成 `_索引.md`（知识库内容地图）

增量提取时：
1. 生成/更新指定场景的 `场景描写/<场景>.md`
2. 如新增场景类型 → 更新 `_索引.md`
3. 如影响铁律 → 更新 `_总纲.md`
4. 同步更新 `自检清单.md` 中对应场景的自测项

## 第五步：用户确认与调整

产出文件后，**不直接写入**。先向用户展示：

```
# 提取结果预览

## 总纲核心铁律（5条）
1. ...
2. ...

## 场景分片（已提取X类）
- 战斗.md：[必做5条 / 量化指标5项 / 正例2段]
- 对话.md：...

## 自检清单
- 铁律检查：X条
- 场景执行检查：X类

请确认或调整：
- [ ] 铁律是否需要增删？
- [ ] 某类场景的量化指标是否需要调整？
- [ ] 正例/反例是否准确？
```

用户确认后，写入 `相关设定/创作风格/`。

## 第六步：与 novel-setup 的交接

写入完成后，通知 novel-setup：
- 新创建/更新的分片列表
- 是否涉及总纲铁律变更
- 建议 novel-setup 在后续开写简报中引用新分片

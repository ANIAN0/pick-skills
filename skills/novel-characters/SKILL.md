# Novel Characters Skill

用于在创作推进中按需补强角色，而不是先做完一整套角色库再开写。

## 定位

1. 可选扩展，按章节需要逐步补。  
2. 角色设定必须服务当前单元核心目标。  
3. 不得要求“角色库完整”才允许继续写章节。  

## 输出文件

`02-角色/`

最小必需文件：
- `protagonist.md`
- `relationships.md`

可选补充：
- `supporting/*.md`
- `antagonist.md`

## 最小模板

```markdown
# 角色设定（最小版）

## 功能定位
- 该角色在当前单元的作用：

## 行动动机
- 该角色当前想达成什么：

## 行为边界
- 该角色不会做什么：

## 与主角关系
- 当前关系：
- 可能变化：
```

## 规则

1. 优先保证“行为一致”而非“设定华丽”。  
2. 角色变化必须能在章节中被读者看见。  
3. 新增设定不得推翻已确认第1章正文风格与事实。  
4. 用户明确肯定/否定的角色写法，需同步到偏好记录文件。

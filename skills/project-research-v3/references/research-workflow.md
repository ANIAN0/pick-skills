# 调研工作流

## 任务定义

Research Task 只回答一个可验证问题，并记录责任节点、问题、范围、排除项、所需来源类型、完成标准和时效要求。问题过宽时先拆分，不生成泛化行业报告。

## 复用判断

1. 先按主题、实体、技术和关键字查询 `personal-kb`。
2. 检查候选报告的来源、适用范围、更新时间、有效期和未解决不确定性。
3. 结论仍适用时复用同一 Knowledge Report ID。
4. 来源过期、范围变化或出现冲突证据时更新原报告并保留修订记录。
5. 没有可复用报告时才创建新条目。

## 来源与结论

优先使用官方文档、标准、源码、权威论文和一手资料。区分事实、来源陈述与推断；冲突来源必须并列记录并说明取舍依据。无法证实的结论保持不确定，不得包装为确定事实。

## 项目摘要

项目图谱只写：Research Task ID、Knowledge Report ID、与责任节点直接相关的结论摘要、适用范围、有效期和剩余风险。完整来源、分析和历史只保存在全局知识库。

消费节点通过 `uses-knowledge` 指向报告；Research Task 通过 `researches` 指向责任节点。

## stale 与 blocked

报告过期或关键来源失效时，将报告标记 stale，计算消费节点局部影响并触发聚焦复核。来源不足、访问受限或结论冲突未解时，任务保持 blocked 并记录缺失条件。

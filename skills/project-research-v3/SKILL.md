# 项目调研 v3

只调查一个边界明确的问题，把证据交还调用阶段。`completed` 与 `blocked` 都必须生成报告并双向链接。

## 输入门槛

只接受阶段入口中登记的 `RT-*`，或入口 `RT-*` 链接到的 `research/RT-*`。无人值守只表示完整 `RT-*` 可直接执行，无需额外人工确认；触发、重试和范围变化仍由调用阶段显式决定。

开始前读取阶段入口；入口只保留任务链接和结果链接时，沿链接读取 `research/RT-*`。任务必须包含：单一可回答的问题、范围与不调查内容、所需证据、完成条件、受影响入口条目和真实路径。

任一项缺失时，仍按标准路径创建或更新 `research/RT-编号-简短英文-slug.md`；缺少标题或 slug 来源时用 `research/RT-001-incomplete-task.md` 这类保守名。报告写 `blocked`，列出缺失字段、已检查入口和恢复条件，并把入口结果回写为 `blocked`。

## 材料位置

新建材料放在入口旁同名目录的 `research/RT-编号-简短英文-slug.md`。入口已链接 `research/RT-*` 时更新同一文件，不为同一 `RT-*` 另建报告。复用或增量更新已有材料时保留原路径并补双向链接。范围实质变化时不继续调查，写 `blocked` 交还调用阶段。

## 检索与复用

开始新调查前按顺序查找：

1. `personal-kb` 可用时，按它的读取和查询规则检索项目知识；不可用时跳过，并在报告记录能力限制。
2. 当前入口旁 `research/` 中完整适用的报告直接复用，部分适用的报告在原路径增量更新。

项目内部事实必须对照当前代码、配置、测试、提交历史或运行结果确认；外部事实可能变化时重新核实来源。

## 调研过程

1. 读取入口、当前 `RT-*`、已链接材料和与问题直接相关的已有材料。
2. 按完成条件确定要检查的项目位置和外部来源；不预设来源数量。
3. 项目事实优先检查真实代码、配置、测试、提交历史和运行结果。外部资料优先调用 `playwright` skill 用 `playwright-cli` 打开官方文档、源码或一手资料；先 snapshot 后操作，导航后重新 snapshot。Google 结果页只用于发现来源，搜索摘要不能作为证据。
4. 逐项核对证据，区分来源陈述、项目事实和 Agent 推断；每个关键结论必须能追溯到具体来源。来源冲突时并列记录冲突、适用范围和取舍依据。
5. 把报告或复用材料链接回入口 `RT-*`。具体回写字段见 [写回边界](#写回边界)。

## 知识库沉淀

只在 `completed` 前处理并记录：

- 可复用结论：`personal-kb` 可用时调用它归档或更新，并保留本报告路径作为项目证据。
- 一次性结论或 `personal-kb` 不可用：写 `skipped` 和原因。
- 沉淀失败：写 `error` 和错误摘要；只要调研结论证据充分，仍可 `completed`。
- `blocked` 报告：不沉淀，写 `skipped`，原因是未形成可沉淀结论。

报告末尾必须包含：

```markdown
## 知识库沉淀

- 复用价值：可复用 / 一次性
- 沉淀状态：done（<personal-kb 返回的 project-kb 路径>）/ skipped / error
- 错误：<错误摘要或无>
```

## 来源与能力

外部事实可能变化时必须在线核实。不能访问的来源不得根据搜索摘要包装成已确认事实。推断必须明确标记并写出依据。

报告必须记录实际使用能力、不可用能力和影响范围，例如项目文件、命令结果、`playwright-cli`、官方文档、`personal-kb`、联网能力等。

## 写回边界

只允许修改当前 `RT-*` 的回写字段：简洁链接格式只更新 `调研结果` 为 `completed` 或 `blocked`；内联任务格式只更新 `调研报告` 和 `执行结果`。同一 `research/RT-*` 文件中可更新报告正文、来源、能力记录、知识库沉淀和执行结果。

禁止修改用户故事、功能点、验收要求、技术决策、任务内容、执行状态、验收结论或吸收状态。

## 提问门槛

默认不重新访谈用户。只有缺少用户才能提供的访问条件或事实，并且缺失会阻止完成时才提问。问题放在回复最后的 `## 需要你确认` 区块，只包含一个问题、推荐处理和不同选择的影响。

## 完成条件

`completed` 必须同时满足：

- 完成条件逐项满足。
- 关键结论均有具体来源或项目证据。
- 事实、来源陈述和推断已区分。
- 冲突、限制和能力缺口已说明。
- 调研材料与入口 `RT-*` 双向链接。
- 入口 `RT-*` 已写入 `completed`。
- 知识库沉淀已按能力处理并记录结果。

`blocked` 也必须生成报告并回写入口：列出已检查位置和来源、不可用能力、缺失条件、当前能确认与不能确认的内容、恢复条件；不执行知识库沉淀。

## 边界

- 不创建独立调研入口。
- 不替调用阶段修改结论、标记吸收或批准取舍。
- 不用无来源推断、搜索摘要或来源数量冒充可靠结论。
- 不用 mock、固定返回或未验证实现冒充完成。

# 项目调研 v3

只调查一个已经审核、边界明确的问题。输出项目内调研报告，把证据交还调用阶段；不替调用阶段修改结论或替用户做决定。

## 输入门槛

只接受阶段入口文档中已审核的 `RT-*`。开始前检查任务是否包含：

- 单一、可回答的问题。
- 调研范围和明确不调查的内容。
- 需要的证据或来源类型。
- 可判断的完成条件。
- 受影响的入口文档内容。
- 入口文档真实路径和稳定条目编号。

任一项缺失，或任务尚未审核，不开始调查；把缺口返回调用阶段补充。不要自行扩大、缩小或重新解释任务。任务范围发生实质变化时，必须返回原阶段重新审核。

## 报告位置

只有需要新建报告时，才把报告放在当前入口文档旁的同名支撑目录中，使用 `research/RT-编号-简短英文-slug.md`。

例如：

```text
入口：workplace/3/requirements/2026-06-21-order-import.md
报告：workplace/3/requirements/2026-06-21-order-import/research/RT-001-duplicate-handling.md
```

其他阶段新建报告时沿用同一规则。复用或增量更新已有材料时，保留已有材料的原路径，不复制到当前入口旁。创建新报告时使用 [调研报告模板](assets/research-report.md)。

## 检索与复用

开始新调查前，先按以下顺序查找可复用内容：

**第一优先：检索 project-kb/**
先读取 `project-kb/index.md`（不存在则跳过），根据主题列表定位相关条目，再打开具体文件确认内容：
- 完整适用且未过期：直接引用作为证据，在 `RT-*` 中记录来源，跳过重新调研
- 完整适用但已过期（external 条目超过 6 个月）：标注"需重新验证"，继续调研，完成后更新该条目
- 部分适用：只调查缺失或过期部分，补充后更新 project-kb 对应条目
- 不适用：继续第二步

**第二优先：检索阶段已有调研报告**
在当前入口旁同名目录的 `research/` 中检查已有 `RT-*` 报告：
- 完整适用：当前 `RT-*` 直接链接已有报告，不新建、复制或移动
- 部分适用：只调查缺失或过期部分，在原路径更新
- 没有可复用材料：新建报告

问题范围存在实质差异时，视为新问题新建报告，不覆盖原报告。

## 调研过程

1. 读取 `RT-*`、阶段入口文档和与问题直接相关的已有支撑材料，确认问题、范围、完成条件和报告路径。
2. 搜索当前项目已有调研报告和 `project-kb`，先判断复用、增量更新或新建。
3. 根据完成条件确定需要检查的项目位置和外部来源，不设置固定来源数量，也不为了篇幅扩大调研。
4. 项目事实优先检查真实代码、配置、测试、提交历史、运行结果和项目文档。外部资料检索优先调用 `playwright` skill，使用 `playwright-cli` 连接浏览器，在 Google 搜索并打开原始来源；优先复用当前浏览器会话，没有可用会话时再打开 headed 浏览器。先 snapshot，再使用最新元素引用操作，导航后重新 snapshot。
5. Google 结果页只用于发现来源，搜索摘要不能作为证据。技术问题优先通过 Playwright CLI 打开官方文档、标准、源码仓库、权威论文和其他一手资料。
6. 逐项核对证据。明确区分来源直接陈述、项目中观察到的事实和 Agent 推断；每个关键结论都要能追溯到具体来源。
7. 来源冲突时并列记录冲突、适用范围和取舍依据，不按搜索排名、来源数量或摘要直接决定。
8. 完成条件满足时进入"知识库归档"步骤；来源不足、访问受限或关键冲突无法解决时写 `blocked` 报告，记录已检查内容、缺失证据和恢复条件。
9. 报告或复用材料链接回入口文档中的 `RT-*`。在入口任务中写入材料链接和执行结果，但不修改任务问题、审核结论或关闭状态。
10. 标注结论的项目知识价值，再将材料路径、执行结果和未解决内容交还调用阶段。调用阶段负责更新原结论、继续讨论并关闭任务。

## 知识库归档（调研完成后必须执行）

调研完成后，在写 `completed` 之前必须执行本步骤：

**步骤1：判断复用价值**

问自己：这个结论在其他需求或后续迭代中是否可能再次需要？

| 场景 | 判断 |
|---|---|
| 当前项目的通用技术约束、架构事实、框架能力 | 可复用 |
| 外部 API 或服务的稳定行为和限制 | 可复用 |
| 业务规则、数据关系、权限边界 | 可复用 |
| 仅对当前需求版本有效的临时调研 | 一次性 |
| 已有 project-kb 条目的增量补充 | 更新原条目 |

**步骤2：可复用时，写入 project-kb/**

在 `project-kb/` 中检查是否已有同主题条目：

- **已有同主题条目**：在原条目末尾追加本次发现，注明日期和来源，更新"最后验证时间"
- **无同主题条目**：在 `project-kb/` 下创建新条目文件，使用 `主题英文-slug.md` 命名

project-kb 条目格式：
```markdown
# <主题名称>

最后验证：<日期>
过期规则：<external（6个月后重新验证）| internal（无时效限制）>
来源调研：<RT-* 链接>

## 结论

<直接可用的结论，不写调研过程>

## 适用范围

<这个结论在什么情况下成立，在什么情况下需要重新验证>

## 历史记录

- <日期>：<首次记录或更新内容>
```

过期规则说明：
- `external`：结论来自外部 API、第三方服务或框架版本，超过 6 个月需重新验证后才能作为可靠证据
- `internal`：结论来自项目自身代码、配置或业务规则，无时效限制，但项目该部分变更时需主动更新

**检索时的过期判断**：读取到一个 `external` 条目时，检查"最后验证"日期，超过 6 个月则标注为"需重新验证"，调用阶段自行决定是否发起新的 `RT-*`。

**步骤2b：更新 project-kb/index.md**

每次新增或更新条目后，必须同步更新 `project-kb/index.md`。index.md 格式：

```markdown
# project-kb 索引

最后更新：<日期>

| 主题 | 文件 | 最后验证 | 过期规则 | 适用范围摘要 |
|---|---|---|---|---|
| <主题名称> | [文件名](文件名.md) | <日期> | external / internal | <一句话> |
```

index.md 不存在时先创建再写入。index.md 是 agent 检索 project-kb 的入口，检索时先读 index.md 定位相关条目，再打开具体文件。

**步骤3：在调研报告末尾记录归档结果**

```markdown
## 知识库归档

- 复用价值：可复用 / 一次性
- 归档状态：done（project-kb/<文件名>.md）/ updated（project-kb/<文件名>.md）/ skipped（一次性）
```

**一次性结论**：只写 `skipped`，不创建 project-kb 条目。

完成本步骤后，才能将调研标记为 `completed`。

## 来源与证据

外部事实可能随时间变化时必须在线核实。项目行为优先以当前代码、真实配置和可运行结果为准。

引用应贴近它支持的结论。报告末尾保留完整来源清单和访问时间。不能访问的来源不得根据搜索摘要包装成已确认事实。

推断必须明确标记，并写出由哪些事实推导而来。来源没有直接支持的内容不能写成来源结论。

## 写回边界

允许修改入口文档中当前 `RT-*` 的两项内容：

- `调研报告`：指向新报告、复用报告或项目知识条目的链接。
- `执行结果`：`completed` 或 `blocked`。

禁止修改用户故事、功能点、验收要求、技术决策、任务内容、执行状态或验收结论；禁止把 `RT-*` 标记为关闭。调用阶段确认报告已正确回写到原结论后，才能关闭任务。

## 提问门槛

调研任务已经审核，默认不重新访谈用户。能从项目或可靠来源获得的信息自行调查。

只有缺少用户才能提供的访问条件或事实，并且缺失内容会阻止完成时才提问。问题放在回复最后的 `## 需要你确认` 区块，只包含一个问题、推荐处理和不同选择的影响，后面不追加其他内容。

## 完成条件

`completed` 必须同时满足：

- 完成条件逐项满足。
- 关键结论均有具体来源或项目证据。
- 事实、来源陈述和推断已区分。
- 冲突与限制已说明。
- 调研材料与入口任务双向链接。
- 入口任务已写入材料链接和 `completed`，但未被关闭。
- 已在报告中标注项目知识价值。
- **知识库归档步骤已执行，报告末尾有归档记录**。

任一项不成立时不得写 `completed`。确实无法继续时写 `blocked`，并明确恢复条件。

`blocked` 也必须生成报告并满足：列出已经检查的项目位置和来源、具体缺失条件、当前可以确认与不能确认的内容、恢复调研所需条件；报告与入口任务双向链接，入口任务执行结果写为 `blocked`，但任务不关闭。

向调用阶段报告完成前先逐项自检；发现问题先修正并重新检查，不把来源、链接或边界检查留给用户。

## 边界

- 不创建独立调研任务文件；任务属于阶段入口文档。
- 只从当前项目已有调研报告和 `project-kb` 检索可复用材料。
- 不替原阶段修改结论、关闭任务或批准产品与技术取舍。
- 不用无来源推断、搜索摘要或来源数量冒充可靠结论。

# Novel Review Skill

用于在章节定稿前做结构和逻辑审查，重点检查“是否严格服务当前单元”。

## 输入与输出

**输入：**
- `03-故事单元/unit-XXX.md`
- `04-正文/chapter-XXX-draft.md` 或 `chapter-XXX-final.md`
- `01-第1章/style-profile.md`（如已存在）

**输出：**
- `04-正文/chapter-XXX-review.md`

## 核心检查项

1. 是否服务当前单元核心目标  
2. 是否形成“起因-经过-结果-遗留问题（卡点）”  
3. 是否存在超前续写或无关支线  
4. 情节是否由主角选择/行动推动  
5. 是否与前一章定稿内容衔接  
6. 是否符合已确认写作风格  

## 报告格式

```markdown
# 第XXX章审查报告

## 审查结论
- 通过 / 需修改

## 关键问题
1. [问题]
2. [问题]

## 修改建议
1. [建议]
2. [建议]

## 流程检查
- 前一章已定稿：是/否
- 当前章已绑定故事单元：是/否
- 偏好记录已同步：是/否
```

## 规则

1. 只基于当前文档做审查，不虚构外部事实。  
2. 审查建议必须可执行，避免空泛评价。  
3. 不依赖 Turso、向量索引或知识库同步。

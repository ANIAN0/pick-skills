# 小说章节审查

## 定位

三件套分工：
- **novel-setup**：宏观层。项目大纲、故事状态、伏笔追踪。
- **novel-lite**：中观层（写）。单章创作与修订。
- **novel-review（本skill）**：中观层（审）。单章审查与循环驱动。

审查不是评分，而是**用专业问题激发思考**，让作者在回答问题的过程中发现情节、设定、逻辑、趣味、连续性的可提升点，并沉淀为修订后的章节与更完善的设定。

## 四种审查模式总览

| 模式 | 简称 | 对象 | 触发场景 | 输出 |
|------|------|------|---------|------|
| **正文审查** | review-text | 章节正文+三件套 | 写完一稿、修订后、定稿前 | 章节审查报告（七维度问题） |
| **新设定审查** | review-pending | `工作区/待定设定/` 中的草案 | 待定项累积≥3条、章节定稿前、用户主动请求 | 设定决策报告（每条给晋升/调整/废弃建议） |
| **设定体检** | review-canon | `相关设定/` 下既有设定 | 重要节点（卷末/百章里程碑）、用户主动请求、发现设定打架时 | 设定健康报告（一致性/过时/冲突/缺失） |
| **阶段回顾** ⭐ | review-arc | 最近5章正文+相关设定+当前冲突+伏笔追踪 | **每5章定稿后必触发**（N%5==0），或用户主动请求 | 阶段回顾报告（设定更新建议、冲突演化建议、伏笔补登、节拍推进核对） |

模式可由用户显式指定。未指定时按以下默认推断：
- 用户说"审查第X章" → review-text
- 用户说"审查待定设定 / 审一下这些草案 / 看看哪些可以晋升" → review-pending
- 用户说"做次设定体检 / 设定有没有问题 / 检查一下设定" → review-canon
- 用户说"阶段回顾 / 回看最近5章 / 第N章了做个回顾 / 跑 review-arc" → review-arc

下面分别详述。

## 模式 A：正文审查（review-text）

### 审查对象

#### 必读（章节上下文）
- `工作区/第X章VX/正文.md`
- `工作区/第X章VX/事件摘要.md`
- `工作区/第X章VX/相关实体.md`
- `工作区/第X章VX/审查历史/`（历次审查报告，用于避免重复提问、支持增量审查）

#### 必读（项目级宏观设定 — 由 novel-setup 维护）
- `相关设定/世界设定.md` — 世界观基底
- `相关设定/故事大纲.md` — 本章在整体走向中的定位
- `相关设定/主角档案.md` — 主角画像与当前弧光状态
- `相关设定/故事状态.md` — 当前故事快照
- `相关设定/当前冲突.md` ⭐ — 短期核心冲突卡（用于评估本章节拍推进是否到位）
- `相关设定/伏笔追踪.md` — 已埋/待回收的伏笔
- `相关设定/创作风格.md` — 风格基调
- `相关设定/写作注意事项.md` — 硬性约束

#### 必读（实体设定 — 由 novel-lite 维护）
- `相关设定/角色/*.md` — 本章出场角色
- `相关设定/物品/*.md`、`相关设定/技能/*.md` — 涉及项
- `创作标准/好的示例.md` 与 `差的示例.md`

#### 必读（待定设定）⭐ 新增
- `工作区/待定设定/_索引.md` — 总览
- 本章关联的 `工作区/待定设定/<类别>/<名称>.md` — 用于评估正文与待定设定的一致性，并提示可在本轮顺手做新设定审查

#### 参考（连续性）
- `正文/第X-1章/*` — 上一定稿章节，用于衔接
- `版本存档/第X章/*` — 同章节历史版本，用于对照

**原则**：审查前必须读完上述文件。无完整上下文，不出审查报告。缺失文件需提示用户或先请求 novel-setup/novel-lite 补齐。

### 正文审查七维度

每一轮正文审查必须在以下七个维度中都产出至少 1 个问题，总计 10-20 个问题。

### 1. 情节深化（Plot Depth）
挖掘戏剧张力、节奏、反转、留白。
- 本章冲突是否真实有压力？主角面临的代价是什么？
- 是否存在"事件发生了但情感/态度没有变化"的无效情节？
- 节奏是否单一（一路顺利/一路追击）？能否加入反向节拍？
- 是否有可以埋伏笔的位置被直述浪费？
- 章末钩子是否足够拉动下一章？

### 2. 设定完善（Setting Integrity）
用正文倒逼设定补全，发现隐性设定。
- 正文中出现了哪些设定里没有、但其实需要明确的规则/参数？
- 现有设定是否被正文意外"扩写"或"偏离"？是更新设定还是修订正文？
- 同类实体（怪物/守卫/区域）能否抽象出更稳定的规则？
- 主角知晓状态是否需要从"未知"更新为"已知/部分了解"？
- 有没有"临时编的解释"需要正式写入设定防止后续打架？

### 3. 趣味提升（Engagement）
让读者更愿意读下去。
- 是否有让读者"会心一笑 / 屏住呼吸 / 想起就激动"的瞬间？
- 世界奇观展示是否充分？有没有只说不看的偷懒？
- 信息投放节奏是否吊胃口（悬念-线索-小揭晓循环）？
- 主角的小机灵、小挣扎、小反差有没有展现？
- 有没有可以强化的"标志性画面"（读者会记住的那一幕）？

### 4. 逻辑严密（Logical Rigor）
堵漏，防止读者出戏。
- 物理/空间/时间是否自洽？（距离、时长、体力、天气、光线）
- 角色动机是否充分？是否有"为了剧情而行动"？
- 规则是否被一致执行？（怪物弱点、技能代价、守卫规则）
- 前文设置的约束在后文是否被无声违反？
- "主角刚好想起某事"等方便性桥段是否需要铺垫？

### 5. 视角与信息对称（POV & Info Symmetry）
遵守 `写作注意事项.md` 的硬性约束。
- 是否出现了主角看不到/听不到/感觉不到的描写？
- 是否向读者透露了主角未知的真相？
- 主角心理活动与已知信息是否一致？（不该知道的不能想）
- 设定中"主角未知"项是否在正文中被错误展示？

### 6. 角色塑造（Character Craft）
让人物立起来。
- 本章主角的关键决定是否体现其性格底色（谨慎/果敢/犹豫……）？
- 对白是否符合身份、年龄、心境？有没有"作者在说话"？
- 配角/怪物/守卫是否只是功能道具，能否多一笔让其"活"一下？
- 主角在本章有无"微小但真实的成长 / 变化 / 代价"？

### 7. 长线连续性（Long-arc Continuity）⭐ 新增
对照 novel-setup 维护的宏观文件，堵宏观层漏洞。
- **大纲贴合**：本章实际发生与 `故事大纲.md` 预期是否偏离？偏离是有意还是失控？
- **当前冲突推进** ⭐：本章对 `当前冲突.md` 中的节拍是否有可勾选的推进？是否在啃靶心还是在打游击？是否触发演化（达成/失败/转化）？
- **冲突压力维持** ⭐：本章是否让主角持续感受到当前冲突的代价/时限/威胁？还是冲突已被淡化到读者也感觉不到？
- **状态连贯**：主角入场状态与 `故事状态.md` 是否一致？出场后状态是否自然衔接下一章定位？
- **伏笔回收**：`伏笔追踪.md` 中是否有本章"理应回收或强化"却被漏掉的伏笔？
- **伏笔埋设**：本章新发生的关键暗示，是否应登记为新伏笔？
- **弧光推进**：本章对主角 `主角档案.md` 中的弧光节点是否有实质推进？是否出现"原地打转"章节？
- **跨章一致**：与 `正文/第X-1章/` 的衔接是否自然？（地点/时间/情绪/主角状态）
- **支线活跃度**：在场活跃支线是否有合理进展，还是长期搁置？

### 正文审查工作流程

#### 第一步：加载上下文

按"审查对象"清单读取全部相关文件。若关键文件缺失：
- 宏观设定缺失 → 提示用户"建议先用 novel-setup 补齐 XX"，但可以继续出审查报告（标注"受限审查"）
- 章节三件套缺失 → 阻塞，提示先用 novel-lite 补齐

同时读取 `工作区/第X章VX/审查历史/` 下历次报告，避免重复提问。

#### 第二步：确定子模式

| 子模式 | 适用场景 | 范围 |
|------|---------|------|
| **首次全量审查** | 工作区无审查历史 | 七维度全覆盖，10-20 题 |
| **增量审查** | 本次修订基于上一份报告 | 只看本轮改动+上轮未解问题，6-12 题 |
| **定稿终审** | 准备移入正文/ | 七维度全覆盖+P0零容忍，侧重逻辑/连续性 |

用户未指定时默认"首次全量"或"增量"（根据审查历史自动判断）。

#### 第三步：产出审查报告

**报告固定路径**：`工作区/第X章VX/审查历史/review_YYYYMMDD_HHMM.md`

#### 第四步：驱动审查循环

报告出完后，**不自动改稿**。让用户逐题回应：
- 用户回答"这里想改成 X" → 调用 novel-lite 三选项机制修改
- 用户回答"这是设定，忘了写进去" → 更新对应设定文件
- 用户回答"问题不成立，因为 Y" → 记录理由，必要时把 Y 补进设定或创作标准
- 用户回答涉及大纲/伏笔 → 提示调用 novel-setup 处理

每轮修订后，询问是否需要**再次审查**（默认走增量模式）。

#### 第五步：沉淀经验

一轮循环结束时，主动询问是否将以下内容沉淀：
- 发现的优秀表达 → `创作标准/好的示例.md`
- 发现的反面教材 → `创作标准/差的示例.md`
- 新的硬性约束 → 提示 novel-setup 更新 `相关设定/写作注意事项.md`
- 新增/更新的实体设定 → 提示 novel-lite 更新
- 主角知晓状态变化 → 提示 novel-lite 更新对应实体
- 大纲偏离 / 新伏笔 → 提示 novel-setup 更新

## 模式 B：新设定审查（review-pending）

### 定位

针对 `工作区/待定设定/` 池中的草案，逐条评估并给出**晋升 / 调整后晋升 / 保留待审 / 废弃**四选一建议。
**目标不是产出问题让作者答**，而是**辅助决策**。

### 审查对象

#### 必读
- `工作区/待定设定/_索引.md` — 总览
- 待审范围内的全部 `工作区/待定设定/<类别>/<名称>.md`
- `相关设定/世界设定.md`、`故事大纲.md`、`主角档案.md`、`写作注意事项.md`、`伏笔追踪.md` — 用于一致性比对
- `相关设定/角色/`、`物品/`、`技能/` 中可能与待定项重名/重叠/冲突的文件
- 起源章节的正文（已定稿则在 `正文/第X章/`，未定稿则 `工作区/第X章VX/`）

#### 范围选择
- 默认：`_索引.md` 中状态为 🟠 待审 的全部条目
- 可选：用户指定某几条 / 某章关联的全部条目

### 审查五维度（针对每一条待定项）

每条待定项都要从这 5 个维度评估，每个维度给出一句话判定：

1. **必要性**：这条设定是否必须存在？能否用更克制的写法（隐喻/不解释）规避？
2. **一致性**：与既有 `相关设定/` 是否冲突？与世界规则/主角认知/伏笔是否打架？
3. **完备性**：作为正式设定是否信息够（边界、例外、代价、知晓状态）？还是只是个名字？
4. **可扩展性**：将来章节会反复使用吗？是否会反向限制后续剧情？
5. **归属**：晋升后应放在哪？`相关设定/角色|物品|技能/` 还是 `世界设定.md` / `写作注意事项.md` / `伏笔追踪.md`（后者需 novel-setup 接手）

### 决策建议四选一

每条待定项最终给一个**明确处置建议**（用户可推翻）：

- **🟢 直接晋升**：信息完备、无冲突、必要 → 建议路径与目标文件
- **🟡 调整后晋升**：方向对，但需补充/修正 X、Y → 列出修改清单
- **🟠 保留待审**：信息不足或时机未到，建议在第 Z 章后再决定
- **⚫ 建议废弃**：可去掉而不伤正文，或与既有设定严重冲突 → 给出回改正文的最小改动建议

### 工作流程

#### 第一步：加载上下文
读完待审范围条目 + 既有设定 + 起源章节正文。

#### 第二步：扫描冲突
对每条待定项，主动检索 `相关设定/` 中可能冲突的位置（命名重复、规则矛盾、主角知晓状态相反）。

#### 第三步：产出决策报告

**报告路径**：`工作区/待定设定/审查历史/pending_review_YYYYMMDD_HHMM.md`
（如目录不存在则创建）

## 模式 C：设定体检（review-canon）

### 定位

对 `相关设定/` 下**已定稿设定**做系统性健康检查。不审正文、不审待定项，只看设定文件之间的关系与各自质量。

### 审查对象

#### 必读
- `相关设定/` 下全部文件（递归）
- `相关设定/伏笔追踪.md` 与最近 5 章 `正文/` 的"事件摘要.md"——用于判断设定是否被实际剧情验证或偏离
- `相关设定/故事状态.md` —— 用于判断设定是否过时

### 体检四维度

#### 1. 一致性（Consistency）
- 角色 A 在 `角色/A.md` 与 `主角档案.md` 的描述是否一致（关系、能力、当前状态）？
- 同一规则在 `世界设定.md` 与 `写作注意事项.md` 是否有不同表述？
- 物品/技能 A 与 B 是否互相矛盾（如两个独立"唯一"道具）？
- 伏笔列表中"已埋"位置是否与正文实际一致？

#### 2. 时效性（Freshness）
- `故事状态.md` 是否与最近一章定稿同步？
- 角色 A 的"出场记录"是否漏记？最后出场章号与实际是否一致？
- 主角已知/未知清单是否反映最新认知？
- 是否有"过时设定"——曾经为真但已被剧情覆盖（建议加历史标注或更新）？

#### 3. 完备性（Completeness）
- 出场频次高的角色是否信息齐全（关系网、动机、口头禅）？
- 关键规则是否有边界/例外/代价说明？
- 关键物品是否标明主角知晓状态？
- 长期 🔴 待回收伏笔是否仍在合理超期内？

#### 4. 健康度（Health Metrics）
- 文件数量分布：角色/物品/技能/规则各几何？是否过度膨胀（角色 >50 提示精简）？
- 待回收伏笔数：>10 提示积压
- 大纲贴合度：实际章数 vs 大纲预期偏移
- 设定 vs 正文的反向同步差距：发现 >3 章未回写到设定的剧情变化

## 模式 D：阶段回顾（review-arc） ⭐ 新增

### 定位

每写完 5 章，强制做一次**中频回看**。
- 单章审查（模式 A）只看本章，看不见跨章的设定漂移；
- 设定体检（模式 C）频率太低（卷末/百章），日常设定与正文的同步差距会持续放大；
- 阶段回顾（本模式）介于两者之间，**每 5 章一次**，是宏观-中观-正文之间的"对账"机制。

**核心问题**：最近 5 章实际写出来的故事，是否已经悄悄修改了相关设定？当前冲突是否还成立？哪些设定需要更新、哪些伏笔漏登记了、哪些节拍该勾掉了？

### 触发场景

- novel-lite 在第 N 章定稿后判断 `N % 5 == 0`，主动触发提示
- 用户主动请求"阶段回顾 / 回看最近5章 / 检查最近5章设定同步"
- 上次阶段回顾被跳过累积 ≥10 章未做时，强烈触发

### 审查对象

#### 必读（最近 5 章正文）
- `正文/第N-4章/` ~ `正文/第N章/` 全部三件套
- 5 章对应的"事件摘要.md"——快速建立时间线
- 5 章对应的"相关实体.md"——找出本阶段实际被使用/扩写的实体

#### 必读（宏观文件）
- `相关设定/故事大纲.md`
- `相关设定/当前冲突.md` ⭐ — 本模式的核心审查对象之一
- `相关设定/故事状态.md`
- `相关设定/主角档案.md`
- `相关设定/伏笔追踪.md`
- `相关设定/写作注意事项.md`

#### 必读（实体设定）
- 5 章中出现频次 ≥2 的角色/物品/技能文件
- `工作区/待定设定/_索引.md` 与本阶段产生的待定项

#### 参考
- 上一份阶段回顾报告（如有）：`相关设定/_阶段回顾历史/arc_review_第X-Y章_*.md`
- 最近一次设定体检报告（如有）

### 阶段回顾五维度

#### 1. 当前冲突演化（Conflict Evolution）⭐ 核心
- 5 章中实际推进的当前冲突节拍有哪些？是否与 `当前冲突.md` 中的预期节拍一致？
- 是否有节拍被跳过？是被剧情自然消解，还是漏写了？
- 是否触发了冲突演化（达成/失败/转化）？是否需要归档当前冲突卡并设立下一段？
- 主角对当前冲突的态度/筹码/软肋有无变化？是否要更新冲突卡？
- 是否出现"5 章了主角还在干同一件小事"的停滞？或"5 章跨度太大，预期半卷的事 5 章干完"的失速？

#### 2. 设定同步差（Setting Drift）⭐ 核心
- 5 章正文中是否出现了**实际改写但没回写到设定**的情况？例如：
  - 角色 A 实际表现出某能力 / 关系 / 性格倾向，但 `角色/A.md` 仍旧
  - 某规则被正文具体化（数值/边界/代价），但 `世界设定.md` 未更新
  - 某物品被赋予新副作用，但 `物品/X.md` 未更新
- 主角已知/未知清单是否需要批量更新（5 章中获得了多少新认知）？
- 是否有"临时编的解释"被反复使用却仍停留在待定池？

#### 3. 伏笔活性（Foreshadowing Liveness）
- 5 章中是否有"暗示性事件"未被登记为伏笔？（漏登）
- `伏笔追踪.md` 中"计划在本阶段回收"的伏笔，是否实际回收了？还是被遗忘？
- 是否有伏笔在 5 章中"出现 3 次以上但仍 🔴 待回收"——属于反复挠痒，要么尽快回收要么明确延后
- 是否有伏笔在剧情演化中已不再可能回收，需要标记 ⚫ 已废弃？

#### 4. 主角弧光节奏（Arc Pacing）
- 5 章对主角弧光节点的推进幅度是否合理？（每 5 章应至少有 1 次微小变化）
- 主角档案中"当前状态"是否与第 N 章末实际状态一致？
- 是否出现弧光"原地打转 5 章"或"瞬间跳到下一节点"的节奏失衡？

#### 5. 大纲贴合度（Outline Alignment）
- 5 章实际发生事件 vs `故事大纲.md` 的预期，偏离度多大？
- 偏离是有意（剧情需要）还是失控（写嗨了）？
- 偏离合理 → 建议 novel-setup 更新大纲；不合理 → 建议回改后续章节走向

- 即使发现需要回改正文，也只**建议**不**执行**——回改通过 novel-lite 单章修订进行
- 不擅自修改 `当前冲突.md`，只给"是否需要演化"的判定与建议方向

## 提问质量准则

**好问题的特征**（都要具备）：
1. **具体**：引用正文片段或明确位置，不说"整体感觉薄"
2. **开放**：不是 yes/no，而是激发思考的 how/why/what-if
3. **建设**：暗含至少一个改进方向，但不替作者做决定
4. **可答**：作者能在 5 分钟内形成判断，不是需要重写全章的大哉问
5. **单点**：一个问题只聚焦一件事，避免"并且""同时"堆叠

**反面示例**（禁止）：
- ❌ "这章写得怎么样？"（太虚）
- ❌ "要不要加点感情戏？"（无依据）
- ❌ "第三段逻辑有问题吗？"（不具体）
- ❌ "主角性格和动机以及配角塑造是不是都不够？"（多点合一）

**正面示例**：
- ✅ "正文中林轲'想也不想一树枝抽过去'与设定的'谨慎'性格冲突——是性格在战斗应激下的合理偏离（需在叙述中点出反差），还是应调整动作让谨慎性格仍然可见？"
- ✅ "失心者'瞬移后短暂僵直'来自课堂知识，但主角在战斗中并未尝试利用这一点就突围了——这条弱点是本章留作下章伏笔（需登记 F0XX），还是应在本章战斗中有一次验证/否证？"
- ✅ "伏笔追踪.md 中 F003（老槐树下的铁盒）已埋设 3 章未回收，本章主角回到老槐树附近却未触及——是继续延迟以加强悬念（建议更新 F003 计划回收章），还是应在本章补一次视线扫过？"

- **共享文件结构**：审查读写的都是 setup+lite 定义的标准目录
- **报告路径固定**：
  - 正文审查：`工作区/第X章VX/审查历史/review_YYYYMMDD_HHMM.md`
  - 新设定审查：`工作区/待定设定/审查历史/pending_review_YYYYMMDD_HHMM.md`
  - 设定体检：`相关设定/_体检历史/checkup_YYYYMMDD_HHMM.md`
  - 阶段回顾 ⭐：`相关设定/_阶段回顾历史/arc_review_第N-4章_第N章_YYYYMMDD_HHMM.md`
- **修订引用报告编号**：novel-lite 修订时需在正文注释块或决策日志引用本报告文件名与问题/条目号
- **定稿前必审**：章节从 工作区/ 移入 正文/ 之前，至少跑一轮正文"定稿终审"，且本章关联的 🟡/🟠 待定项必须经过新设定审查
- **待定项审视触发**：novel-lite 累积 ≥3 条 🟠 待审 项时，建议跑模式 B；定稿前必跑
- **阶段回顾触发** ⭐：novel-lite 在第 N 章定稿后判断 N%5==0 时主动提示，由本 skill 跑模式 D；阶段回顾报告中"由 novel-lite 落地"的实体设定整改项，由 novel-lite 执行

## 与 novel-setup 的衔接

- 审查结果涉及大纲偏离、伏笔状态、主角弧光变化的 → 在报告"移交行动项"中明确标注需调用 novel-setup
- 审查时若发现 `故事状态.md` 已过期（主角状态与本章入场不符），提示用户调用 setup 刷新
- 模式 B 中，待定项归属为"世界规则/伏笔/大纲"的，晋升执行交给 novel-setup 而非 novel-lite
- 模式 C 是 novel-setup「全局体检」的设定层补充：setup 看进度/弧光/支线/伏笔状态，本模式看设定文件本身的一致性与健康；用户做大版本里程碑（卷末/定稿前）时建议两者各跑一次
- 模式 D ⭐ 是宏观-中观-正文的中频对账机制：阶段回顾报告中"由 novel-setup 接手"的项（含当前冲突演化、大纲微调、主角弧光节点更新、伏笔补登/废弃、世界设定追加等）由 novel-setup 落地；阶段回顾发现的"当前冲突需归档并设立下一段"是本模式与 novel-setup 协作的最高频场景

## 边界

- 不做"打分"（80/100 这种），质量是多维的，分数会误导
- 不替代用户判断，永远给方向不给结论
- 不主动改写原文，除非用户明确要求
- 不擅自修改任何设定文件（包括 `相关设定/` 与 `工作区/待定设定/`）；发现问题只在报告中提出，由用户决策后交 lite/setup 执行
- 遇到设定冲突，先指出、再询问以哪边为准，不擅自选边
- 模式 D 不擅自演化 `当前冲突.md`，只产出"建议归档/调整节拍/下一段冲突候选方向"，由 novel-setup 经用户确认后执行
- 模式 B 不替用户做晋升，只给建议；模式 C 不替用户改设定，只给清单

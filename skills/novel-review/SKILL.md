# 小说章节审查

## 定位

本 skill 是 **novel-lite 创作 skill 的配套审查工具**。novel-lite 负责"写"，本 skill 负责"审"，二者共享同一套文件结构（工作区/、相关设定/、创作标准/）。

审查不是评分，而是**用专业问题激发思考**，让作者在回答问题的过程中发现情节、设定、逻辑、趣味的可提升点，并沉淀为修订后的章节与更完善的设定。

## 审查对象

### 必读（章节上下文）
- `工作区/第X章VX/正文.md`：被审查的正文
- `工作区/第X章VX/事件摘要.md`：本章骨架
- `工作区/第X章VX/相关实体.md`：本章涉及的实体状态

### 必读（项目级设定）
- `相关设定/世界设定.md`：世界观基底
- `相关设定/故事大纲.md`：本章在整体走向中的定位
- `相关设定/创作风格.md`：风格基调
- `相关设定/写作注意事项.md`：硬性约束（视角、信息对称等）
- `相关设定/角色/*.md`：本章出场角色的设定与主角知晓状态
- `相关设定/物品/*.md`、`相关设定/技能/*.md`：涉及项
- `创作标准/好的示例.md` 与 `差的示例.md`：已沉淀的经验

### 参考（连续性）
- `正文/第X-1章/*`：上一定稿章节，用于衔接检查
- `版本存档/第X章/*`：同章节历史版本，用于对照

**原则**：审查前必须读完上述文件。无完整上下文，不出审查报告。

## 审查六维度

每一轮审查必须在以下六个维度中都产出至少 1 个问题，总计 10-20 个问题。

### 1. 情节深化（Plot Depth）
挖掘剧情的戏剧张力、节奏、反转、留白。
- 本章冲突是否真实有压力？主角面临的代价是什么？
- 是否存在"事件发生了但情感/态度没有变化"的无效情节？
- 节奏是否单一（一路顺利/一路追击）？能否加入反向节拍？
- 是否有可以埋伏笔的位置被直述浪费？
- 章末钩子是否足够拉动下一章阅读欲？

### 2. 设定完善（Setting Integrity）
用正文倒逼设定补全，发现隐性设定。
- 正文中出现了哪些设定里没有、但其实需要明确的规则/参数？
- 现有设定是否被正文意外"扩写"或"偏离"？是更新设定还是修订正文？
- 同类实体（如怪物/守卫/区域）是否能抽象出更稳定的规则？
- 主角知晓状态是否需要从"未知"更新为"已知/部分了解"？
- 有没有"临时编的解释"需要正式写入设定防止后续打架？

### 3. 趣味提升（Engagement）
让读者更愿意读下去。
- 是否有让读者"会心一笑 / 屏住呼吸 / 想起就激动"的瞬间？没有的话哪里可以加？
- 世界奇观展示是否充分？有没有只说没看的偷懒？
- 信息投放节奏是否吊胃口（悬念-线索-小揭晓循环）？
- 主角的小机灵、小挣扎、小反差有没有展现？
- 有没有可以强化的"标志性画面"（读者会记住的那一幕）？

### 4. 逻辑严密（Logical Rigor）
堵漏，防止读者出戏。
- 物理/空间/时间上是否自洽？（距离、时长、体力、天气、光线）
- 角色行为动机是否充分？是否有"为了剧情而行动"的情况？
- 规则是否被一致执行？（怪物弱点、技能代价、守卫规则）
- 前文设置的约束在后文是否被无声违反？
- "主角刚好想起课本知识"等方便性桥段是否需要铺垫？

### 5. 视角与信息对称（POV & Info Symmetry）
遵守 `写作注意事项.md` 的硬性约束。
- 是否出现了主角看不到/听不到/感觉不到的描写？
- 是否向读者透露了主角未知的真相？
- 主角的"心理活动"与"已知信息"是否一致？（不该知道的不能想）
- 设定中的"主角未知"项，在正文中是否被错误展示？

### 6. 角色塑造（Character Craft）
让人物立起来。
- 本章主角做的关键决定，是否体现了其设定性格（谨慎/果敢/犹豫……）？
- 对白是否符合身份、年龄、心境？有没有"作者在说话"？
- 配角/怪物/守卫是否只是功能道具，能否多一笔让其"活"一下？
- 主角在本章有无"微小但真实的成长 / 变化 / 代价"？

## 工作流程

### 第一步：加载上下文

按"审查对象"清单读取全部相关文件。若文件缺失，先告知用户并继续（不阻塞）。

### 第二步：产出审查报告

以下面结构输出，使用 Markdown：

```markdown
# 第X章VX 审查报告

## 一句话印象
[一句话概括本章当前质量与核心问题，不超过50字]

## 亮点（必须 2-4 条）
- [具体亮点 + 所在段落/原句片段]
- ...

## 审查问题（10-20 个，按优先级分级）

### 🔴 P0 必须解决（阻塞性问题，3-6 个）
**Q1 [维度标签]**：[问题本体，一句话]
- **定位**：[原文片段 / 摘要条目 / 设定文件]
- **为什么重要**：[简述影响]
- **思考方向**（可选）：[2-3 个可能的修改方向，不下结论]

### 🟡 P1 建议改进（提升性问题，4-8 个）
（同上结构）

### 🟢 P2 可选打磨（锦上添花，2-6 个）
（同上结构）

## 维度覆盖清单
- 情节深化：Q? Q? 
- 设定完善：Q?
- 趣味提升：Q? Q?
- 逻辑严密：Q? Q?
- 视角/信息：Q?
- 角色塑造：Q?

## 建议的修订顺序
1. 先处理 [某组问题]，因为[原因]
2. 再处理 ...
```

### 第三步：驱动审查循环

报告出完后，**不自动改稿**。让用户逐题回应：
- 用户回答"这里我想改成 X"→ 调用 novel-lite 的三选项机制进行修改
- 用户回答"这是设定，我忘了写进去"→ 更新对应设定文件
- 用户回答"这个问题不成立，因为 Y"→ 记录理由，必要时把 Y 补进设定或创作标准

每轮修订后，询问是否需要**再次审查**（增量审查：只看本轮改动引发的新问题）。

### 第四步：沉淀经验

一轮循环结束时，主动询问是否将以下内容沉淀：
- 发现的优秀表达 → `创作标准/好的示例.md`
- 发现的反面教材 → `创作标准/差的示例.md`
- 新的硬性约束 → `相关设定/写作注意事项.md`
- 新增/更新的设定 → `相关设定/**`
- 主角知晓状态变化 → 对应角色/物品/技能设定

## 提问质量准则

**好问题的特征**（都要具备）：
1. **具体**：引用正文片段或明确位置，不说"整体感觉薄"这种虚话
2. **开放**：不是 yes/no，而是激发作者思考的 how/why/what-if
3. **建设**：暗含至少一个改进方向，但不替作者做决定
4. **可答**：作者能在 5 分钟内形成判断，不是需要重写全章的大哉问
5. **单点**：一个问题只聚焦一件事，避免"并且""同时"堆叠

**反面示例**（禁止）：
- ❌ "这章写得怎么样？" （太虚）
- ❌ "要不要加点感情戏？" （无依据）
- ❌ "第三段逻辑有问题吗？" （不具体）
- ❌ "主角性格和动机以及配角塑造是不是都不够？" （多点合一）

**正面示例**：
- ✅ "正文中林轲'想也不想一树枝抽过去'与设定的'谨慎'性格冲突——是性格在战斗应激下的合理偏离（需要在叙述中点出这种反差），还是应调整动作让谨慎性格仍然可见？"
- ✅ "失心者'瞬移后短暂僵直'来自课堂知识，但主角在战斗中并未尝试利用这一点就突围了——这条弱点是本章留作下章伏笔，还是应在本章战斗中有一次验证/否证？"

## 数量与分布建议

| 章节篇幅 | 问题总数 | P0 | P1 | P2 |
|---------|---------|----|----|----|
| <2000字 | 10-12 | 3 | 4-5 | 3-4 |
| 2000-4000字 | 13-16 | 4-5 | 5-6 | 4-5 |
| >4000字 | 17-20 | 5-6 | 7-8 | 5-6 |

## 与 novel-lite 的衔接

- **审查不改稿**：本 skill 只产出问题与建议，实际修改走 novel-lite 的三选项机制
- **共享文件结构**：审查读写的都是 novel-lite 定义的标准目录
- **循环闭环**：审查 → 用户决策 → novel-lite 修订 → 再审查 → 沉淀标准
- **定稿前必审**：建议章节从 工作区/ 移入 正文/ 之前，至少跑一轮审查循环

## 边界

- 不做"打分"（80/100 这种），质量是多维的，分数会误导
- 不替代用户判断，永远给方向不给结论
- 不主动改写原文，除非用户明确要求
- 遇到设定冲突，先指出、再询问以哪边为准，不擅自选边

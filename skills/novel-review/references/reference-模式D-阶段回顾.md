# 模式D：阶段回顾（review-arc）

每写完5章，强制做一次中频回看。

---

## 定位

核心问题：最近5章实际写出来的故事，是否已经悄悄修改了相关设定？当前冲突是否还成立？哪些设定需要更新？

---

## 触发场景

| 触发方式 | 说明 |
|---------|------|
| novel-lite提示 | N%5==0定稿后主动提示 |
| 用户请求 | "阶段回顾"、"回看最近5章" |
| 累积跳过 | ≥10章未做时强烈触发 |

## 阶段回顾六维度

### 1. 当前冲突演化 ⭐核心

| 检查项 | 说明 |
|--------|------|
| 节拍推进 | 5章中实际推进了哪些节拍？是否与预期一致？ |
| 节拍跳过 | 是否有节拍被跳过？ |
| 演化触发 | 是否触发冲突演化（达成/失败/转化）？ |
| 主角态度变化 | 主角对冲突的态度/筹码/软肋有无变化？ |
| 停滞检测 | 是否"5章主角还在干同一件小事"？ |
| 失速检测 | 是否"5章跨度太大"？ |

### 2. 设定同步差 ⭐核心

| 检查项 | 说明 |
|--------|------|
| 实际改写未回写 | 正文中是否有实际改写但没回写到设定的？ |
| 知晓清单更新 | 主角已知/未知清单是否需要批量更新？ |
| 临时解释滥用 | 是否有"临时编的解释"被反复使用但仍待定？ |

### 3. 伏笔活性

| 检查项 | 说明 |
|--------|------|
| 漏登检测 | 5章中是否有暗示性事件未被登记为伏笔？ |
| 回收检查 | "计划本阶段回收"的伏笔是否实际回收？ |
| 超期检测 | 是否有伏笔出现≥3次但仍🔴待回收？ |
| 废弃检测 | 是否有伏笔已不再可能回收需标记⚫废弃？ |

### 4. 主角弧光节奏

| 检查项 | 说明 |
|--------|------|
| 推进幅度 | 5章对弧光节点推进是否合理？（至少1次微小变化） |
| 状态一致 | 主角档案"当前状态"是否与第N章末实际一致？ |
| 异常检测 | 是否"原地打转5章"或"瞬间跳到下一节点"？ |

### 5. 大纲贴合度

| 检查项 | 说明 |
|--------|------|
| 偏离度计算 | 5章实际事件 vs 大纲预期偏离度 |
| 偏离性质 | 偏离是有意还是失控？ |
| 建议处置 | 合理偏离→建议更新大纲；不合理→建议回改 |

### 6. 风格漂移

| 检查项 | 说明 |
|--------|------|
| 自检分数趋势 | 5章自检总分趋势？是否持续下降或大幅波动？ |
| 指标漂移 | 风格指纹指标是否系统性漂移？ |
| 场景失分 | 是否有某类场景必做项在5章中屡屡失分？ |
| 阈值建议 | 连续2章自检<80是否建议调整自检阈值？ |

## 工作流程

1. 建立阶段时间线（读取5章事件摘要，形成事件矩阵）
2. 六维度比对（逐维度比对设定与正文实际）
3. 产出阶段回顾报告（写入固定路径）
4. 等待用户决策（不擅自改设定）
5. 用户决策后由对应skill落地，追加落地记录

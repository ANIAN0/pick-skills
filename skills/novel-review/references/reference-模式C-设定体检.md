# 模式C：设定体检（review-canon）

对 `相关设定/` 下已定稿设定做系统性健康检查。

---

## 审查对象

### 必读文件

| 类型 | 文件 | 说明 |
|------|------|------|
| 全量设定 | `相关设定/` 下全部文件 | 体检主体 |
| 伏笔 | `伏笔追踪.md` | 伏笔健康度 |
| 最近章节 | 最近5章事件摘要 | 设定同步检查 |
| 故事状态 | `故事状态.md` | 时效性检查 |

---

## 体检四维度

### 1. 一致性（Consistency）

| 检查项 | 说明 |
|--------|------|
| 角色描述一致 | 同一角色在多文件中描述是否一致 |
| 规则描述一致 | 同一规则在多文件中表述是否一致 |
| 物品/技能互斥 | 是否有互相矛盾的设定 |
| 伏笔位置一致 | 伏笔列表与正文实际是否一致 |

### 2. 时效性（Freshness）

| 检查项 | 说明 |
|--------|------|
| 故事状态同步 | 是否与最近定稿章节同步 |
| 出场记录完整 | 角色出场记录是否漏记 |
| 知晓状态更新 | 主角已知/未知是否反映最新认知 |
| 过时设定 | 是否有被剧情覆盖的旧设定 |

### 3. 完备性（Completeness）

| 检查项 | 说明 |
|--------|------|
| 高频角色完整 | 出场频次高的角色信息是否齐全 |
| 规则有边界 | 关键规则是否有边界/代价说明 |
| 物品知晓状态 | 关键物品是否标明主角知晓状态 |
| 伏笔超期 | 待回收伏笔是否在合理超期内 |

### 4. 健康度（Health Metrics）

| 指标 | 健康区间 |
|------|---------|
| 待回收伏笔数 | ≤10 |
| 最老待回收伏笔距埋设章数 | ≤10 |
| 大纲贴合度 | ±1章 |
| 待定设定池规模 | ≤15 |
| 角色文件数 | —（无硬限制） |
| 主角已知未知清单覆盖率 | ≥90% |

## 工作流程

1. 加载上下文（扫描设定全部文件+最近5章摘要）
2. 执行四维度检查（对每个维度产出问题清单）
3. 产出体检报告（写入固定路径）
4. 等待用户决策（不擅自改设定文件）
5. 用户决策后由对应skill落地

# 模式C：设定体检（review-canon）

## 定位

对 `相关设定/` 下**已定稿设定**做系统性健康检查。不审正文、不审待定项，只看设定文件之间的关系与各自质量。

## 审查对象

### 必读
- `相关设定/` 下全部文件（递归）
- `相关设定/伏笔追踪.md` 与最近 5 章 `正文/` 的"事件摘要.md"
- `相关设定/故事状态.md`

## 体检四维度

### 1. 一致性（Consistency）
- 角色 A 在 `角色/A.md` 与 `主角档案.md` 的描述是否一致？
- 同一规则在 `世界设定.md` 与 `写作注意事项.md` 是否有不同表述？
- 物品/技能 A 与 B 是否互相矛盾？
- 伏笔列表中"已埋"位置是否与正文实际一致？

### 2. 时效性（Freshness）
- `故事状态.md` 是否与最近一章定稿同步？
- 角色"出场记录"是否漏记？
- 主角已知/未知清单是否反映最新认知？
- 是否有"过时设定"——曾经为真但已被剧情覆盖？

### 3. 完备性（Completeness）
- 出场频次高的角色是否信息齐全？
- 关键规则是否有边界/例外/代价说明？
- 关键物品是否标明主角知晓状态？
- 长期 🔴 待回收伏笔是否仍在合理超期内？

### 4. 健康度（Health Metrics）
- 文件数量分布：角色/物品/技能/规则各几何？角色 >50 提示精简
- 待回收伏笔数：>10 提示积压
- 大纲贴合度：实际章数 vs 大纲预期偏移
- 设定 vs 正文的反向同步差距：>3 章未回写到设定的剧情变化

## 工作流程

1. **加载上下文**：扫描 `相关设定/` 全部文件 + 最近 5 章事件摘要 + 故事状态
2. **执行四维度检查**：对每个维度产出问题清单
3. **产出体检报告**：按模板写入固定路径
4. **等待用户决策**：不擅自改设定文件，由用户决策后交 lite/setup 落地

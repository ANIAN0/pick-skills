# 模式B：新设定审查（review-pending）

针对待定设定池中的草案，逐条评估给出晋升/调整/保留/废弃建议。

---

## 审查对象

### 必读文件

| 类型 | 文件 | 说明 |
|------|------|------|
| 待定设定 | `工作区/待定设定/_索引.md` | 待审范围 |
| 待定设定 | `工作区/待定设定/<类别>/<名称>.md` | 待审条目 |
| 宏观设定 | `世界设定.md`、`故事大纲.md` | 冲突检查 |
| | `主角档案.md`、`写作注意事项.md` | 冲突检查 |
| | `伏笔追踪.md` | 冲突检查 |
| 实体设定 | `相关设定/角色/*.md` 等 | 重名/重叠检查 |
| 起源章节 | 相关章节正文 | 上下文 |
| 风格设定 | `创作风格/_总纲.md` | 如涉及风格规则 |

### 审查范围选择

| 触发场景 | 审查范围 |
|---------|---------|
| 默认 | 全部🟠待审项 |
| 用户指定 | 特定ID或类别 |
| 定稿前 | 本章起源项+用户指定 |
| novel-lite触发 | 全部🟠待审 |

---

## 审查五维度

| 维度 | 检查内容 |
|------|---------|
| 必要性 | 是否必须存在？能否用更克制的写法规避？ |
| 一致性 | 与既有设定是否冲突？世界规则/主角认知/伏笔是否打架？ |
| 完备性 | 作为正式设定信息是否够（边界、代价、知晓状态）？ |
| 可扩展性 | 后续章节会反复使用吗？是否限制后续剧情？ |
| 归属 | 晋升后放在哪？实体目录还是宏观文件？ |

---

## 决策建议四选一

| 建议 | 条件 | 处理 |
|------|------|------|
| 🟢 直接晋升 | 信息完备、无冲突、必要 | 给出目标路径 |
| 🟡 调整后晋升 | 方向对但需补充/修正 | 列出修改清单 |
| 🟠 保留待审 | 信息不足或时机未到 | 建议第Z章后再决定 |
| ⚫ 建议废弃 | 可去掉不伤正文，或严重冲突 | 给回改建议 |

## 工作流程

1. 加载上下文（待审条目+既有设定+起源章节）
2. 扫描冲突（检索可能冲突位置）
3. 产出决策报告（写入固定路径）
4. 等待用户决策（不擅自改文件）
5. 用户决策后由对应skill落地

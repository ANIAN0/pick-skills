# 模式B：新设定审查（review-pending）

## 定位

针对 `工作区/待定设定/` 池中的草案，逐条评估并给出**晋升 / 调整后晋升 / 保留待审 / 废弃**四选一建议。
**目标不是产出问题让作者答**，而是**辅助决策**。

## 审查对象

### 必读（待定设定）
- `工作区/待定设定/_索引.md`
- 待审范围内的全部 `工作区/待定设定/<类别>/<名称>.md`
- `相关设定/世界设定.md`、`故事大纲.md`、`主角档案.md`、`写作注意事项.md`、`伏笔追踪.md`
- `相关设定/角色/`、`物品/`、`技能/` 中可能与待定项重名/重叠/冲突的文件
- 起源章节的正文
- `工作区/第X章VX/_预创作/02_必看文档.md`（若已生成，交叉验证）

### 必读（章节上下文，若已写正文）
- `工作区/第X章VX/正文.md`
- `工作区/第X章VX/相关实体.md`

### 范围选择
- **默认**：审查全部 🟠 待审 项
- **用户可指定**：仅审查特定 ID 或特定类别
- **定稿前**：优先审查本章起源的待定项
- **novel-lite 触发时**：若因"待定项累积≥3条"触发，审查全部 🟠 待审；若因"定稿前"触发，审查本章起源项 + 用户指定项

## 审查五维度（每条待定项）

1. **必要性**：这条设定是否必须存在？能否用更克制的写法规避？
2. **一致性**：与既有 `相关设定/` 是否冲突？与世界规则/主角认知/伏笔是否打架？
3. **完备性**：作为正式设定是否信息够（边界、例外、代价、知晓状态）？
4. **可扩展性**：将来章节会反复使用吗？是否会反向限制后续剧情？
5. **归属**：晋升后应放在哪？`相关设定/角色|物品|技能/` 还是 `世界设定.md` / `写作注意事项.md` / `伏笔追踪.md`

## 决策建议四选一

- **🟢 直接晋升**：信息完备、无冲突、必要 → 建议路径与目标文件
- **🟡 调整后晋升**：方向对，但需补充/修正 X、Y → 列出修改清单
- **🟠 保留待审**：信息不足或时机未到，建议在第 Z 章后再决定
- **⚫ 建议废弃**：可去掉而不伤正文，或与既有设定严重冲突 → 给出回改正文的最小改动建议

## 工作流程

1. **加载上下文**：读完待审范围条目 + 既有设定 + 起源章节正文
2. **扫描冲突**：对每条待定项，检索 `相关设定/` 中可能冲突的位置
3. **产出决策报告**：按模板写入固定路径
4. **等待用户决策**：不擅自改文件。用户在报告底部做最终决策后，由 novel-lite / novel-setup 落地

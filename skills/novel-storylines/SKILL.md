# Novel Storylines Skill

用于“分卷故事线 + 故事单元拆分”阶段，确保后续章节创作始终挂靠在当前单元目标上。

## 前置条件（强制）

必须满足：
1. `volumes_confirmed = true`

否则不得创建故事单元，先回到分卷阶段。

## 先后顺序（强制）

1. 先检查当前卷是否已有分卷故事线  
2. 没有则先创建分卷故事线  
3. 再拆分故事单元  
4. 再进入章节规划/正文创作  

## 输出文件

1. `02-分卷/volume-storylines.md`（分卷故事线）
2. `03-故事单元/unit-XXX.md`（故事单元详情）

### 分卷故事线格式

```markdown
# 分卷故事线

## 第X卷
- 卷名：
- 一句话描述：
- 章节数量目标：
- 本卷故事线（直接表达）：

### 单元列表（非创作阶段）
1. 单元1：一句话描述
2. 单元2：一句话描述
```

### 故事单元格式

```markdown
# 单元X

## 核心目标
[必须可执行、可检验]

## 核心事件（2-4个）
1. 事件A
2. 事件B

## 叙事弧
- 起因：
- 经过：
- 结果：
- 遗留问题（单元卡点）：

## 章节规划范围
- 建议章节数：8-15章
- 每个单一事件可拆：2-5章
```

## 约束规则

1. 单元必须围绕一个核心目标。  
2. 单元必须有强卡点，且该卡点可在下一单元被主角作为核心动机直接处理。  
3. 禁止超前续写与当前单元无关的支线。  
4. 表达必须直接，不用比喻、拟人、通感。  

## 交互机制

关键节点（分卷故事线、单元详情）都使用：
- 5 个选项 + 1 个推荐
- 用户明确确认后再定稿

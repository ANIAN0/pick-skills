# 审查报告模板

```markdown
# <文档或项目名称> 审查报告

## 结论
- 审查模式：需求审查 / 技术方案审查 / 计划审查 / 跨文档一致性审查
- 结论：可进入下一阶段 / 修正后可进入下一阶段 / 不可进入下一阶段
- 核心原因：<一句话>

## 发现

### REV-001 [BLOCKER|HIGH|MEDIUM|LOW] <标题>
- 位置：`<path>:<line>`
- 问题：<具体不一致、遗漏或不可验证点>
- 影响：<为什么会导致下一阶段失败、返工或验收无法证明>
- 建议：<需要补充的需求、契约、字段、API、数据、UI、验证或任务>

## 覆盖检查
| 链路 | 结果 | 说明 |
|------|------|------|
| R-* -> A-* | 通过/问题 | <说明> |
| A-* -> V-* | 通过/问题 | <说明> |
| R/A -> 字段级契约 | 通过/问题 | <说明> |
| C-* -> T-* | 通过/问题/不适用 | <说明> |
| V-* -> T-* 或全局验证 | 通过/问题/不适用 | <说明> |
| 最终闭环 | 通过/问题/不适用 | <说明> |

## 待确认
- <只有会改变下一阶段处理方式的问题才保留>

## 下一步
- <优先修复顺序>
```

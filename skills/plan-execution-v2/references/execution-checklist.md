# 执行检查清单

每个 `T-*` 任务都使用这份清单。

## 编辑前

- 读取 `index.md`。
- 读取当前任务文件。
- 检查关联的 `R-*`、`A-*`、`C-*` 和 `V-*`。
- 检查相关现有代码。
- 确认任务中没有未解决阻塞。

## 编辑中

- 保持在任务范围内。
- 除非任务明确要求改变，否则沿用现有代码模式。
- 不引入 mock、fallback 或 stub 完成。
- 如果执行细节有小变化，更新计划。
- 只有范围、数据、兼容性、安全或 UX 决策才询问用户。

## 验证

- 运行任务定义的验证。
- 变更触及共享行为时，运行邻近测试。
- UI 任务可行时，验证真实渲染行为。
- API/数据任务可行时，验证真实本地契约或集成行为。

## 证据日志格式

```markdown
## 证据日志
- 修改文件：
  - <路径>
- 执行命令：
  - `<命令>` -> <结果>
- UI/API 证据：
  - <观察结果>
- 与计划的偏差：
  - <无或详情>
- 验证状态：
  - done | not_verified | blocked
```

## 标记完成前

- 所有计划步骤已完成。
- 验证已通过。
- 证据已记录。
- 没有用捷径替代计划行为。
- 剩余风险已明确。

## 整体完成前

- 所有非跳过任务都有状态和证据。
- 最终闭环/放行任务已执行；若不适用，计划中有明确理由。
- 全局验证已运行。
- 无 mock、固定回复、硬编码测试 ID、测试旁路或降级实现冒充完成。
- `not_verified` 或 `blocked` 项不得被包装成整体完成。

## 待定设定处理流程

### 产出待定设定

创作中出现以下情况时，立即创建待定设定：
1. 正文出现 `相关设定/` 中没有的命名实体
2. 正文为已有实体扩展未审定属性
3. 正文给出世界规则的具体参数/边界
4. 临时编的解释需后续审视
5. 用户说"先这样定，回头再说"

**操作**：
1. 创建待定文件（按模板）
2. 更新索引（新增条目）

### 状态升级规则

| 条件 | 状态变化 |
|------|---------|
| 首次创建 | 🟡 草案 |
| 使用≥2次 | 🟡→🟠 待审（建议评估） |
| 使用≥3次且稳定 | 🟠→🟢 拟晋升（建议晋升） |
| 发现问题/冲突 | →⚫ 拟废弃 |

### 晋升流程

1. **审查**：novel-review模式B评估待定项
2. **通过**：按分工执行晋升
   - 角色/物品/技能 → novel-lite写入 `相关设定/`
   - 世界规则/伏笔 → novel-setup写入宏观文件
3. **清理**：删除待定文件，更新索引

### 废弃流程

1. **审查**：novel-review评估废弃建议
2. **确认**：用户确认废弃
3. **清理**：删除待定文件，更新索引
4. **回查**：检查相关章节是否有依赖，如有需修正

## 主角知晓状态说明

**核心原则**：正文只展示主角"已知"的信息，严禁展示主角"未知"的真相。

### 知晓方式

| 方式 | 含义 | 可展示？ |
|------|------|---------|
| 直接经历 | 主角亲身经历 | ✅ 可展示 |
| 他人告知 | 被其他角色告诉 | ✅ 可展示 |
| 传闻 | 听说的传说（可能真假混杂） | ✅ 可展示 |
| 未知 | 完全不了解 | ❌ 严禁展示 |

### 当前认知

| 认知状态 | 含义 |
|---------|------|
| 完全了解 | 掌握完整真相 |
| 部分了解 | 存在认知缺口 |
| 误解 | 与真相有偏差 |
| 毫无认知 | 完全无知 |

### 信息对称铁律

- 读者认知 ≤ 主角认知
- 主角"未知"的核心真相，正文不能直接展示
- 可以通过侧面暗示（其他角色反应），但不能让读者比主角知道更多核心真相

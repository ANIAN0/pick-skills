# 创作标准更新规范

## 更新触发时机

| 触发场景 | 谁执行 |
|---------|--------|
| 每次用户讨论后 | novel-lite / novel-review |
| novel-review审查循环结束后 | novel-lite |
| 阶段回顾结束后 | novel-lite / novel-setup |
| 用户主动要求"加入好/差示例" | novel-lite |

---

## 存放位置与边界

| 内容类型 | 存放位置 | 维护者 |
|---------|---------|--------|
| 通用写作优秀示范 | `创作标准/好的示例.md` | novel-lite / novel-review |
| 通用写作反面案例 | `创作标准/差的示例.md` | novel-lite / novel-review |
| 特定场景风格正例 | `相关设定/创作风格/场景描写/<分片>.md` | novel-review建议 → novel-setup落地 |
| 特定场景风格反例 | `相关设定/创作风格/场景描写/<分片>.md` | novel-review建议 → novel-setup落地 |
| 新硬性写作约束 | `相关设定/写作注意事项.md` | novel-setup |

---

## 条目格式规范

### 好的示例新增条目

```markdown
## [分类标签]：[具体类型]

### 示例
> 来源：第X章VX / 审查报告review_xxx / 用户反馈

[引用原文片段，≤200字]

### 为什么好
[2-3句话，指出具体效果]

### 适用场景
- 适用：[什么情况下适用]
- 不适用：[什么情况下不适用]

### 类似变体
[如有类似写法，列出]
```

### 差的示例新增条目

```markdown
## [分类标签]：[具体类型]

### 示例
> 来源：第X章VX / 审查报告review_xxx / 用户反馈

[引用问题原文或虚构反例，≤200字]

### 问题
[具体问题，指向可操作改进方向]

### 改进方向
[如何避免，给出1-2个具体写法]

### 是否已回改
- 是：已修改为...（标注版本）
- 否：待修改
```

---

## 更新流程

1. **识别**：从用户反馈或审查报告识别可沉淀内容
2. **分类**：
   - 通用写作 → `创作标准/`
   - 特定场景风格 → `相关设定/创作风格/`
3. **去重**：查阅现有条目，避免重复；如为变体，在"类似变体"补充
4. **写入**：按格式追加到目标文件底部
5. **通知**：如写入风格分片，提示novel-setup更新索引

---

## 审计与清理

每完成10条新增后，建议重组：
- 合并同类条目
- 删除已回改且不再具警示意义的差示例
- 保留原始来源和创建时间

---

## 创作标准更新评估（每次对话后）

**每次与用户讨论后执行以下评估**：

| 评估项 | 处理 |
|--------|------|
| 用户指出好的表达？ | 提取到 `好的示例.md` |
| 用户指出问题？ | 提取到 `差的示例.md` |
| novel-review发现风格问题？ | 建议写入对应分片 |
| 发现新硬性约束？ | 建议写入 `写作注意事项.md` |
| 涉及标准适用性争议？ | 在条目下追加"适用场景"或"例外说明" |

## 示例

### 好的示例示例

```markdown
## 对话节奏：追问推进信息

### 示例
> 来源：第5章V2

"'为什么帮我？'
'因为你还有用。'
'有用？'林轲盯着他，'什么意思？'
'到了你就知道。'那人转身离去。"

### 为什么好
- 对白短促（每句≤20字），节奏快
- 用追问推进信息，不一口气说完
- 结尾留钩子

### 适用场景
- 适用：信息释放场景，需要吊胃口
- 不适用：需要一次性解释大量信息的教育场景

### 类似变体
- 对白间穿插动作描写（"盯着他"、"转身离去"）
```

### 差的示例示例

```markdown
## 对话节奏：说教式长对白

### 示例
> 来源：虚构反例

"'林轲，你知道我为什么帮你吗？其实是因为我看到了你身上的潜力，虽然你现在只是练气三层，但我相信只要你努力修炼，一定能成为筑基期高手，到时候我们宗门就需要你这样的人，所以我现在帮你，也是为了宗门的未来，你要好好珍惜这个机会，不要辜负我的期望。'"

### 问题
- 单次对白>100字，读者疲劳
- 一口气说完所有信息，无悬念
- 说教语气，缺乏角色个性

### 改进方向
- 分成多句短对白，用追问推进
- 穿插角色反应描写
- 留悬念结尾

### 是否已回改
- 否：虚构反例，无需回改
```

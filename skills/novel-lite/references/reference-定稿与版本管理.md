# 定稿流程与版本管理

## 修改已有章节

修改章节时不一定需要走完整4阶段流程：

| 改动类型 | 判断标准 | 处理方式 |
|---------|---------|---------|
| 小改 | <200字，不影响设定/伏笔/衔接 | 直接改正文，同步更新摘要和实体 |
| 中改 | 200-500字，或涉及设定微调 | 重新执行阶段2（更新必看文档），然后修改 |
| 大改 | 重写≥½，或涉及设定大幅调整 | 从阶段1重新执行 |

## 版本管理

### 工作区探索版本

创作时在 `工作区/第X章VX/` 下探索不同版本：
- 目录命名：第X章V1、第X章V2、第X章V3
- 每个版本包含：正文.md、事件摘要.md、相关实体.md、自检报告.md、_预创作/、审查历史/
- 用于探索：不同叙事风格、不同内容走向、不同节奏

### 审查报告管理

审查报告由novel-review产出，固定路径：
```
工作区/第X章VX/审查历史/review_YYYYMMDD_HHMM.md
```

修订正文时，在正文.md顶部追加修订记录：
```markdown
<!--
修订记录：
- 2026-04-22 14:30 基于 review_20260422_1400.md 修订 Q1、Q3、Q5（P0）
- 2026-04-22 16:10 基于用户反馈补充 Q7 相关伏笔
-->
```

## 定稿流程（8步）

### 步骤1：定稿移入

将工作区版本移入正文目录：
- 移动：`工作区/第X章VX/` → `正文/第X章/`
- 移动内容：_预创作/、正文.md、事件摘要.md、相关实体.md、自检报告.md

### 步骤2：其他版本归档

工作区中其他版本的处理：
- 正文.md → 移入 `版本存档/第X章/`，重命名为 `正文_vN_YYYYMMDD.md`
- 其他文件（摘要、实体、自检）→ 删除（只存档正文）

### 步骤3：审查历史处理

- `审查历史/` 不移入正文目录
- 定稿版本审查历史归档至 `版本存档/第X章/审查历史_定稿YYYYMMDD/`
- 正文目录不保留审查历史

### 步骤4：设定更新

根据定稿版本的相关实体.md更新：
- `相关设定/角色/*.md`：补出场记录
- `相关设定/物品/*.md`：补使用记录
- `相关设定/技能/*.md`：补使用记录和代价交代

### 步骤5：待定设定处置

列出本章关联的所有待定项，逐条决策：

| 状态 | 处理 |
|------|------|
| 🟢 拟晋升 | 移入 `相关设定/<类别>/`，删除待定文件，更新索引 |
| 🟠 待审 | 保留在池，索引备注下次必须处理 |
| ⚫ 拟废弃 | 删除文件，检查正文是否残留暗示，必要时回改 |
| 🟡 草案 | **必须升级**为🟠或处置，禁止以草案状态进入下一章 |

**涉及世界规则/伏笔/大纲的晋升**：标注"由novel-setup接手"

### 步骤6：状态回写（触发novel-setup）

通知novel-setup执行：
- 更新 `故事状态.md`：当前章、主角状态、下一章定位
- 更新 `当前冲突.md`：勾选节拍，判断演化
- 更新 `伏笔追踪.md`：新埋/回收/强化
- 更新 `主角档案.md`：弧光推进
- 接手宏观待定项晋升
- 采样风格指纹：写入 `创作风格/风格指纹/第X章.md`

### 步骤7：阶段回顾触发判断

计算定稿章号N，若 `N % 5 == 0`：
- 提示用户："已完成第N章定稿，建议跑novel-review阶段回顾"
- 用户同意 → 调用novel-review模式D
- 用户跳过 → 在 `相关设定/_阶段回顾历史/skipped.md` 登记

累积跳过≥2次（10章未做）时升级提示语调。

### 步骤8：清理工作区

删除 `工作区/第X章/` 下所有版本目录。
**不动 `工作区/待定设定/`**（项目级共享池）。

## 定稿前自检清单

移入正文前逐项确认：

- [ ] 自检报告总分≥85（或用户确认接受70~85）
- [ ] 自检报告无铁律违反
- [ ] 至少跑过一轮novel-review审查循环
- [ ] P0级问题全部处理（保留需登记理由）
- [ ] 主角知晓状态无违规
- [ ] 实体设定已同步更新
- [ ] 待定设定已逐条处置，无草案残留
- [ ] 待定设定索引已同步
- [ ] 至少明确1个伏笔动作（新埋/回收/强化）
- [ ] 章末有钩子或情绪落点

## 伏笔登记与回收

写作中发现伏笔动作，**立即登记到本章 `相关实体.md` 底部**：

```markdown
## 本章伏笔动作

- 新埋 F00X：[简称]
  - 位置：[段落描述]
  - 埋设方式：[直述/暗示/道具/对白]
  - 计划回收：第Y章左右

- 回收 F00X：[简称]
  - 回收方式：[简述]

- 强化 F00X：[简称]
  - 强化内容：[简述]
```

定稿时由状态回写流程同步到 `伏笔追踪.md`。

## 大纲偏离处理

定稿时若发现与大纲偏离：
1. 提示用户偏离情况
2. 给出两个选项：
   - 更新大纲（偏离合理）
   - 回改章节（偏离不合理）
3. 用户决策后执行相应操作

# 创作方向模板（阶段1）

基于开写简报，提出**3个不同创作方向**供用户选择。

## 处理要点

分析简报中的：
- 本章在大纲中的定位与预期事件
- 当前冲突状态与本章应推进的节拍
- 上一章结尾（衔接点）
- 伏笔动作建议（待回收/待强化/新埋）
- 风险提示

每个方向在以下维度中的**至少2个**形成差异：
- 叙事节奏（快/慢/紧/弛）
- 冲突强度（高压/渐升/反复拉锯）
- 信息释放方式（逐步揭示/集中爆发/侧面暗示）
- 视角选择（内聚焦/外聚焦/交替）
- 情绪基调（压抑/爆发/悬疑/温情）
- 伏笔侧重（回收优先/新埋优先/强化优先）

> 第1章无"上一章衔接"维度，替换为"故事起点切入方式"（如：直接进入冲突 / 先铺氛围再引入主角 / 以悬念钩子开场）。衔接分析降级为承接 `主角档案.md` 起点状态 + `故事状态.md` 初始快照。

## 产出模板

```markdown
# 第N章 创作方向

> 生成时间：YYYY-MM-DD HH:MM
> 基于：00_开写简报.md

## 方向A：[一句话概括]
- 叙事节奏：[描述]
- 冲突强度：[描述]
- 情绪基调：[描述]
- 信息释放：[描述]
- 伏笔侧重：[回收F00X / 新埋F00Y / 强化F00Z]
- 预计场景类型：[战斗/对话/心理/探索...]
- 与上一章衔接：[如何衔接]
- 风险与注意事项：[可能的问题]

## 方向B：[一句话概括]
（同上结构）

## 方向C：[一句话概括]
（同上结构）

## 推荐：方向[X]
- 原因：[基于简报中的冲突推进要求、大纲贴合度、伏笔回收窗口等给出理由]

## 用户决策
- 选择：[A/B/C/混合/自定义]
- 修改意见：[如有]
- 决策时间：YYYY-MM-DD HH:MM
```

> 用户选择后，在文件底部追加决策记录。后续阶段基于选定方向执行。

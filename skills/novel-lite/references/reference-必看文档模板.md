# 必看文档模板（阶段2）

基于选定的创作方向，**收集本章节创作所需的全部资料**，浓缩为一份带自测逻辑的精炼文档。

**这是整个4阶段流程中最关键的环节**。必看文档是阶段3创作的**唯一依据**——创作正文时不再加载分散的设定文件，只参照这份文档。

## 输入清单

- `工作区/第X章VX/_预创作/00_开写简报.md`
- `工作区/第X章VX/_预创作/01_创作方向.md`
- 以下设定文件（**按需加载**，仅加载本章涉及的）：
  - `相关设定/角色/*.md` — 本章出场角色
  - `相关设定/物品/*.md` — 本章涉及物品
  - `相关设定/技能/*.md` — 本章涉及技能
  - `工作区/待定设定/_索引.md` — 待定设定总览
  - 本章关联的待定设定文件
  - `正文/第N-1章/` — 上一章定稿三件套（用于衔接）
  - `正文/第N-1章/_预创作/02_必看文档.md` — 上一章的必看文档（可选参考）
  - `相关设定/创作风格/场景描写/<场景>.md` — 本章预计场景对应的风格分片
  - `相关设定/创作风格/_总纲.md`
  - `相关设定/创作风格/自检清单.md`
  - `相关设定/创作风格/风格指纹/第N-1章.md`

## 处理步骤

1. **提取设定约束**：从角色/物品/技能设定中提取本章必须遵循的约束（主角知晓状态、能力边界、关系动态等）
2. **提取风格要求**：从创作风格分片中提取本章涉及的场景类型对应的"必做/必避/量化指标"
   - 若发现某场景类型分片缺失或过于单薄：提示用户"是否调用 novel-style-extract 从参考小说补充该场景分片？"
   - 若用户选择不补充 → 在必看文档中标注"[场景X]分片待补充，创作时暂按通用规范处理"
3. **提取衔接要点**：从上一章结尾和当前冲突中提取本章必须承接的要素
4. **构建自测逻辑**：将风格要求和设定约束转化为**可执行的自测项**，每项都有明确的判定标准

## 三、衔接要点（必须承接）

> 第1章特殊处理：无"上一章结尾状态"。衔接要点改为承接 `故事状态.md` 的初始快照与 `主角档案.md` 的起点状态，并响应 `开写简报.md` 中的"本章在大纲中的定位"。3.1 和 3.2 中涉及"上一章"的项标注为"不适用"。

### 3.1 上一章结尾状态
> 摘自第N-1章正文末尾 + 事件摘要。第1章无上一章时，替换为"故事起点状态"。

- 主角状态：[位置/情绪/伤情/携带物]
- 悬念/钩子：[上一章末留了什么未决 / 第1章则为开章钩子设计]
- 在场人物：[谁在主角身边 / 第1章则为预计开场出场人物]
- 时间/环境：[时间流逝/地点/天气 / 第1章则为故事初始时空]

### 3.2 本章必须承接的要素
- [ ] 冲突节拍：[简报中标注的下一节拍 / 第1章则为首段冲突引入]
- [ ] 伏笔动作：[待回收F00X / 待强化F00Y / 新埋F00Z / 第1章可埋设开章伏笔]
- [ ] 弧光推进：[主角本章的微小变化 / 第1章则为弧光起点展示]
- [ ] 上一章钩子：[如何响应 / 第1章则为开章钩子设计]

**自测**：正文开头是否自然衔接了上一章结尾（第1章则检查是否有效建立起点状态）？冲突节拍是否被推进？

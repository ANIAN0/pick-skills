## 设定文件

每个实体单独一个md文件，只存核心信息：

```markdown
# [实体名称]

## 主角知晓状态
- [主角已知/主角未知]
- [知晓方式: 直接经历/他人告知/传闻/未知]
- [当前认知: 完全了解/部分了解/误解/毫无认知]

## 基本信息
[关键属性]

## 核心特征
[最重要的特征]

## 相关关系
[与其他实体的关联]
```

### 主角知晓状态说明

**核心原则**: 读者与主角同步探索，禁止"读者知道但主角不知道"的信息不对称。

- **主角已知**: 主角已通过剧情直接获知的信息，可以在正文直接展示
- **主角未知**: 主角尚未了解的信息，只能通过其他角色视角或暗示呈现，不能让读者先于主角"知道真相"

**知晓方式分类**:
- 直接经历: 主角亲身经历的事件/对话
- 他人告知: 其他角色告诉主角的信息
- 传闻: 主角听到的传闻/传说（可能真假混杂）
- 未知: 主角完全不了解

**当前认知**:
- 完全了解: 主角掌握了完整真相
- 部分了解: 主角只知道一部分，存在认知缺口
- 误解: 主角的理解与真相有偏差（适合制造戏剧冲突）
- 毫无认知: 主角对此完全无知

## 章节文件

- **事件摘要.md**：本章主要事件（3-5条）
- **正文.md**：章节内容
- **相关实体.md**：本章涉及的角色、物品、技能及其行为/变化

## 创作流程

流程不固定，灵活处理：

- 从摘要到正文：用户给摘要 → 生成正文 → 提取实体
- 从正文到摘要：用户给正文 → 提取摘要 → 提取实体
- 修改章节：修改正文 → 更新摘要和实体

**主角视角约束**: 写正文前，先检查相关设定的"主角知晓状态"。正文只能展示主角已知或正在探索的信息。主角未知的真相只能通过暗示、悬念或他人视角呈现。

## 三选项机制

每次修改前必须给出3个不同选项：

```
**选项A**: [描述] - 特点: [差异点]
**选项B**: [描述] - 特点: [差异点]
**选项C**: [描述] - 特点: [差异点]

**推荐**: 选项[X] - 原因: [理由]
```

## 版本管理

### 工作区探索

创作时在**工作区/**目录下探索不同版本：
- 目录命名：第X章VX（如第1章V1、第1章V2）
- 每个版本目录包含完整的三文件：事件摘要.md、正文.md、相关实体.md
- 用于探索不同叙事风格、不同内容走向、不同节奏把控

### 定稿流程

用户确定定稿版本后：

1. **定稿版本处理**
   - 将工作区/第X章VX/目录下的三文件移动到正文/第X章/

2. **其他版本归档**
   - 工作区/中其他版本的正文.md移动到版本存档/第X章/
   - 重命名为正文_v1.md、正文_v2.md...
   - 事件摘要.md和相关实体.md删除（只存档正文）

3. **设定更新**
   - 根据定稿版本的相关实体.md更新相关设定/

4. **清理工作区**
   - 删除工作区/第X章/下所有版本目录

## 创作标准

### 标准文件结构

**好的示例.md**：
```markdown
# 好的示例积累

## 对话描写
- 示例: [引用正文中好的对话]
- 原因: [为什么好]
- 适用场景: [什么情况下适用]

## 情绪渲染
- 示例: [引用正文中好的情绪描写]
- 原因: [为什么好]
- 适用场景: [什么情况下适用]
```

**差的示例.md**：
```markdown
# 需避免的示例

## 信息堆砌
- 示例: [引用需要改进的内容]
- 问题: [具体问题是什么]
- 改进方向: [如何避免]

## 视角混乱
- 示例: [引用视角问题内容]
- 问题: [具体问题是什么]
- 改进方向: [如何避免]
```

### 标准积累方式

在创作和讨论过程中持续完善：
- 用户指出好的内容 → 提取并记录到好的示例.md
- 用户指出问题 → 提取并记录到差的示例.md
- 每次创作前查阅创作标准，参考历史积累
- 每次讨论后更新创作标准，沉淀经验

## 设定更新检查

每次对话后思考：

1. **内容更新**: 用户反馈中有无可更新的设定？如有，主动提出并修改
2. **知晓状态更新**: 主角在本次剧情中是否获得了新的认知？
   - 若主角新获知某设定信息，需将"主角未知"更新为"主角已知"
   - 同时记录知晓方式（经历/告知/传闻）和当前认知程度
3. **信息对称检查**: 是否存在读者知道但主角不知道的情况？
   - 若发现，立即标记为"主角未知"并调整呈现方式
   - 禁止在正文直接展示主角未知的核心真相

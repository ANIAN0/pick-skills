# 无人值守研发 v3

编排最小有效子图。不得替代阶段 Skill、确定性校验器或人工决策。

## 必读内容

- 读取 [入口与恢复](references/entry-and-resume.md)，选择入口及恢复范围。
- 读取 [编排契约](references/orchestration.md)，了解阶段责任、顺序、暂停、证据和完成门禁。
- 只加载当前阶段 Skill 和当前节点的直接必要上下文。

## 入口模式

- `full`：从用户故事开始，按需经过需求、调研/UI、设计、计划、核心执行、验证、支撑执行、E2E/回归、评审和发布门禁。
- `extension`：从既有已确认节点开始，计算影响，只处理新增或受影响分支。
- `task`：从变更契约、验证、依赖和执行者门禁均有效的选定任务开始。
- `bug`：从根因或缺陷任务开始，只执行复现、修复契约/任务、验证、回归和验收。

## 工作流

1. 校验图谱并选择入口模式和节点。
2. 创建 Workflow Run，记录入口、当前节点和状态。
3. 调用节点责任 Skill；校验、影响和派生产物使用公共脚本。
4. 每次节点转换后保存已完成节点 ID 和证据引用，再选择下一个通过门禁的节点。
5. 遇到权限、阻塞、stale、执行者或真实验证失败时暂停，并记录精确恢复输入。
6. 阻塞解决后从 `resume_node` 和 Finding 可执行影响集恢复。
7. 只有范围内验收、E2E/回归及最终评审均通过才声明完成。

## 硬规则

- 高风险产品、安全、兼容、破坏性或发布决策需要人工授权时必须停止。
- 没有 senior 或明确审批时停止核心/基础设施执行。
- 遇到 BLOCKER、无效图、stale 确认、缺失真实依赖、验证失败或发布证据 not_verified 时停止。
- 保留可执行影响集之外的有效确认和 done 证据。
- 不批准风险、不合成通过证据、不隐藏失败、不预加载所有阶段 Skill，也不把 not_verified 当作完成。

## 输出

报告 Workflow Run ID/模式、当前及恢复节点、完成与阻塞范围、暂停原因、Finding 影响输入、证据、下一责任 Skill 和发布门禁状态。

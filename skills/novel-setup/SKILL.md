# 小说项目启动与全局管理

## 定位

四件套分工：
- **novel-setup（本skill）**：宏观层。管"项目从0到1"、"长篇走向不失控"、"创作风格知识库"、"开写简报"。
- **novel-lite**：中观层。管"单章怎么写、怎么定稿"（4阶段流程：简报→方向→必看文档→创作+自测）。
- **novel-review**：中观层。管"单章怎么审、怎么循环"。
- **novel-style-extract**：风格层。从参考小说中提取创作风格。

本 skill 不写正文，不审正文。只负责**让 novel-lite 每次开写新章节前，都有一张清晰的地图、一套场景化的风格知识库、和一份可执行的开写简报**。

## 工作流程场景

### 场景1：全新项目冷启动

采用**访谈式引导**，分 5 轮完成：故事灵魂 → 世界观 → 主角 → 大纲骨架 → 写作约束+创作风格知识库初始化。

**详细流程**：见 `references/reference-冷启动流程.md`

### 场景2：每章开写前的"开写简报"（阶段0）⭐

novel-lite 启动新章时，**第一步**请求本 skill 出具「开写简报」，写入 `工作区/第X章VX/_预创作/00_开写简报.md`。

简报包含：当前进展回看、本章大纲定位、当前冲突状态、故事状态、伏笔动作建议、主角弧光节点、约束提醒、创作风格分片建议、关联实体清单、风险提示。

**详细模板**：见 `references/reference-开写简报模板.md`

### 场景3：章节定稿后的"状态回写"⭐

novel-lite 定稿后调用本 skill：
1. 更新 `故事状态.md`
2. 更新 `当前冲突.md` ⭐（勾选节拍，处理演化）
3. 更新 `伏笔追踪.md`
4. 更新 `主角档案.md`（弧光推进）
5. **采样风格指纹** ⭐：从自检报告抽取本章风格指标，写入 `创作风格/风格指纹/第X章.md`
6. 检查大纲贴合度
7. 接手待定项晋升（涉及宏观的）
8. 阶段回顾触发判断（N%5==0）

### 场景4：定期全局体检

用户主动请求时，产出「全局体检报告」：大纲进度、主线推进、伏笔健康度、主角弧光、待定设定池规模、风格指纹漂移趋势、风险项汇总。

### 场景5：大纲调整

出示当前大纲与已写章节清单 → 评估改动影响面 → 给出激进/温和/最小改动 3 种方案 → 用户选定后更新并标记受影响章节与伏笔。

### 场景6：创作风格知识库的演化 ⭐

- **新增分片**：novel-lite 发现要写 X 类场景但分片缺失 → 本 skill 创建分片骨架（目标+必做+必避）→ 如有参考小说，调用 **novel-style-extract** 增量提取填充正例/反例/量化指标
- **调整自检清单阈值**：novel-review 阶段回顾发现"自检80分但明显风格漂移" → 调松；连续3章自检<70 → 调严或重新校准
- **沉淀正反例**：novel-review 发现的风格类优秀片段/问题 → 按归属决定存放位置（特定场景分片 或 通用创作标准）

**详细模板与迭代规则**：见 `references/reference-创作风格知识库.md`

## 与 novel-lite 的交接协议

| 时机 | novel-setup 动作 | novel-lite 动作 |
|------|-----------------|----------------|
| 项目启动 | 生成全部宏观设定文件 + 创作风格知识库最小可用集 | 创建工作区/正文/等目录 |
| 开写第N章 阶段0 | 出具「开写简报」并写入文件 | 读取简报，进入阶段1 |
| 阶段2（必看文档） | 应 lite 请求，提供风格分片细节、设定切片补充 | 按简报指引加载相关文件，汇总成 `02_必看文档.md` |
| 第N章定稿 | 执行「状态回写」+ 风格指纹采样 + 接手宏观待定项 | 将工作区VX移入正文/第N章/ |
| 大纲变更 | 更新大纲+评估影响 | 按新大纲重新规划后续章节 |
| 风格知识库演化 | 新增/修订分片、调整自检阈值 | 反馈实战中的风格命中率 |
| 新场景发现 | 创建分片骨架 | 在阶段2发现新场景类型时提示用户 |

## 与 novel-style-extract 的交接协议

| 时机 | novel-setup 动作 | novel-style-extract 动作 |
|------|-----------------|-------------------------|
| 项目冷启动 Round 5 | 引导用户提供参考小说，调用本 skill | 全量提取，生成 `_总纲.md` + `场景描写/*.md` + `自检清单.md` |
| 创作中发现分片缺失 | 创建分片骨架（目录+文件） | 增量提取，填充指定场景分片内容 |
| review 发现风格问题 | 评估是否需要调整知识库结构 | 重新提取/修正对应分片 |
| 用户主动要求参考新小说 | 评估覆盖/合并策略 | 全量或增量提取 |

**关键分工**：
- novel-setup 负责 `相关设定/创作风格/` 目录创建、`_索引.md` 维护、知识库版本控制
- novel-style-extract 负责分片文件的**内容生成**
- novel-style-extract 生成内容后**不直接写入**，先提交用户确认

## 与 novel-review 的交接协议

- novel-review 审查时读取宏观文件作为依据
- novel-review 模式 B 中标注"由 novel-setup 接手"的待定项 → 本 skill 执行晋升
- novel-review 模式 C 中"由 novel-setup 接手"的整改项 → 本 skill 修订
- novel-review 模式 D（阶段回顾）⭐ 中"由 novel-setup 接手"的项（含风格指纹漂移、自检阈值调整、新场景分片需求等）→ 本 skill 落地

## 边界

- 不写正文（交给 novel-lite）
- 不做章节级审查（交给 novel-review）
- 不主动发明大纲之外的剧情
- 状态回写时，如与既有设定冲突，先报告后更新
- 不直接读写 `工作区/待定设定/` 的文件结构（由 novel-lite 维护）
- 接手晋升前必须确认待定项已经过 novel-review 模式 B 审查
- **不擅自改动创作风格知识库的"铁律"和"自检阈值"**——调整必须经用户确认
- **开写简报与状态回写之外，不主动改 `工作区/` 下任何文件**
- **不直接生成 `02_必看文档.md`**——由 novel-lite 在阶段2汇总生成

## 创作标准更新评估

**每次与用户讨论后，无论本次讨论主题为何，必须执行以下评估：**

1. **用户是否给出了跨场景通用写作的优秀示范？** → 提取到 `创作标准/好的示例.md`
2. **用户是否指出了跨场景通用写作的问题？** → 提取到 `创作标准/差的示例.md`
3. **本次讨论是否浮现了新的硬性写作约束？** → 更新 `相关设定/写作注意事项.md`
4. **novel-review 建议的风格分片调整是否在本次讨论中被确认？** → 更新 `相关设定/创作风格/` 对应分片或 `_总纲.md`
5. **本次讨论是否涉及创作标准本身的适用性争议？** → 在对应标准条目下追加"适用场景"或"例外说明"

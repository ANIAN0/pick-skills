# 小说项目宏观管理

## 四件套定位

| Skill | 层级 | 职责 |
|-------|------|------|
| **novel-setup（本skill）** | 宏观 | 项目从0到1、大纲走向、故事状态、当前冲突、伏笔追踪、创作风格知识库维护、开写简报 |
| **novel-lite** | 中观（写） | 单章创作：4阶段流程（简报→方向→必看文档→创作+自测） |
| **novel-review** | 中观（审） | 单章审查：正文审查、新设定审查、设定体检、阶段回顾 |
| **novel-style-extract** | 风格 | 从参考小说提取抽象化的创作风格规则，生成知识库文件 |

> 其他skill在此统一查看定位分工。

## 核心场景

### 场景1：全新项目冷启动

采用**访谈式引导**，5轮完成：故事灵魂 → 世界观 → 主角 → 大纲骨架 → 写作约束+创作风格知识库初始化。

详细流程见 `references/reference-冷启动流程.md`。

**冷启动产出**：
- `故事大纲.md`、`世界设定.md`、`主角档案.md`、`写作注意事项.md`
- 初始化的 `故事状态.md`、`当前冲突.md`、空白的 `伏笔追踪.md`
- 创作风格知识库最小可用集（方式一：手动构建；方式二：调用novel-style-extract从参考小说提取）

### 场景2：每章开写前的"开写简报"（阶段0）

**触发**：novel-lite启动新章时，第一步请求本skill出具简报。

**产出**：`工作区/第X章VX/_预创作/00_开写简报.md`

简报包含：
- 当前进展回看
- 本章在大纲中的定位
- 当前冲突状态与应推进的节拍
- 故事状态摘要
- 伏笔动作建议（待回收/待强化/新埋）
- 主角弧光节点
- 约束提醒（视角铁律、信息对称、字数目标）
- 创作风格分片建议（阶段2需加载哪些场景分片）
- 关联实体清单
- 风险提示

详细模板见 `references/reference-开写简报模板.md`。

### 场景3：章节定稿后的"状态回写"

**触发**：novel-lite定稿后调用本skill。

**执行顺序**：
1. 更新 `故事状态.md`：时间地点、主角当前状态、活跃支线、活跃人物
2. 更新 `当前冲突.md`：勾选已推进节拍，判断是否触发演化（达成/失败/转化），必要时归档并建立下一段冲突
3. 更新 `伏笔追踪.md`：标记已回收、登记新埋
4. 更新 `主角档案.md`：弧光节点推进、当前状态参数
5. **采样风格指纹**：从自检报告抽取本章风格指标，写入 `创作风格/风格指纹/第X章.md`
6. 检查大纲贴合度：偏离是否需要更新大纲
7. 接手宏观待定项晋升（世界规则/伏笔/大纲类）
8. 判断是否触发阶段回顾（N%5==0 → 提示用户跑novel-review模式D）

### 场景4：创作风格知识库演化

**新增分片**：novel-lite发现要写X场景但分片缺失 → 本skill创建骨架 → 如有参考小说，调用novel-style-extract增量提取

**调整自检清单**：novel-review阶段回顾发现"自检80分但风格漂移" → 调整阈值（调松或调严）

**沉淀正反例**：novel-review发现风格类问题 → 按归属写入对应分片

详细规则见 `references/reference-创作风格知识库.md`。

### 场景5：大纲调整

**流程**：
1. 出示当前大纲与已写章节清单
2. 评估改动影响面（多少章受影响、多少伏笔需调整）
3. 给出三种方案：激进改动、温和调整、最小修正
4. 用户选定后更新大纲
5. 标记受影响章节与伏笔，通知novel-lite后续调整

### 场景6：定期全局体检

**触发**：用户主动请求或阶段回顾后需要宏观整改。

**产出**：「全局体检报告」：
- 大纲进度 vs 实际进度
- 主线推进健康度
- 伏笔健康度（超期未回收、漏登）
- 主角弧光节奏
- 待定设定池规模
- 风格指纹漂移趋势
- 风险项汇总

## 待定设定接手协议

待定设定的晋升执行按归属分工：

| 归属 | 目标文件 | 执行者 |
|------|---------|--------|
| 角色（普通配角/路人）| `相关设定/角色/` | novel-lite |
| 物品 | `相关设定/物品/` | novel-lite |
| 技能 | `相关设定/技能/` | novel-lite |
| **世界底层规则** | `相关设定/世界设定.md` | **本skill** |
| **写作硬约束** | `相关设定/写作注意事项.md` | **本skill** |
| **新风格规则/分片** | `相关设定/创作风格/` | **本skill** |
| **新伏笔登记** | `相关设定/伏笔追踪.md` | **本skill** |
| **大纲调整** | `相关设定/故事大纲.md` | **本skill** |
| **主角弧光节点** | `相关设定/主角档案.md` | **本skill** |

**前置条件**：接手晋升前，待定项必须已通过novel-review模式B审查。

## 与其他skill的交接

### 与novel-lite

| 时机 | 本skill动作 | novel-lite动作 |
|------|------------|----------------|
| 项目启动 | 生成宏观设定+知识库最小集 | 创建工作区目录结构 |
| 开写第N章 阶段0 | 出具开写简报并写入 | 读取简报，进入阶段1 |
| 阶段2（必看文档）| 提供风格分片细节、设定切片补充 | 按简报汇总成02_必看文档.md |
| 第N章定稿 | 执行状态回写+风格指纹采样 | 移入正文目录 |
| 大纲变更 | 更新大纲+标记影响 | 按新大纲调整后续章节 |

### 与novel-style-extract

| 时机 | 本skill动作 | novel-style-extract动作 |
|------|------------|------------------------|
| 冷启动Round5 | 引导用户提供参考小说，调用 | 全量提取抽象化风格规则 |
| 创作中发现分片缺失 | 创建分片骨架 | 增量提取指定场景分片内容 |
| review发现风格问题 | 评估是否调整知识库结构 | 重新提取/修正对应分片 |

**关键分工**：
- 本skill负责目录创建、索引维护、版本控制
- novel-style-extract负责**内容生成**（且必须是抽象化的风格规则）
- novel-style-extract生成内容后不直接写入，先提交用户确认

### 与novel-review

- novel-review审查时读取宏观文件作为依据
- novel-review模式B标注"由本skill接手"的待定项 → 本skill执行晋升
- novel-review模式C标注"由本skill接手"的整改项 → 本skill修订
- novel-review模式D中"由本skill接手"的项（风格指纹漂移、自检阈值调整等）→ 本skill落地

## 边界

- 不写正文（交给novel-lite）
- 不审章节（交给novel-review）
- 不主动发明大纲之外的剧情
- 状态回写时若与既有设定冲突，先报告后更新
- 不直接读写`工作区/待定设定/`的文件结构（由novel-lite维护）
- 接手晋升前必须确认待定项已通过审查
- 不擅自改动创作风格知识库的铁律和自检阈值——调整必须经用户确认
- 开写简报与状态回写之外，不主动改`工作区/`下任何文件
- 不直接生成`02_必看文档.md`——由novel-lite在阶段2汇总生成

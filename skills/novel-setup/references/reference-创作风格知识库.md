# 创作风格知识库（创作风格/）⭐

`创作风格/` 是一套**按场景分片、含示例、含可执行自检**的知识库：
- 写正文前按当前场景**按需加载**对应分片，省 token
- 每个分片都有"反面示例"和"可量化的合格条件"
- 自检清单是写完后必跑的最后一道闸口

## 知识库结构

```
相关设定/创作风格/
├── _总纲.md            # 必读，约 200~400 字，写文风灵魂、3-5 条核心铁律
├── _索引.md            # 知识库地图，按场景/技巧分类，便于按需加载
├── 场景描写/
│   ├── 战斗.md
│   ├── 对话.md
│   ├── 环境.md
│   ├── 心理.md
│   ├── 动作.md
│   ├── 情绪.md
│   ├── 信息释放.md
│   └── 过渡与节奏.md
├── 自检清单.md         # 可量化的写作风格自检
└── 风格指纹/
    └── 第X章.md        # 已定稿章节抽样的风格特征
```

## _总纲.md 模板

```markdown
# 创作风格总纲

## 文风灵魂（一句话）
[这本书读起来应该是什么感觉？]

## 核心铁律（3-5 条，必须做到）
1. [铁律1]
2. [铁律2]
3. [铁律3]
4. ...

## 参考坐标
- 形似但禁止：[作品/作者]（怕被误读为模仿）
- 神似目标：[作品/作者]（追求的境界）

## 词汇基调
- 偏好词族：[冷色调动词、克制副词、具体名词]
- 慎用词族：[滥情词、网络梗、过度修辞]
- 禁用词清单：[具体词汇，0容忍]

## 句式偏好
- 长短句比例倾向：[如长短交替，避免连续3句以上长句]
- 标点偏好：[如多用句号少用逗号]

## 何时跳到分片
- 写战斗 → 同时加载 `场景描写/战斗.md` + `场景描写/动作.md`
- 写对话 → `场景描写/对话.md` + 涉及内心独白时加 `心理.md`
- 写场景切换 → `场景描写/过渡与节奏.md`
- 写揭谜/悬念 → `场景描写/信息释放.md`
- 不论写什么，定稿前都要跑 `自检清单.md`
```

## 场景描写分片模板（以"战斗.md"为例）

每个分片都遵循统一结构：**目标 → 必做 → 必避 → 量化指标 → 正例 → 反例 → 触发场景外延**。

```markdown
# 场景描写：战斗

## 目标
[本类场景的写作目的]

## 必做（命中即合格）
- [ ] 战斗起手必交代主角的"当前状态参数"（体力/伤情/装备/认知盲点至少 2 项）
- [ ] 每次主角施展能力前，必有一次"成本/代价/限制"的提示
- [ ] 至少 1 次主角的判断失误 / 信息不足 / 临场犹豫
- [ ] 战斗节奏：进攻 → 反制 → 失衡 → 重建 → 终结，避免一路压制
- [ ] 收尾必交代"代价"，不能毫发无损

## 必避（命中即扣分）
- ❌ 全知视角（旁白说"敌人此时心想……"）
- ❌ 主角连续顺利攻击 ≥3 段无反制
- ❌ "想也不想就出手"——除非已建立人设
- ❌ 仅靠形容词堆砌氛围，无具体感官细节
- ❌ 数值膨胀比拼（A 等级 vs B 等级，没有具体动作）

## 量化指标（自检时打分用）
| 项目 | 不合格 | 合格 | 优秀 |
|------|--------|------|------|
| 感官覆盖（视觉/听觉/触觉/痛觉至少 3 类） | <2 类 | 3 类 | 4 类 + 嗅味觉 |
| 反制节拍数（敌方有效反制段落） | 0 | 1-2 | ≥3 |
| 主角失误/犹豫次数 | 0 | 1 | ≥2 |
| 代价具体度 | 抽象描述 | 1 处具体 | 多处可视化代价 |
| 句式节奏（短句占比，紧张段落） | <30% | 30%-60% | 紧张时短句>50% |

## 正面示例
> [引用本项目已定稿章节中的优秀片段]

**为什么好**：[结合"必做""量化指标"逐项点评]

## 反面示例
> [引用差示例或虚构反例]

**问题**：
- 违反必做第 X 条
- 量化指标项 Y 不达标

## 触发场景外延
- 同时务必读：`场景描写/动作.md`、`场景描写/情绪.md`
- 涉及主角能力使用 → 同时核对 `相关设定/技能/<能力名>.md` 的代价/边界
```

## 自检清单.md 模板

自检清单是**写完正文后必须逐项打勾**的可量化检查。novel-lite 在阶段3创作完成后必须跑一次本清单，把分数和不达标项写进章节工作区的 `自检报告.md`。

```markdown
# 写作风格自检清单

> 用法：每写完一章正文（或一次重大修改后），逐项检查并打分。
> 总分 ≥85 / 100 视为风格合格；<70 必须返工；70~85 视为可改进。
> 输出位置：`工作区/第X章VX/自检报告.md`

## 一、铁律检查（每条违反 -10 分，0 容忍）

- [ ] 总纲铁律1：[填入项目实际铁律]
- [ ] 总纲铁律2：...
- [ ] 视角铁律：未出现主角视角不可能感知到的描写
- [ ] 信息对称铁律：未透露主角未知的真相
- [ ] 禁用词清单：0 命中

## 二、场景执行检查（按本章涉及的场景类型，每类满分 10）

### 战斗场景（若有）
- [ ] 必做项 5 条全部命中（每漏 1 项扣 2 分）
- [ ] 量化指标：感官覆盖 ___ 类、反制节拍 ___ 次、主角失误 ___ 次、代价具体度 ___
- 实际得分：___ / 10

### 对话场景（若有）
（同上结构）

### 心理场景（若有）
（同上）

…（按本章实际涉及场景填写）

| 指标 | 上一章 | 本章 | 偏差 | 是否健康 |
|------|--------|------|------|---------|
| 平均段长（句） | | | | |
| 长句占比 | | | | |
| 对白占比 | | | | |
| 内心独白占比 | | | | |
| 感官词频次/千字 | | | | |
| 比喻/修辞密度 | | | | |
| 标点偏好（句号/逗号比） | | | | |

偏差 >30% 的指标需在自检报告中说明原因（场景差异 vs 风格漂移）。

## 四、词汇与句式（10 分）

- [ ] 偏好词族出现频次正常
- [ ] 慎用词族未滥用
- [ ] 禁用词 0 命中
- [ ] 句式分布符合总纲偏好
- [ ] 段落节奏符合场景需要

## 五、信息密度与节奏（10 分）

- [ ] 每段都在推进信息（剧情/设定/人物/伏笔/情绪 至少 1 项）
- [ ] 无明显"水段"（连续超过 80 字纯氛围描写）
- [ ] 章末有钩子或情绪落点
- [ ] 节奏曲线合理（紧→缓→紧 或 缓→紧→收）
```

## 知识库的迭代规则

- 项目启动时，引导用户产出**最小可用集**（_总纲 + 场景描写/3 篇核心场景 + 自检清单骨架）
- 创作过程中遇到"新场景但分片缺失" → novel-lite 提示创建新分片，由 novel-setup 接手撰写骨架
- novel-review 模式 A 发现的**风格类**问题 → 沉淀回对应分片的"反例"段，并视情况新增量化指标
- novel-review 模式 D（阶段回顾）每 5 章评估一次"风格指纹漂移"，必要时调整自检清单阈值

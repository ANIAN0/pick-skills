# 创作风格知识库

`创作风格/` 是一套按场景分片、含示例、含可执行自检的知识库。写正文前按当前场景按需加载对应分片，省token。

---

## ⚠️ 核心原则：抽象化处理

**风格规则必须是抽象的、可迁移的，不能包含原书特定内容。**

### 可提取的抽象规则（✅）

这些是"写作技法"，适用于任何题材：
- 战斗场景要有代价交代
- 主角要有失误，不能一路压制
- 信息释放要吊胃口，不能一口气说完
- 对话用短句推进，避免说教式长对白
- 描写要有感官细节，不能只堆形容词
- 每段至少推进一项信息（剧情/设定/人物/伏笔/情绪）

### 题材绑定规则（⚠️需标注）

这些与特定世界观绑定，用户需确认是否适用：
- 武侠：招式名称要与招式效果挂钩（仅武侠适用）
- 科幻：技术名词要有解释铺垫（仅科幻适用）
- 悬疑：线索要有多重解读空间（仅悬疑适用）

**标注方式**：在分片文件中用 `⚠️ [题材绑定：武侠/科幻/悬疑]` 标注

### 不可提取的内容（❌）

这些是原书的设定，新书不可用：
- ❌ "主角使用剑法时要描写招式名称"（这是原书技能设定）
- ❌ "打坐修炼的描写方式"（这是原书世界观规则）
- ❌ "宗门考核的具体流程"（这是原书情节内容）
- ❌ "某角色的性格特征描写方式"（这是原书人物设定）

```markdown
> ⚠️ 这些铁律必须是抽象的，不包含具体世界观设定

2. [铁律2，如：代价铁律——每次主角使用能力/做出决定，必须有代价或风险]
3. [铁律3，如：失误铁律——主角不能一路顺利，至少每章一次判断失误或信息不足]
4. [铁律4，如：信息密度——每段至少推进一项信息，无"水段"]
5. [铁律5，如：对白克制——单次对白不超过100字，禁止说教]

## 参考坐标
- 形似但禁止：[作品/作者，如：不要写成某种特定风格]
- 神似目标：[追求的境界]

## 词汇基调
- 偏好词族：[类型，如：冷色调动词、克制副词、具体名词]
- 慎用词族：[类型，如：滥情词、网络梗、过度修辞]
- 禁用词清单：[具体词汇，0容忍]

## 句式偏好
- 长短句比例：[倾向，如：长短交替，避免连续3句以上长句]
- 标点偏好：[倾向，如：多用句号少用逗号，短促有力]

## 何时加载分片
- 写战斗 → `场景描写/战斗.md` + `场景描写/动作.md`
- 写对话 → `场景描写/对话.md` + 涉及内心独白加 `心理.md`
- 写揭谜/悬念 → `场景描写/信息释放.md`
- 写场景切换 → `场景描写/过渡与节奏.md`
- 不论写什么 → 定稿前跑 `自检清单.md`
```

```markdown

## 一、铁律检查（每条违反-10分，0容忍）
```

## 知识库迭代规则

### 初始化
- 项目冷启动时，产出**最小可用集**（总纲+3类核心场景分片+自检清单骨架）
- 可手动构建，或调用novel-style-extract从参考小说提取

### 运行中演化
- **新增分片**：novel-lite发现新场景但分片缺失 → novel-setup创建骨架 → novel-style-extract增量提取
- **调整阈值**：novel-review阶段回顾发现"自检80分但风格漂移" → 调松阈值
- **沉淀正反例**：novel-review发现风格问题 → 写入对应分片的正例/反例段

### ⚠️ 关键约束
- 铁律和自检阈值调整必须经用户确认
- novel-style-extract生成的内容必须先经用户确认再写入
- 正例优先使用本项目已定稿的优秀片段，其次才使用参考小说的抽象化提取

## 分片加载规则

novel-lite在阶段2构建必看文档时：

1. 先加载 `_总纲.md`（始终加载）
2. 根据本章预计场景类型，加载对应分片（如战斗.md、对话.md）
3. 加载 `自检清单.md`（提取自测项）
4. 加载上一章风格指纹（N>1时）
5. 将上述内容浓缩为 `02_必看文档.md`

**不加载全量知识库**——只按需加载，避免token浪费。

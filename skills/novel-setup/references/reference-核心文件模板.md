# 核心文件模板

本模板提供宏观设定文件的详细格式。创建或更新时按此格式填写。

---

## 1. 故事大纲.md

```markdown
# 故事大纲

## 一句话梗概
[30字内，故事的灵魂，如："一个落魄少年在修仙界逆袭，却发现逆袭的代价是失去人性"]

## 核心冲突
- 主要冲突：[主角vs谁/什么，具体矛盾，如：主角vs宗门制度，想要公平却被打压]
- 深层冲突：[价值观/身份层面的撕扯，如：追求力量 vs 守护人性]

## 主题
[故事想表达什么，1-2句，如："真正的强大不是碾压他人，而是不被碾压"]

## 类型/题材
[玄幻/都市/科幻/悬疑...，如：东方玄幻，偏向热血成长]

## 整体结构

### 第一卷：[卷名]（预计第1-30章）
- 核心事件：[卷级主线事件，如：主角入门宗门，通过考核，遭遇背叛]
- 主角变化：[起点状态→终点状态，如：迷茫少年→决心复仇的入门弟子]
- 关键章节：
  - 第1章：[一句话预期，如：主角登场，展示处境困境]
  - 第2章：[一句话预期]
  - 第3章：[一句话预期]
  - ...（列出预计的转折点章节）
- 本卷末钩子：[引向下一卷的悬念，如：发现背叛者的真实身份]

### 第二卷：[卷名]（预计第31-60章）
- 核心事件：[事件]
- 主角变化：[变化]
- 关键章节：...

### 第三卷：...

## 主线与支线
- 主线：[主线名称] — 状态：推进中
- 支线A：[支线名称] — 状态：待启动/推进中/已完结
- 支线B：...

## 结局方向
[大致走向，可多版本并存]
- 版本1：[结局描述]
- 版本2：[结局描述]（备选）
```

## 2. 主角档案.md

```markdown
# 主角档案：[姓名]

## 基本画像
- 年龄：[具体年龄或区间]
- 外貌：[核心特征，2-3句]
- 身份/职业：[故事开始时的身份]
- 性格底色：[3-5关键词，如：倔强、隐忍、重情义、有些自卑]
- 口头禅/习惯动作：[如有，如："总是下意识握紧拳头"]

## 成长轨迹（弧光）
- 起点状态：[故事开始时的内在状态，心境+处境]
- 转折节点：
  - 节点1：[预计第X章，事件→变化，如：第10章遭遇背叛→不再相信任何人]
  - 节点2：[预计第Y章，事件→变化]
  - 节点3：...
- 终点状态：[故事结束时期望达到的状态]

## 当前状态（随剧情更新）
> 最后更新：第X章定稿后

- 当前章节：第X章
- 力量等级/状态：[具体参数]
- 心境：[当前心境描述]
- 主要目标：[短期目标]
- 深层渴望：[长期追求]
- 最大恐惧/软肋：[核心弱点]

## 重要人际关系
| 角色 | 关系类型 | 当前状态 | 备注 |
|------|---------|---------|------|
| [角色A] | [师徒/挚友/敌对] | [信任/疏远/对抗] | [关键事件] |
| [角色B] | ... | ... | ... |

## 主角已知/未知清单（核心真相）
> ⚠️ 创作时严格遵守：正文只能展示主角"已知"的信息

| 真相项 | 状态 | 获知/预计章 | 备注 |
|--------|------|------------|------|
| [真相1：宗门秘密] | ❌ 未知 | 预计第20章 | 切勿提前透露 |
| [真相2：某人的真实身份] | ❌ 未知 | 预计第35章 | 切勿提前透露 |
| [信息3：某技能的代价] | ✅ 已知 | 第5章获知 | 可展示 |
```

## 3. 故事状态.md

```markdown
- 主角所在：[具体地点]
- 当前环境：[天气/氛围，如有]

## 主角当前状态
- 力量等级：[具体参数]
- 心境：[一句话描述]
- 身体状态：[健康/受伤/疲惫]
- 携带关键物品：[列出]
- 最近获得的认知：[本章或最近几章新知道的事]

## 活跃支线
| 支线 | 当前进度 | 下一步 |
|------|---------|--------|
| [支线A] | [进度描述] | [预计动作] |
| [支线B] | ... | ... |

## 活跃人物
| 人物 | 位置 | 与主角关系 | 当前目的 |
|------|------|----------|---------|
| [角色A] | [在主角身边/某处] | [关系状态] | [目的] |
| [角色B] | ... | ... | ... |

## 未解决的悬念
| 悬念 | 埋设章节 | 计划揭晓章节 |
|------|---------|------------|
| [悬念1] | 第X章 | 第Y章 |
| [悬念2] | ... | ... |

## 下一章定位
- 预计发生：[大致事件]
- 需推进主线/支线：[列出]
- 需回收伏笔：[伏笔编号]
- 需新埋伏笔：[建议]
```

## 4. 伏笔追踪.md

```markdown
# 伏笔追踪

## 图例
- 🟢 已回收 — 揭晓完毕
- 🟡 进行中 — 已铺垫但未揭晓
- 🔴 待回收 — 只埋设未铺垫
- ⚫ 已废弃 — 不再使用

## 伏笔列表

### F001 [伏笔简称]
- 状态：🔴 待回收
- 埋设位置：第X章（具体段落/事件）
- 埋设方式：[直述/暗示/道具/对白/人物反应]
- 埋设内容：[具体埋了什么]
- 计划回收：第Y章左右
- 回收方式：[预想的揭晓方式]
- 关联设定：[相关实体/规则]
- 强化记录：[如有强化，列出章节]

### F002 [伏笔简称]
- 状态：🟡 进行中
- 埋设位置：第X章
- 强化位置：第Y章、第Z章
- 计划回收：第W章
- ...

### F003 [伏笔简称]
- 状态：🟢 已回收
- 埋设位置：第X章
- 回收位置：第Y章
- 回收方式：[实际揭晓方式]
- 效果评估：[好/一般/需改进]
```

## 5. 当前冲突.md

```markdown
# 当前冲突

> 最后更新：第X章定稿后
> 本冲突起始于：第Y章
> 预计延续至：第Z章左右

## 一句话冲突
[主角vs谁/什么/必须解决什么/否则怎样，30字内]

## 冲突结构

### 表层冲突
- 对手/障碍：[具体的人/势力/局面]
- 主角的目标：[要达成什么]
- 时限/压力：[多久内必须解决，不解决会怎样]

### 深层冲突
- 价值观层面：[主角内在的撕扯]
- 与弧光的关系：[本冲突如何推动主角成长节点]

## 双方筹码与风险
| 方 | 持有筹码 | 软肋 |
|----|---------|------|
| 主角 | [资源、信息、能力、关系] | [弱点] |
| 对手 | [威胁、信息差、资源优势] | [弱点] |

## 推进节拍
> 用于判断每章是否在推进此冲突

| 节拍 | 内容 | 预计章节 | 状态 |
|------|------|---------|------|
| 节拍1 | [事件/认知/对抗] | 第Y+?章 | ⬜ 待推进 |
| 节拍2 | [事件] | 第Y+?章 | ⬜ 待推进 |
| 节拍3 | [事件] | 第Y+?章 | ⬜ 待推进 |
| 节拍4 | [高潮/转折] | 第Y+?章 | ⬜ 待推进 |
| ✅ 已完成 | 第X章 [内容] | — | 已完成 |

## 演化预案
- 若主角达成：[下一段冲突预想]
- 若主角失败：[下一段冲突预想]
- 若中途转化：[可能的走向]

## 并行冲突（如有）
- 冲突B：[简述] — 状态：活跃/待触发/已归档
  - 与主线冲突的交互：[促进/牵制/转化]

## 历史冲突链
| 时间 | 冲突简述 | 结局 |
|------|---------|------|
| 第A章—第B章 | [上一轮冲突] | 达成/失败/转化 |
```

## 6. 写作注意事项.md

```markdown
# 写作注意事项

## 视角规则
- 主视角：[第一人称/第三人称限制视角]
- 切换规则：[禁止切换/允许但有条件]
- ⚠️ 核心：正文只能展示主角能感知到的事物

## 信息对称规则
- 读者认知 ≤ 主角认知（或说明例外）
- 主角"未知"的真相，正文不能直接展示
- 可以通过侧面暗示（如其他角色的反应），但不能让读者比主角知道更多核心真相

## 章节长度
- 目标范围：[如2500-4000字]
- 最短不低于：[如2000字]
- 最长不超过：[如5000字]

## 文风禁忌
- 禁止词汇：[具体列出，如：现代梗、网络用语、特定敏感词]
- 禁止表达方式：[如：过度bl描写、说教式长对白]
- 必须保持：[如：古风白话调、冷峻风格]

## 情节禁忌
- 禁止方便性桥段：[如："主角刚好想起某事"需要铺垫]
- 禁止数值膨胀：[如：等级比拼必须有具体动作，不能只说"A级vsB级"]
```

## 更新频率

| 文件 | 更新时机 |
|------|---------|
| 故事大纲.md | 大纲调整时、阶段回顾发现偏离时 |
| 主角档案.md | 弧光节点推进时、定稿后状态回写 |
| 故事状态.md | **每章定稿后** |
| 当前冲突.md | **每章定稿后审视，每5章必更新** |
| 伏笔追踪.md | 新埋/回收/强化时 |
| 写作注意事项.md | 发现新约束时 |

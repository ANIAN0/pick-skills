# 冷启动流程（全新项目）

采用访谈式引导，5轮完成项目基础建设。每轮最多3-4个问题，用户说"不确定"用占位`[待定]`，不阻塞推进。

---

## Round 1 — 故事灵魂

询问并记录到 `故事大纲.md`：

**问题清单**：
1. 一句话梗概：这个故事讲什么？（30字内）
2. 核心冲突：主角vs谁/什么？矛盾的本质是什么？
3. 想表达的主题：读完之后读者应该有什么感受？
4. 类型/题材：玄幻/都市/科幻/悬疑...？有特定的风格倾向吗？

**记录格式**：
```markdown
# 故事大纲

## 一句话梗概
[用户回答]

## 核心冲突
- 主要冲突：[用户回答]
- 深层冲突：[用户回答，如价值观层面]

## 主题
[用户回答]

## 类型/题材
[用户回答]
```

---

## Round 2 — 世界观

询问并记录到 `世界设定.md`：

**问题清单**：
1. 世界规则：这个世界运行的底层逻辑是什么？（魔法体系、科技设定、社会规则）
2. 势力结构：有哪些主要势力/组织/阶层？它们的关系是什么？
3. 地理与时间：故事发生在什么地点？时间背景是古代/现代/未来？
4. 独特卖点：这个世界有什么是其他作品没有的？

**记录格式**：
```markdown
# 世界设定

## 底层规则
[用户回答：具体规则体系]

## 势力结构
- [势力A]：[简介、立场、与主角关系]
- [势力B]：...

## 地理与时间
- 主要地点：...
- 时间背景：...

## 独特卖点
[用户回答]
```

---

## Round 3 — 主角

询问并记录到 `主角档案.md`：

**问题清单**：
1. 基本画像：姓名、年龄、外貌、身份、职业
2. 性格底色：3-5个关键词描述核心性格
3. 起点状态：故事开始时主角的内在状态（心境、处境、困境）
4. 期望终点：故事结束时主角应该变成什么样？（弧光终点）
5. 最大软肋：主角最大的恐惧/弱点是什么？

**记录格式**：
```markdown
# 主角档案：[姓名]

## 基本画像
- 年龄/外貌/身份：[用户回答]
- 性格底色：[3-5关键词]
- 口头禅/习惯动作：[如有]

## 成长轨迹（弧光）
- 起点状态：[内在状态]
- 终点状态：[期望达到的状态]
- 转折节点：[待定，后续补充]

## 最大软肋
[用户回答]

## 当前状态（初始）
- 当前章节：第0章
- 心境：[起点心境]
- 主要目标：[短期目标]
```

---

## Round 4 — 大纲骨架

询问并完善 `故事大纲.md`：

**问题清单**：
1. 预计多少卷/幕？
2. 第一卷的核心事件是什么？主角在第一卷会有什么变化？
3. 第一卷大致多少章？关键章节是哪些？
4. 结局大致方向是什么？（可模糊）

**记录格式**（追加到大纲）：
```markdown
## 整体结构

### 第一卷：[卷名]（预计第1-X章）
- 核心事件：[事件]
- 主角变化：[起点→终点]
- 关键章节：
  - 第1章：[一句话预期]
  - 第2章：[一句话预期]
  - ...

### 第二卷：[卷名]
- [待定]

## 结局方向
[用户回答，可多版本并存]
```

## Round 5 — 写作约束 + 创作风格知识库初始化

### 5A. 硬约束

询问并记录到 `写作注意事项.md`：

**问题清单**：
1. 视角：第一人称？第三人称限制视角？是否允许切换？
2. 信息对称：读者能知道主角不知道的事吗？
3. 章节长度：目标字数范围？
4. 文风禁忌：有什么词/表达方式绝对不能用？

**记录格式**：
```markdown
# 写作注意事项

## 视角规则
- 主视角：[第一/第三限制]
- 切换规则：[允许/禁止/限定条件]

## 信息对称规则
- 读者认知 ≤ 主角认知（或说明例外）

## 章节长度
- 目标范围：[如2500-4000字]

## 文风禁忌
- 禁止出现：[现代梗/网络用语/...]
```

### 5B. 创作风格知识库初始化

询问用户是否有参考小说。提供两种方式：

---

**方式一：手动构建（无参考小说或用户自己定义风格）**

**问用户**：你想让这本书读起来是什么感觉？

**操作步骤**：

1. 让用户回答"文风灵魂"：一句话描述这本书应该有的阅读感
2. 让用户列出3-5条"核心铁律"：写正文时必须遵守的规则
3. 问用户：这本书最常出现哪3-5类场景？（战斗/对话/心理/探索/信息释放...）
4. 为每类场景创建骨架文件（只填目标/必做/必避，正反例空着）

**产出**：
- `_总纲.md`：文风灵魂 + 铁律 + 词汇基调（用户自定）
- `场景描写/<场景>.md`：骨架版（目标/必做/必避）
- `自检清单.md`：基于铁律生成最小版
- 空的 `风格指纹/` 目录

---

**方式二：从参考小说提取（推荐）**

**问用户**：有没有参考小说？风格上想借鉴哪本书？

**关键操作**：
1. 用户提供参考小说路径或粘贴片段
2. **调用 novel-style-extract 进行全量提取**
3. 用户确认后写入 `相关设定/创作风格/`

**⚠️ 重要**：novel-style-extract 会执行**抽象化处理**——提取的是"写作技法规则"而非"原书具体内容"。例如：
- ✅ 可提取："战斗场景要有代价交代、要有主角失误"
- ✅ 可提取："对话用短句推进，避免说教式长对白"
- ❌ 不提取："主角使用剑法时要描写招式名称"（这是武侠题材特定内容）
- ❌ 不提取："打坐修炼的描写方式"（这是原书世界观设定）

**如果用户的参考小说题材与本项目不同**（如参考武侠但写科幻），novel-style-extract会：
- 只提取通用技法规则
- 标注题材特定规则为"[题材绑定，需用户确认是否适用]"
- 用户可以选择只采用通用部分

## 冷启动后的首次开写

冷启动完成后，用户准备写第1章时：

1. **novel-lite请求本skill出具第1章开写简报**
2. 本skill生成简报（衔接点改为"故事起点与首段冲突引入方式"）
3. novel-lite进入阶段1（创作方向）→ 阶段2（必看文档）→ 阶段3（创作）

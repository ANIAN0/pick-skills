# 冷启动流程（场景1：全新项目）

采用**访谈式引导**，分 5 轮完成：

**Round 1 — 故事灵魂**
询问并记录到 `故事大纲.md`：
1. 一句话梗概
2. 核心冲突是什么
3. 想表达的主题
4. 类型/题材

**Round 2 — 世界观**
询问并记录到 `世界设定.md`：
1. 世界规则（魔法/科技/超自然的底层逻辑）
2. 社会结构（势力/阶层/组织）
3. 地理与时间背景
4. 世界的"独特卖点"

**Round 3 — 主角**
询问并记录到 `主角档案.md`：
1. 基本画像
2. 性格底色
3. 起点状态与期望终点（弧光）
4. 最大软肋

**Round 4 — 大纲骨架**
询问并完善 `故事大纲.md`：
1. 预计多少卷/幕
2. 第一卷大致走向与关键事件
3. 结局方向（可模糊）

**Round 5 — 写作约束 + 创作风格知识库初始化** ⭐

**5A. 硬约束（写入 `写作注意事项.md`）**
1. 视角规则
2. 信息对称规则
3. 章节长度目标

**5B. 创作风格知识库初始化（创建 `创作风格/`）**

提供两种方式（用户选择）：

**方式一：手动构建（用户无参考小说或想自己定义）**
1. 文风灵魂一句话 + 3-5 条核心铁律 → `_总纲.md`
2. 列出本书最常出现的 3-5 类核心场景 → 为每类创建 `场景描写/<场景>.md`，先填"目标 / 必做 / 必避"骨架
3. 用 _总纲 的核心铁律和场景描写的必做项生成最小可用版 `自检清单.md`
4. 创建空的 `风格指纹/` 目录

**方式二：从参考小说提取（Recommended）⭐**
1. 询问用户是否有参考小说
2. 如有 → 调用 **novel-style-extract** 进行全量提取
   - 用户提供参考小说文件路径或粘贴片段
   - novel-style-extract 分析并生成 `_总纲.md` + `场景描写/*.md` + `自检清单.md`
   - 用户确认后写入 `相关设定/创作风格/`
3. 如无 → 回退到方式一

> 方式二的优势：提取出的风格规则有具体文本支撑，正例直接来自参考小说，量化指标有统计数据基础。

**冷启动完成后**生成：
- 初始化的 `故事状态.md`（指向"第0章，准备开写第1章"）
- 初始的 `当前冲突.md`（首段冲突，对应第一卷开篇 3-10 章的核心矛盾）
- 空白 `伏笔追踪.md`
- 创作风格知识库最小可用集

**原则**：每轮最多问 3-4 个问题。用户说"不确定"先用占位"[待定]"，不阻塞推进。

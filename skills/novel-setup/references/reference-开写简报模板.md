# 开写简报模板

novel-lite启动新章时，**第一步**请求本skill出具开写简报。

简报写入 `工作区/第X章VX/_预创作/00_开写简报.md`，后续阶段只需读文件，不重复加载宏观设定，节省token。

---

## 产出模板

```markdown
# 第N章 开写简报

> 生成时间：YYYY-MM-DD HH:MM
> 由 novel-setup 产出，作为 novel-lite 4阶段流程的阶段0输入。

## 一、当前进展回看（≤200字）
[一段话总结故事走到哪了]
- 主角刚刚做了什么：
- 处于什么状态：
- 上一章结尾留了什么钩子：

## 二、本章在大纲中的定位
- 所在卷/幕：[第一卷]
- 大纲预期事件：[一句话]
- 对主线推进：[如何推进主线]
- 与上一章衔接：
  - 第1章特殊处理：无上一章，此项为"故事起点与首段冲突引入方式"

## 三、当前冲突状态
> 摘自当前冲突.md

- 一句话冲突：[30字内]

- **本章应推进的下一节拍**：[节拍编号 + 建议兑现形态]

- 所在地点：[地点]
```

## 第1章开写简报特殊处理

第1章无"上一章"概念，以下字段需调整：

| 字段 | 第1章处理 |
|------|----------|
| 当前进展回看 | 替换为"故事起点：主角处于[状态]，面临[困境]" |
| 与上一章衔接 | 替换为"故事起点与首段冲突引入方式" |
| 本章应推进的下一节拍 | 使用当前冲突.md的首个节拍 |
| 上一章定稿 | 不加载 |
| 风格指纹 | 不加载，标注"基准章，无比对对象" |

## 简报产出流程

1. **读取宏观文件**：故事状态.md、当前冲突.md、故事大纲.md、伏笔追踪.md、写作注意事项.md
2. **读取主角档案**：当前状态、弧光节点、已知/未知清单
3. **读取风格知识库索引**：确定本章预计场景类型对应的分片
4. **判断衔接点**：N>1时读取上一章定稿三件套；N=1时使用起点状态
5. **生成简报**：按模板写入文件
6. **通知novel-lite**：简报已产出，可进入阶段1

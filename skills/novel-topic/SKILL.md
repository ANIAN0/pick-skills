# Novel Topic Skill

用于完成“选题确认”阶段。这个技能只解决一件事：把题材和核心方向定下来，并得到用户明确确认。

## 核心职责（仅本阶段）

1. 生成候选选题方案（每轮5个）
2. 给出推荐选项及一句话理由
3. 根据用户反馈重写下一轮
4. 记录最终确认的选题

## 交互规则（强制）

每轮必须包含：
1. 5 个互斥选项（A-E）
2. 1 个推荐选项（说明推荐理由）
3. 引导用户选择或改写

只有当用户明确说“确认/定稿/就这个”时，才算选题确认完成。

## 输出文件

`00-选题/topic.md`

```markdown
# 选题定稿

## 最终确认
- 题材：
- 子题材：
- 核心主题：
- 主角基本设定：
- 目标读者：

## 选题一句话说明
[直接表达，不使用修辞]

## 确认记录
- 用户确认语句：
- 确认时间：
```

## 工作流程

1. 收集用户偏好（题材、核心冲突、主角类型）
2. 输出 5 个选题候选 + 1 个推荐
3. 根据反馈多轮迭代
4. 用户明确确认后写入 `topic.md`
5. 将 `topic_confirmed` 置为 `true`

## 技能调用

**前置条件：** 项目已初始化  
**后置操作：** 引导进入 `novel-logline`（三句话故事阶段）

## 注意事项

1. 不替用户拍板，最终选择权始终在用户。  
2. 不要提前进入世界观、角色、大纲等阶段。  
3. 表达必须直接，不用文学修辞。

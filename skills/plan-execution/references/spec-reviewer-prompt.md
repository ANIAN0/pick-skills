# Spec合规性审核 Subagent 提示词模板

委派Spec合规性审核subagent时使用此模板。

**目的**：验证实现者构建的内容符合规范（不多不少）

**只在实现者完成后委派。**

```
Agent tool (general-purpose):
  description: "审核 Task N 的Spec合规性"
  prompt: |
    你正在审核实现是否符合规范。

    ## 要求内容

    {完整复制任务规范要求}

    ## 实现者声称构建的内容

    {从实现者报告复制}

    ## 关键：不要相信报告

    实现者可能完成得异常快。报告可能不完整、不准确或过于乐观。
    你必须独立验证一切。

    **不要**：
    - 相信他们说的实现了什么
    - 相信他们声称的完整性
    - 接受他们对需求的解释

    **要**：
    - 阅读他们写的实际代码
    - 逐行对比实际实现和需求
    - 检查他们声称实现但缺失的部分
    - 查找他们未提及的额外功能

    ## 你的工作

    阅读实现代码并验证：

    **缺失的要求**：
    - 是否实现了所有请求的内容？
    - 是否有跳过或遗漏的要求？
    - 是否声称某功能工作但实际未实现？

    **额外/不需要的工作**：
    - 是否构建了未请求的内容？
    - 是否过度工程化或添加不必要功能？
    - 是否添加了未在规范中的"锦上添花"？

    **误解**：
    - 是否以不同方式解释了需求？
    - 是否解决了错误的问题？
    - 是否以正确方式实现了正确功能？

    **通过阅读代码验证，而非相信报告。**

    报告：
    - ✅ Spec合规（代码检查后一切匹配）
    - ❌ 发现问题：[具体列出缺失或多余内容，附file:line引用]
```

## 处理审核结果

**发现问题**：
- 让同一实现subagent修复
- 重新进行Spec审核
- 循环直到通过

**绝不要**接受"差不多符合"Spec。审核发现问题 = 未完成。

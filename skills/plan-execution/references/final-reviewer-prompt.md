# 最终审核 Subagent 提示词模板

委派最终审核subagent时使用此模板。

**目的**：验证所有任务合在一起满足需求文档的验收标准（全局视角，弥补单任务审核的局部盲区）

**只在所有任务的状态均为"完成"后委派。**

```
Agent tool (general-purpose):
  description: "全局最终审核：验证整体实现满足需求验收标准"
  prompt: |
    你正在做最终审核。所有任务已单独完成并通过了各自的Spec审核和代码质量审核。
    你的任务是从全局视角检查：**整体实现是否满足需求文档的验收标准**。

    ## 需求文档验收标准

    {从需求文档的"验收标准"章节复制完整内容}

    ## 任务清单（所有任务已完成）

    {复制任务清单的执行索引表格}

    ## 审核重点

    **验收标准覆盖**：
    - 需求文档中每条验收标准是否都有对应的实现？
    - 是否有验收标准被遗漏，未被任何任务覆盖？
    - 功能是否可以端到端走通，还是存在断点？

    **任务间集成**：
    - 各任务的实现是否能正确衔接？（接口、数据格式、依赖关系）
    - 是否存在任务A和任务B各自通过审核，但组合后出现问题的情况？

    **遗漏检测**：
    - 有没有"每个任务都没问题，但整体缺了一块"的情况？
    - 错误处理、边界情况、用户反馈等横切关注点是否完整？

    **不要**：
    - 重复单任务的Spec审核（那已经完成了）
    - 重新审查代码质量细节（那也已经完成了）
    - 相信任务完成报告——独立验证端到端流程

    **要**：
    - 对照需求文档验收标准逐条核查
    - 实际运行或追踪关键用户路径
    - 发现跨任务的集成问题

    报告：
    - ✅ 通过（所有验收标准满足，集成无问题）
    - ❌ 发现问题：[具体描述缺失或集成问题，指出哪条验收标准未满足]
```

## 处理审核结果

**发现问题**：
- 分析问题属于哪个任务的责任范围
- 重新打开该任务（状态改回"待执行"），委派实现subagent修复
- 修复后重新进行最终审核

**通过**：
- 通知用户所有验收标准已满足，可进行 git 提交

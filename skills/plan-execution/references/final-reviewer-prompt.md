# 最终审核 Subagent 提示词模板

委派最终审核 subagent 时使用此模板。

**目的**：从全局视角验证整体实现满足需求文档的验收标准（弥补单模块审核的局部盲区）。
**范围**：仅做**集成核对**，不执行端到端测试（本套 skill 默认不做 E2E）。

**只在所有模块状态均为"完成"后委派。**

```
Agent tool (general-purpose):
  description: "全局最终审核：验证整体实现满足需求验收标准（不跑 E2E）"
  prompt: |
    你正在做最终审核。所有模块已完成并通过了各自的模块审核。
    你的任务是从全局视角检查：**整体实现是否满足需求文档的验收标准**。

    ## 需求文档验收标准

    {从需求文档"验收标准"章节复制完整内容}

    ## 模块清单（全部完成）

    {复制计划文件的执行索引表格}

    ## 审核重点

    **1. 验收标准覆盖**
    - 需求文档每条验收标准是否都有对应实现？
    - 是否有验收标准被遗漏，未被任何模块覆盖？

    **2. 模块间集成（静态核对，非端到端运行）**
    - 各模块对外接口/数据格式是否一致？
    - 前端调用的 API 字段名/类型是否与后端一致？
    - 是否存在"模块各自通过审核但组合后会出问题"的迹象？

    **3. 横切关注点**
    - 错误处理、边界情况、用户反馈是否完整？
    - **前端覆盖**：需求文档"页面清单"中的每个页面是否都已实现并可访问？

    **4. 测试整体性**
    - `workplace/1.X/test/` 下是否覆盖：后端单元、后端集成（关键路径）、前端单元/组件？
    - 是否有 README.md 列出运行命令？覆盖率是否达标？
    - **是否出现 e2e/ 目录或端到端测试文件？若有，标记为问题（本套 skill 不做 E2E）**

    **不要**：
    - 重复单模块的审核（已完成）
    - 重新审查代码质量细节（已完成）
    - 实际运行端到端流程或模拟完整用户路径（不在本范围）
    - 编写或要求新增端到端测试

    **要**：
    - 对照需求验收标准逐条核查
    - 通过阅读关键模块的对外接口/调用点，发现跨模块的集成问题
    - 阅读 `workplace/1.X/test/README.md` 确认测试套件完整性

    报告：
    - ✅ 通过（验收标准全覆盖，接口一致，测试套件完整，无 E2E 残留）
    - ❌ 需修复（列出问题，按模块归类，标记前端/后端/测试问题）

    如果需修复，列出：
    - [模块名]：[问题描述] → [建议修复方式]
```

# 代码质量审核 Subagent 提示词模板

委派代码质量审核subagent时使用此模板。

**目的**：验证实现构建良好（干净、可测试、可维护）

**只在Spec合规性审核通过后委派。**

```
Agent tool (general-purpose):
  description: "审核 Task N 的代码质量"
  prompt: |
    你正在审核实现的代码质量。

    ## 实现内容

    {从实现者报告复制}

    ## 任务来源

    {任务简要描述}

    ## 审核重点

    **文件组织**：
    - 每个文件是否有单一清晰职责和定义良好的接口？
    - 单元是否分解为可独立理解和测试？
    - 是否遵循计划中定义的文件结构？

    **变更影响**：
    - 新创建的文件是否已经过大？
    - 是否显著增大了已有文件？（关注本次变更的贡献，不要标记已有文件的大小）

    **代码质量**：
    - 命名是否清晰准确？
    - 是否遵循代码库现有模式？
    - 是否整洁可维护？

    报告：
    - **优点**：[列出做得好的地方]
    - **问题**（Critical/Important/Minor）：[具体问题列表]
    - **评估**：Approved | Needs Fix
```

## 处理审核结果

**发现问题**：
- 让同一实现subagent修复
- 重新进行代码质量审核
- 循环直到Approved

**绝不要**跳过代码质量审核或让实现者自检替代它。

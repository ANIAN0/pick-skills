# 实现者 Subagent 提示词模板

委派模块实现subagent时使用此模板。

**目的**：让subagent连续实现模块内的所有子步骤

```
Agent tool (general-purpose):
  description: "实现 模块N: {模块名称}"
  prompt: |
    你正在实现 模块N: {模块名称}

    ## 模块目标

    {完整复制模块详情中的目标、前置依赖}

    ## 子步骤清单（按顺序完成）

    {完整复制模块详情中的所有子步骤}

    ## 接口契约

    {完整复制模块边界与接口契约}

    ## 上下文

    {说明此模块在整体中的位置、依赖关系、架构背景}

    ## 开始前确认

    如果对以下内容有疑问：
    - 模块目标或验收标准
    - 实现策略
    - 依赖或假设
    - 子步骤描述中不清楚的内容
    - 接口契约不明确的地方

    **现在提问。** 有疑问就提出来，不要猜测或假设。

    ## 你的工作

    确认需求后：
    1. **按顺序**完成所有子步骤
    2. 每个子步骤完成后**立即运行相关测试**（若子步骤有测试要求）
    3. 所有子步骤完成后，运行**模块级测试**
    4. 自检（见下方）
    5. 报告结果（见下方格式）

    **工作目录**：{目录路径}

    **重要**：不要提交git，保持变更在工作目录中。

    ## 代码组织

    你能最好地理解一次就能装入上下文的代码，编辑也更可靠当文件聚焦时。记住：
    - 遵循计划定义的文件结构
    - 每个文件应有单一清晰职责和定义良好的接口
    - 模块内保持接口一致，确保对外暴露的契约与模块定义一致
    - 如果创建的文件超出计划意图变大，停止并报告DONE_WITH_CONCERNS - 不要自行拆分
    - 如果修改的已有文件已经很大或混乱，小心处理并在报告中作为担忧说明

    ## 自检清单

    完成所有子步骤后，逐条检查：

    - [ ] 所有子步骤都已实现
    - [ ] 每个子步骤的功能都经过验证（手动测试或单元测试）
    - [ ] 模块对外接口符合契约定义
    - [ ] 没有构建未请求的内容（YAGNI）
    - [ ] 代码遵循项目现有模式
    - [ ] 没有TODO或FIXME遗留（或已在报告中列出）

    ## 报告格式

    完成后报告：

    ```
    ## 实现报告：模块N

    ### 子步骤完成度
    1. [子步骤1名称]：{完成/部分完成/未开始} {备注}
    2. [子步骤2名称]：{完成/部分完成/未开始} {备注}
    ...

    ### 测试情况
    - 子步骤测试：{X/Y 通过}
    - 模块级测试：{通过/失败} {详情}
    - 新增测试文件：[列表]

    ### 变更摘要
    - 新增文件：[列表]
    - 修改文件：[列表]
    - 删除文件：[列表]

    ### 对外接口
    - [接口名]：[描述]
    - [数据格式]：[描述]

    ### 自检结果
    - [ ] 全部通过
    - 未通过项：[列表或"无"]

    ### 担忧（如有）
    - [需要主agent注意的问题]

    ### DONE 或 DONE_WITH_CONCERNS
    ```

    如果有担忧，标记为 DONE_WITH_CONCERNS 而非 DONE。
```

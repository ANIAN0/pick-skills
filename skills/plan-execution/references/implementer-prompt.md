# 实现者 Subagent 提示词模板

委派模块实现 subagent 时使用此模板。

**目的**：让 subagent 连续实现模块内的所有子步骤，并通过模块验收标准验证。

```
Agent tool (general-purpose):
  description: "实现 模块N: {模块名称}"
  prompt: |
    你正在实现 模块N: {模块名称}

    ## 模块目标

    {完整复制模块详情中的目标、前置依赖}

    ## 对应功能点

    {完整复制模块对应的功能点列表（FP-N）}

    ## 对应测试点

    {完整复制模块对应的测试点列表（TC-N）}

    ## 子步骤清单（按顺序完成）

    {完整复制模块详情中的所有子步骤}

    ## 接口契约

    {完整复制模块边界与接口契约}

    ## 验收标准（模块完成后必须执行）

    {完整复制模块验收标准：
    - 如果是后端脚本：复制完整的py脚本内容
    - 如果是前端审查：复制完整的审查清单内容
    }

    ## 上下文

    {说明此模块在整体中的位置、依赖关系、架构背景}

    ## 开始前确认

    如果对以下内容有疑问：
    - 模块目标或验收标准
    - 实现策略
    - 依赖或假设
    - 子步骤描述中不清楚的内容
    - 接口契约不明确的地方
    - 验收脚本/清单中的检查项不清楚

    **现在提问。** 有疑问就提出来，不要猜测或假设。

    ## 你的工作

    确认需求后：
    1. **按顺序**完成所有子步骤
    2. 每个子步骤完成后**立即运行相关测试**（若子步骤有测试要求）
    3. 所有子步骤完成后，运行**模块验收标准**：
       - 后端模块：运行验收py脚本，必须输出ALL PASS
       - 前端模块：按审查清单逐项自检，必须全部勾选
    4. 自检（见下方）
    5. 报告结果（见下方格式）

    **源码工作目录**：{源码目录路径，例如 src/xxx/}
    **测试文件目录（强制）**：{workplace/1.X/test/<层>/<类型>/<本模块>/，X 已被主控替换为真实数字}
    **共享 fixtures 目录**：workplace/1.X/test/fixtures/

    ## 测试位置硬规则

    - 所有新增测试文件必须落在上面"测试文件目录"下，**不得**在源码旁创建测试（不得在 src 旁放 *.test.ts/*.spec.py 等）
    - 共享 mock/fixture 放到 `workplace/1.X/test/fixtures/`
    - 该目录尚未创建则先创建并补一个最小 README（若属本计划首个模块）
    - 路径中 `1.X` 必须是真实数字（如 `workplace/1/test/`）；若看到字面 `1.X`，停止并询问主控
    - **不要创建端到端（E2E）测试**：本套 skill 默认范围只到模块级单元/集成测试。如确有强烈需要，停止并询问主控

    ## 验收标准执行要求

    **后端模块**：
    - 模块完成后必须运行计划阶段定义的验收py脚本
    - 脚本必须全部PASS才算模块完成
    - 如果脚本失败，修复代码后重新运行，直到全部PASS
    - 报告结果时必须包含脚本完整输出

    **前端模块**：
    - 模块完成后必须按照计划阶段定义的审查清单逐项自检
    - 可以先用浏览器/截图工具自查，但必须在报告中诚实汇报
    - 如果清单项未通过，修复后继续自检，直到全部勾选
    - 报告结果时必须包含清单勾选状态

    ## 代码组织

    - 遵循计划定义的文件结构
    - 每个文件应有单一清晰职责和定义良好的接口
    - 模块内保持接口一致，对外暴露的契约与模块定义一致
    - 如果创建的文件超出计划意图变大，停止并报告 DONE_WITH_CONCERNS——不要自行拆分
    - 如果修改的已有文件已经很大或混乱，小心处理并在报告中作为担忧说明

    ## 自检清单

    完成所有子步骤后逐条检查：

    - [ ] 所有子步骤都已实现
    - [ ] 每个子步骤的功能都经过验证（手动测试或单元测试）
    - [ ] 模块对外接口符合契约定义
    - [ ] 没有构建未请求的内容（YAGNI）
    - [ ] 代码遵循项目现有模式
    - [ ] 没有 TODO/FIXME 遗留（或已在报告中列出）
    - [ ] **模块验收标准已通过**：
      - 后端：验收脚本输出 ALL PASS
      - 前端：审查清单全部勾选
    - [ ] **所有新增测试文件均位于 `workplace/1.X/test/` 下，源码旁无测试残留**
    - [ ] **未创建 E2E 测试或 e2e/ 目录**
    - [ ] **若本模块为前端模块，审查清单已全部自检通过**

    ## 报告格式

    完成后报告：

    ```
    ## 实现报告：模块N

    ### 子步骤完成度
    1. [子步骤1名称]：{完成/部分完成/未开始} {备注}
    2. [子步骤2名称]：{完成/部分完成/未开始} {备注}
    ...

    ### 功能点覆盖
    - FP-X：{已实现/未实现} {备注}
    - FP-Y：{已实现/未实现} {备注}
    ...

    ### 验收标准执行结果
    **后端脚本**：
    ```
    {粘贴脚本完整输出}
    ```
    结果：{ALL PASS / 部分失败}

    **前端审查清单**：
    - [ ] TC-X：{勾选/未勾选} {备注}
    - [ ] TC-Y：{勾选/未勾选} {备注}
    ...
    结果：{全部勾选 / 部分未通过}

    ### 测试情况
    - 子步骤测试：{X/Y 通过}
    - 模块级测试：{通过/失败} {详情}
    - 新增测试文件（必须位于 workplace/1.X/test/ 下）：
      - [文件1 完整路径]
      - [文件2 完整路径]
    - 测试运行命令：[实际运行的命令]

    ### 变更摘要
    - 新增文件：[列表]
    - 修改文件：[列表]
    - 删除文件：[列表]

    ### 对外接口
    - [接口名]：[描述]
    - [数据格式]：[描述]

    ### 自检结果
    - [ ] 全部通过
    - 未通过项：[列表或"无"]

    ### 担忧（如有）
    - [需要主 agent 注意的问题]

    ### DONE 或 DONE_WITH_CONCERNS
    ```

    如果有担忧，标记为 DONE_WITH_CONCERNS 而非 DONE。
    **如果验收标准未通过，不要标记DONE或DONE_WITH_CONCERNS，报告当前状态并等待主控指示。**
```

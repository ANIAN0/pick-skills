# Novel Creator Skill

用于管理整本小说的创作流程。这个技能的核心不是一次性输出大量内容，而是通过多轮确认，稳定产出可持续连载的内容。

## 核心目标

1. 让用户先把关键前置内容定稿，再进入后续创作。
2. 每一轮都提供可选方案，帮助用户快速做决策。
3. 保证剧情围绕当前目标推进，避免跑题和超前续写。
4. 记录用户明确喜欢和明确不喜欢的创作习惯，持续复用。

## 固定创作主流程

严格按以下顺序推进，不允许跳步：

1. 选题确认
2. 三句话故事确认（开头/高潮/结尾）
3. 第1章草稿确认（只看故事内容，不看文风）
4. 第1章正文确认（根据草稿确定风格）
5. 分卷确认（简化版）
6. 后续章节创作（必须经过前置检查）

## 交互规则（关键）

以下阶段必须使用同一套交互节奏：
- 三句话故事
- 第1章草稿
- 第1章正文（风格）

每轮都执行：
1. 给出 5 个候选选项
2. 明确标出 1 个推荐选项，并说明推荐理由（一句话即可）
3. 等待用户反馈后重写下一轮
4. 直到用户明确说出“确认/定稿/就这个”才进入下一阶段

如果用户没有明确确认，视为未确认，不得推进。

## 各阶段执行标准

### 阶段1：选题与三句话故事

完成选题后，立刻进入三句话故事确认：
- 开头一句：交代主角起点和直接目标
- 高潮一句：交代最大冲突和决定性选择
- 结尾一句：交代阶段性结果和新问题

语言必须直接，不用比喻、拟人、通感等修辞。

### 阶段2：第1章草稿（先内容后风格）

基于已确认的选题和三句话故事，先写第1章草稿：
- 草稿只写故事事实、行动链和冲突变化
- 不做文风强化，不写花哨表达
- 通过多轮“5选项+推荐”持续修改，直到用户明确确认草稿

### 阶段3：第1章正文与风格定稿

仅在草稿确认后执行：
- 基于同一故事内容生成不同写作风格版本
- 每轮给出 5 个风格选项 + 简短示例 + 推荐项
- 多轮重写直到用户明确确认第1章正文
- 将确认后的风格写入 `01-第1章/style-profile.md`

### 阶段4：分卷设计（简化版）

仅在第1章正文确认后执行。初期只做：
- 卷名
- 该卷内容一句话描述
- 章节数量目标

不做详细卷纲。

## 后续章节创作前置条件

后续章节创作（第2章及以后）必须同时满足：
1. 选题已确认
2. 第1章正文已确认
3. 分卷已确认

任一未完成时，不得直接写新章节，必须引导用户先补齐。

## 后续章节创作检查清单

满足前置条件后，每次开始写章节前都检查：

1. 前一章是否定稿  
   - 未定稿：先完成前一章
2. 当前章节是否已归属某个故事单元  
   - 未归属：先创建故事单元

## 故事单元规范

每个故事单元必须：
- 围绕一个【核心目标】
- 以一个强【单元卡点】收尾，能驱动下一单元
- 包含 2-4 个核心事件，形成起-承-转-合
- 单元总章节数控制在 8-15 章

拆分规则：
- 单一事件可拆 2-5 章
- 行动链每含“动机→执行→反馈→新决策”中的 2-3 个环节，可拆 1 章

## 分卷故事线与故事单元的关系

创建故事单元前先检查：当前卷是否已有分卷故事线。

若没有，先根据该卷的：
- 卷名
- 一句话描述
- 章节数量目标

拆分出合理数量的故事单元，构成分卷故事线。
非创作阶段的故事单元可先写一句话描述。

## 语言与叙事约束（强制）

以下内容必须保持“直接表达”，禁止文学性修辞：
- 开头/高潮/结尾三句话
- 分卷一句话描述
- 分卷故事线
- 故事单元一句话描述
- 故事单元详情
- 章节正文草稿

故事单元与章节正文必须严格服务于【当前单元】核心目标，并形成完整弧线：
- 起因
- 经过
- 结果
- 遗留问题（单元卡点）

单元卡点必须能被主角在下一单元作为核心动机直接处理。禁止超前续写和无关支线。

## 正文质量标准

正文必须同时满足：
1. 读者能清晰理解当前单元逻辑
2. 读者有继续追更的动力

执行要求：
- 删除冗余描写和无关设定
- 情节推进以“主角选择/行动”为核心
- 避免“事件自动发生”
- 允许生成少量小事件，仅用于补逻辑缺口或铺垫冲突
- 小事件必须完全嵌入主线，不得变成独立支线

## 偏好记录机制（新增）

在创作过程中维护两个文件：
- `99-偏好记录/positive-patterns.md`
- `99-偏好记录/negative-patterns.md`

记录原则：
- 只记录会直接影响创作结果的偏好
- 用户明确肯定的写法记入正向记录
- 用户明确否定的写法记入负向记录
- 与创作结果无关的信息不要记录

## 状态管理建议

`state.json` 至少包含以下布尔状态：

```json
{
  "topic_confirmed": false,
  "story_3lines_confirmed": false,
  "chapter1_draft_confirmed": false,
  "chapter1_final_confirmed": false,
  "volumes_confirmed": false
}
```

仅当状态为 `true` 才允许进入依赖阶段。

## 子技能调用路由（新版）

主流程按以下顺序调用：

1. `novel-topic`：选题确认  
2. `novel-logline`：三句话故事确认  
3. `novel-draft`：第1章草稿（纯故事内容）  
4. `novel-polish`：第1章正文定稿与风格沉淀  
5. `novel-volumes`：简化分卷确认  
6. `novel-storylines`：分卷故事线与故事单元  
7. `novel-chapter-plans`：逐章计划  
8. `novel-draft`：后续章节草稿  
9. `novel-polish`：后续章节定稿  
10. `novel-review`：章节审查与修正建议

## 可选扩展技能（不阻塞主流程）

以下技能为“可选增强”，默认不作为主流程前置：

- `novel-world`：世界观补充（按需、最小可用）
- `novel-characters`：角色库补充（按章节需要）
- `novel-outline`：全书宏观大纲（用户主动要求时）
- `novel-kb-update`：知识库同步（仅在项目已实现时）

调用这些技能时，必须满足一个原则：不能阻塞“选题 -> 第1章 -> 分卷 -> 故事单元 -> 章节创作”主链路。

## 目录与阶段对应

- `00-选题/`：选题与三句话故事  
- `01-第1章/`：第1章草稿、正文、风格档案  
- `02-分卷/`：分卷与分卷故事线  
- `03-故事单元/`：故事单元与单元详情  
- `04-正文/`：章节计划、草稿、定稿、审查  
- `99-偏好记录/`：正向/负向创作偏好

## 禁止项

1. 不要在说明中承诺未实现的 Turso/向量索引能力。
2. 不要在用户未明确确认时推进到下一阶段。
3. 不要在当前单元之外扩写剧情。

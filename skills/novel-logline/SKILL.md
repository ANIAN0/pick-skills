# Novel Logline Skill

用于完成“三句话故事确认”阶段。这里不再使用单句 logline，改为三句结构化描述。

## 三句话标准

1. 开头：主角起点 + 当前目标  
2. 高潮：最大冲突 + 决定性选择  
3. 结尾：阶段结果 + 新问题

## 交互规则（强制）

每轮都要提供：
- 5 组三句话候选（A-E）
- 1 组推荐方案（附一句推荐理由）
- 等待用户反馈后改写下一轮

用户未明确“确认”前，禁止推进到第1章草稿阶段。

## 输出格式

**产出文件:** `00-选题/story-3lines.md`

```markdown
# 三句话故事定稿

## 开头
[直接表达，不使用修辞]

## 高潮
[直接表达，不使用修辞]

## 结尾
[直接表达，不使用修辞]

## 确认记录
- 用户确认语句：
- 确认时间：
```

## 工作流程

1. 读取已确认选题 `00-选题/topic.md`
2. 生成 5 组三句话候选 + 1 组推荐
3. 多轮重写，直到用户明确确认
4. 写入 `story-3lines.md`
5. 将 `story_3lines_confirmed` 置为 `true`

## 注意事项

1. 三句话必须可直接用于后续第1章草稿。  
2. 表达必须直接，不用比喻、拟人、通感。  
3. 不要提前写第1章正文或分卷。

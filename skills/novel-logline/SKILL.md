# Novel Logline Skill

帮助创作者用一句话精准概括小说的核心故事，这是创作项目的关键锚点。

## 核心职责

1. **提炼核心** - 从杂乱的想法中提取故事本质
2. **构建结构** - 按照经典Logline结构组织要素
3. **优化表达** - 使一句话既简洁又引人入胜
4. **验证可行性** - 检查Logline是否具备故事潜力

## 什么是Logline

Logline是用一句话概括故事的艺术，包含以下核心要素：

```
[主角] + [主角特征] + [核心冲突] + [行动目标] + [独特卖点/转折]
```

**经典结构：**
- **Who** - 谁是主角？（带有关键特征）
- **What** - 他们想要什么？（目标）
- **What stands in their way** - 什么阻碍了他们？（冲突/对手）
- **What's at stake** - 赌注是什么？（后果）
- **Hook** - 独特卖点或转折

## 工作流程

### 1. 了解故事雏形

通过对话了解用户的基本想法，收集素材：

- 故事发生在什么世界/时代？
- 主角是谁？有什么特别之处？
- 主角想要达成什么？
- 什么/谁在阻止他们？
- 如果失败会怎样？
- 这个故事最吸引人的点是什么？
- 有什么转折或意外元素？

### 2. 提炼核心要素

将用户的描述整理成Logline要素：

```markdown
主角: [姓名/身份 + 关键特征]
目标: [想要达成的事情]
阻碍: [外部 antagonist 或内部冲突]
赌注: [失败的后果]
卖点: [独特之处/类型混合/反转]
```

### 3. 构建初稿

组合要素形成初稿：

```markdown
当[触发事件]发生时，[有特征的主角]必须[行动目标]，
否则[赌注/后果]，但[转折/阻碍]。
```

### 4. 迭代优化

与用户一起打磨：

- 是否过于笼统？（添加具体细节）
- 是否泄露太多？（保留悬念）
- 是否平淡无奇？（强化冲突和赌注）
- 是否太长？（删减冗余词汇）

### 5. 生成备选

提供2-3个不同角度的版本供选择。

### 6. 保存文档

将最终版本和备选保存到项目目录。

## 优秀Logline示例

**《饥饿游戏》：**
> 在反乌托邦的未来，一个自愿代替妹妹参加致命竞技的少女，必须在其他23名青少年的追杀中生存下来，同时还要对抗控制游戏的腐败政权。

**《肖申克的救赎》：**
> 一个被冤枉杀害妻子的银行家，在肖申克监狱的漫长岁月中，用一把小石锤挖了19年隧道，最终实现了自我救赎和复仇。

**《阿甘正传》：**
> 一个智商只有75的阿拉巴马男孩，凭借纯真和坚持，无意中参与了美国历史上几乎所有重大事件，最终成为百万富翁。

## 常见错误

| 错误类型 | 示例 | 改进 |
|----------|------|------|
| 过于笼统 | "一个英雄的冒险故事" | "一个失去记忆的刺客必须在72小时内找回自己的身份，否则将被自己曾经效力的组织追杀至死" |
| 泄露结局 | "...最后他牺牲了" | "...他必须做出最终的抉择" |
| 缺少冲突 | "一个女孩追求梦想成为歌手" | "一个失聪的女孩必须在声音被禁止的世界中，冒着生命危险追求音乐梦想" |
| 主角模糊 | "一群人探索神秘岛屿" | "一位考古学家必须在神秘岛屿上解开父亲失踪之谜，同时面对岛上守护的古老诅咒" |
| 缺少赌注 | "一个男人寻找宝藏" | "一位破产的父亲必须在女儿婚礼前找到传说中的宝藏，否则将永远失去女儿的尊重" |

## 技能调用

**何时调用：**
- 用户明确说"创作一句话故事"
- 用户说"用一句话概括我的故事"
- 用户需要提炼故事核心时
- 选题完成后，进入世界观创作前

**前置条件：**
- 项目已初始化
- 建议先完成选题（可选但推荐）

**后置操作：**
- 可选：调用 novel-world 进入世界观创作
- 可选：调用 novel-characters 进入角色创作
- 可与构思组其他技能并行执行

## 与其他技能的关系

```
novel-logline (可与构思组并行)
    ├── novel-world (独立)
    ├── novel-characters (独立)
    └── novel-outline (依赖构思组全部)
```

## 注意事项

1. **保持简洁** - 目标25-50个中文字符
2. **保留悬念** - 不要透露结局
3. **突出独特** - 强调与同类作品的区别
4. **可迭代调整** - Logline可以在创作过程中修改
5. **用户决定** - 最终版本由用户确定
